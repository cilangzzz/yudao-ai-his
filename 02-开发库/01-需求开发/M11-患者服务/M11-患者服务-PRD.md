## 1. 子系统概述

### 1.1 子系统定位

患者服务子系统是YUDAO-AI-HIS的患者自助服务入口，面向患者用户提供患者门户、预约挂号、报告查询、在线缴费、健康档案查看等自助服务功能。系统支持微信小程序、H5网页、自助机等多种接入渠道，实现患者"指尖上的医院"体验。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 便捷目标 | 减少患者现场排队等待 | 线上挂号占比≥50% |
| 效率目标 | 提升医院运营效率 | 线上缴费占比≥60% |
| 体验目标 | 提升患者就医体验 | 患者满意度≥90分 |
| 数字化目标 | 建立患者健康档案 | 健康档案覆盖率100% |

### 1.3 功能范围

```

#### 2.2.2 预约挂号流程

```

    ├── 支付挂号费 ──→ 微信支付/支付宝支付
    │
    └── 预约成功 ──→ 接收预约成功通知
    │
    └── 就诊当日 ──→ 签到 ──→ 进入候诊队列

## 3. 业务流程

### 3.1 患者注册登录全流程

```mermaid
flowchart TD
    A[打开患者门户] --> B{登录方式}
    B -->|微信| C[微信一键登录]
    B -->|手机| D[手机号+验证码登录]
    C --> E{是否已注册}
    E -->|是| F[直接登录成功]
    E -->|否| G[手机号验证]
    D --> G
    G --> H{是否实名认证}
    H -->|是| I[注册完成]
    H -->|否| J[实名认证]
    J --> K[身份证OCR识别]
    K --> L[人脸识别验证]
    L --> M{认证结果}
    M -->|成功| I
    M -->|失败| N[提示重新认证]
    N --> J
    I --> O[绑定就诊卡]
    O --> P[进入首页]
```

### 3.2 预约挂号流程

```mermaid
flowchart TD
    A[患者选择预约挂号] --> B[选择科室]
    B --> C[查看科室医生列表]
    C --> D[选择医生]
    D --> E[查看医生排班]
    E --> F[选择预约日期时间段]
    F --> G[确认预约信息]
    G --> H{是否需要支付}
    H -->|是| I[支付挂号费]
    H -->|否| J[直接预约成功]

    K -->|失败| L[提示重新支付]
    L --> I
    J --> M[发送预约成功通知]

    O --> P[扫码签到/定位签到]
    P --> Q[进入候诊队列]
    Q --> R[叫号就诊]
```
