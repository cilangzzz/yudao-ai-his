## 1. 功能概述

### 1.1 功能定位

预约挂号模块是患者服务子系统的核心功能，为患者提供在线预约挂号服务。系统支持科室查询、号源查询、预约挂号、预约管理、预约签到等全流程功能，通过微信小程序、H5网页、自助机等多渠道接入，实现患者"指尖上的医院"体验，减少现场排队等待。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 便捷目标 | 减少患者现场排队等待时间 | 线上预约挂号占比≥50% |
| 效率目标 | 提升医院号源利用率 | 号源利用率≥85% |
| 体验目标 | 提升患者就医体验 | 患者满意度≥90分 |
| 管理目标 | 降低患者爽约率 | 爽约率≤5% |

### 1.3 功能范围

```

           │

### 6.2 取消预约流程

```
患者进入我的预约

## 7. 状态机设计

### 7.1 预约状态流转

```
                    ┌─────────────────────────────────────────┐
                    │                                         │
                    ▼                                         │
┌────────────┐    ┌────────────┐    ┌────────────┐          │
│   创建     │───→│   已预约   │───→│   已签到   │          │
│            │    │   (1)      │    │   (2)      │          │
└────────────┘    └────────────┘    └────────────┘          │
                        │                                  │
                        │                                  │
                        │          ┌────────────┐          │
                        ├───┬─────→│   已取消   │          │
                        │   │      │   (3)      │          │
                        │   │      └────────────┘          │
                        │   │                               │
                        │   │      ┌────────────┐          │
                        └───┴─────→│   已过期   │──────────┘
                                   │   (4)      │
                                   └────────────┘

状态说明：
- 已预约(1): 预约成功，等待就诊
- 已签到(2): 患者到院签到成功，进入候诊队列
- 已取消(3): 患者主动取消预约
- 已过期(4): 预约时间过后未签到，自动过期

状态流转规则：
- 已预约 → 已签到：患者到院签到
- 已预约 → 已取消：患者主动取消（距预约时间≥24小时）
- 已预约 → 已过期：预约时间过后30分钟未签到
- 已签到后转为挂号记录，进入候诊状态
- 已取消、已过期状态不可变更
```

### 7.2 支付状态流转

```
┌────────────┐    ┌────────────┐    ┌────────────┐
│   待支付   │───→│   已支付   │───→│   已退款   │
│   (0)      │    │   (1)      │    │   (2)      │
└────────────┘    └────────────┘    └────────────┘
      │
      │
      ▼
┌────────────┐
│   已取消   │
│   (3)      │
└────────────┘

状态说明：
- 待支付(0): 预约创建，等待支付
- 已支付(1): 支付成功
- 已退款(2): 取消预约后退款
- 已取消(3): 支付超时或用户取消支付

状态流转规则：
- 待支付 → 已支付：支付成功
- 待支付 → 已取消：支付超时（10分钟）或用户取消
- 已支付 → 已退款：取消预约后退款
- 已取消、已退款状态不可变更
```

### 8.2 关键业务规则详解

#### BR-PT-006: 预约挂号实名校验

预约挂号必须完成实名认证并绑定就诊卡。

**触发时机**: 创建预约时

**校验逻辑**:
- 患者必须完成实名认证（real_name_verified = 1）
- 患者必须绑定至少一张就诊卡
- 患者必须设置默认就诊卡

**失败处理**: 返回错误原因并跳转对应页面

#### BR-PT-007: 号源可用性校验

预约时必须校验号源是否可用。

**触发时机**: 创建预约时

**校验逻辑**:
- 该医生当日有排班
- 号源剩余数量 > 0
- 预约日期在可预约范围内（7天内）

**失败处理**: 返回"号源已满"，显示同科室其他可选医生

#### BR-PT-008: 重复预约校验

同一患者同一天在同一科室只能有一个有效预约。

**触发时机**: 创建预约时

**校验逻辑**:
- 查询患者当日同科室有效预约数量
- 有效状态包括：已预约、已签到

**失败处理**: 返回"已预约同科室号源"错误

#### BR-PT-024: 每日预约次数限制

每位患者每日最多预约3次。

**触发时机**: 创建预约时

**校验逻辑**:
- 统计当日创建的预约数量
- 不包括已取消和已过期的预约

**失败处理**: 返回"达到每日预约上限"错误

#### BR-PT-016: 预约超时未签到处理

预约时间过后30分钟未签到，预约自动过期。

**触发时机**: 定时检查

**处理逻辑**:
- 标记预约状态为已过期
- 释放号源
- 发送过期通知
