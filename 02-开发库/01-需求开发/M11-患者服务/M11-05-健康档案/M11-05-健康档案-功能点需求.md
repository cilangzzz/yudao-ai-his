## 1. 功能概述

### 1.1 功能定位

健康档案是患者服务子系统的健康数据管理模块，为患者提供个人健康档案查看服务，包括诊疗历史、用药记录、过敏史、既往病史、健康指标记录、健康趋势分析等功能。系统支持患者手动录入健康指标，自动同步就诊数据，并提供AI健康建议，帮助患者全面了解自身健康状况。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 数字化目标 | 建立患者完整健康档案 | 健康档案覆盖率100% |
| 便捷目标 | 患者随时查看健康数据 | 查看响应时间≤2秒 |
| 健康目标 | 提供健康趋势分析 | 健康指标记录参与率≥30% |
| 智能目标 | AI健康建议辅助决策 | AI建议查看率≥50% |

### 1.3 功能范围

```

│   ├── 患者基本信息展示（姓名、性别、年龄、血型）

├── 既往病史管理
│   ├── 慢性病记录查看（诊断时间、治疗情况）
│   ├── 手术史查看（手术名称、手术时间）
```

## 4. 数据字段定义

### 4.1 健康指标记录表 (his_health_indicator)

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| record_id | BIGINT | 是 | 记录ID（主键） |
| patient_id | BIGINT | 是 | 患者ID |
| record_date | DATE | 是 | 记录日期 |
| record_time | TIME | 是 | 记录时间 |
| blood_pressure_systolic | DECIMAL(5,2) | 否 | 收缩压（mmHg） |
| blood_pressure_diastolic | DECIMAL(5,2) | 否 | 舒张压（mmHg） |
| heart_rate | INT | 否 | 心率（次/分） |
| blood_glucose | DECIMAL(4,1) | 否 | 血糖值（mmol/L） |
| glucose_measure_type | TINYINT | 否 | 血糖测量类型：1空腹/2餐后 |
| weight | DECIMAL(5,2) | 否 | 体重（kg） |
| height | DECIMAL(5,2) | 否 | 身高（cm） |
| bmi | DECIMAL(4,2) | 否 | BMI指数（自动计算） |
| temperature | DECIMAL(4,2) | 否 | 体温（℃） |
| health_score | INT | 否 | 健康评分（0-100） |
| abnormal_flag | TINYINT | 是 | 异常标识：0正常/1异常 |
| abnormal_items | VARCHAR(200) | 否 | 异常项列表 |
| record_source | TINYINT | 是 | 记录来源：1手动录入/2就诊同步 |
| create_time | DATETIME | 是 | 创建时间 |
| update_time | DATETIME | 否 | 更新时间 |

| 字段名 | 字段类型 | 必填 | 说明 |

| record_time | DATETIME | 是 | 记录时间 |
| record_source | TINYINT | 是 | 来源：1就诊录入/2手动录入 |
| status | TINYINT | 是 | 状态：1有效/2已确认无效 |
| create_time | DATETIME | 是 | 创建时间 |

### 4.3 既往病史表 (his_medical_history)

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|

| patient_id | BIGINT | 是 | 患者ID |

### 8.2 关键业务规则详解

#### BR-PT-012: 健康指标值范围校验

健康指标值必须在合理范围内，超出范围提示用户确认。

**触发时机**: 记录健康指标时

**校验逻辑**:
```
indicator_ranges = {
    'blood_pressure_systolic': [60, 250],    // 收缩压
    'blood_pressure_diastolic': [40, 150],   // 舒张压
    'heart_rate': [30, 200],                  // 心率
    'blood_glucose': [1.0, 35.0],            // 血糖
    'weight': [20, 300],                      // 体重
    'height': [50, 250]                       // 身高
}

range = indicator_ranges[input.indicator_type]
IF input.value < range[0] OR input.value > range[1] THEN
    RETURN warning("数值超出正常范围，请核实后确认")
END IF
```

#### BR-PT-029: 健康指标评分计算

根据患者健康指标计算健康评分，评分范围0-100分。

**触发时机**: 记录健康指标后

**计算逻辑**:
```
score = 100  // 基础分

// 血压评分
IF health_indicator.blood_pressure_systolic > 140 
   OR health_indicator.blood_pressure_diastolic > 90 THEN
    score = score - 10
END IF

// 血糖评分
IF health_indicator.blood_glucose > 7.0 THEN
    score = score - 10
END IF

// BMI评分
bmi = health_indicator.weight / (health_indicator.height / 100) ^ 2
IF bmi < 18.5 OR bmi > 24 THEN
    score = score - 5
END IF

// 心率评分
IF health_indicator.heart_rate < 60 OR health_indicator.heart_rate > 100 THEN
    score = score - 5
END IF

// 确保分数在0-100范围内
score = MAX(0, MIN(100, score))

RETURN score
```

#### BR-PT-030: 健康趋势分析计算

根据历史健康数据计算指标变化趋势，识别上升/下降/稳定趋势。

**触发时机**: 查看趋势图时

**计算逻辑**:
```
// 获取历史数据
history_data = SELECT result_value, record_date 
               FROM his_health_indicator 
               WHERE patient_id = :patient_id 
               AND indicator_type = :indicator_type
               ORDER BY record_date DESC
               LIMIT 10

IF COUNT(history_data) < 2 THEN
    RETURN trend = 'INSUFFICIENT_DATA'
END IF

// 计算变化率
changes = []
FOR i = 1 TO COUNT(history_data) - 1 DO
    change_rate = (history_data[i].value - history_data[i-1].value) 
                  / history_data[i-1].value * 100
    changes.append(change_rate)
END FOR

avg_change = AVG(changes)

// 判断趋势
IF avg_change > 5 THEN
    trend = 'RISING'
ELSE IF avg_change < -5 THEN
    trend = 'DECLINING'
ELSE
    trend = 'STABLE'
END IF

RETURN { trend, avg_change, data: history_data }
```

#### BR-PT-035: AI健康建议集成

集成AI服务接口，提供健康档案智能分析和建议。

**触发时机**: 请求AI健康建议时

**集成逻辑**:
```
// 检查健康数据是否充足
health_data = SELECT * FROM his_health_indicator 
              WHERE patient_id = :patient_id

IF COUNT(health_data) < 3 THEN
    RETURN error("健康数据不足，建议完善健康档案")
END IF

// 每日生成次数限制
today_count = SELECT count(*) 
              FROM his_ai_health_advice 
              WHERE patient_id = :patient_id 
              AND advice_date = CURRENT_DATE

IF today_count >= 1 THEN
    RETURN today_advice  // 显示今日已生成建议
END IF

TRY
    // 调用AI健康分析服务
    advice = CALL ai_service.health_analysis({
        'patient_id': patient.patient_id,
        'health_indicators': recent_indicators,
        'chronic_diseases': patient.chronic_diseases,
        'allergies': patient.allergies
    })
    
    // 记录AI建议
    INSERT INTO his_ai_health_advice (
        patient_id, advice_date, health_score, 
        risk_level, advice_content, chronic_advice
    ) VALUES (
        :patient_id, CURRENT_DATE, advice.health_score,
        advice.risk_level, advice.content, advice.chronic_advice
    )
    
    RETURN advice
CATCH exception
    LOG error("AI健康建议异常", exception)
    RETURN error("AI服务暂时不可用，请稍后再试")
END TRY
```
