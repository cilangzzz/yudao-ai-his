#### BR-PT-003: 验证码校验规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PT-003 |
| 规则名称 | 短信验证码校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 验证码(verify_code) |
| 规则描述 | 短信验证码5分钟有效，错误5次锁定手机号30分钟 |
| 触发时机 | 验证码验证时 |
| 校验逻辑 | ```
// 伪代码
verify_code = SELECT * FROM sms_verify_code 
              WHERE phone = :phone 
              ORDER BY create_time DESC 
              LIMIT 1

// 验证码不存在
IF verify_code IS NULL THEN
    RETURN error("请先获取验证码")
END IF

// 验证码过期（5分钟）
IF NOW() - verify_code.create_time > 300 THEN
    RETURN error("验证码已过期，请重新获取")
END IF

// 错误次数检查
error_count = SELECT count(*) FROM verify_code_log 
              WHERE phone = :phone 
              AND create_time > NOW() - INTERVAL 30 MINUTE
              AND result = 'FAIL'

IF error_count >= 5 THEN
    RETURN error("验证码错误次数过多，请30分钟后再试")
END IF

// 验证码匹配
IF input.code != verify_code.code THEN
    INSERT INTO verify_code_log (phone, result, create_time) 
    VALUES (:phone, 'FAIL', NOW())
    RETURN error("验证码错误，请重新输入")
END IF

// 验证成功
INSERT INTO verify_code_log (phone, result, create_time) 
VALUES (:phone, 'SUCCESS', NOW())

RETURN success()
``` |
| 失败处理 | 返回具体错误原因，更新错误计数 |
| 优先级 | P0 |

#### BR-PT-004: 身份证OCR校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PT-004 |
| 规则名称 | 身份证OCR识别校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 实名认证(realname_verify) |
| 规则描述 | 身份证照片必须清晰可识别，OCR识别失败需提供手动输入 |
| 触发时机 | 实名认证上传身份证时 |
| 校验逻辑 | ```
// 伪代码
TRY
    ocr_result = CALL id_card_ocr_api({
        'front_image': input.id_card_image_front,
        'back_image': input.id_card_image_back
    })
    
    IF ocr_result.status != 'SUCCESS' THEN
        RETURN error("身份证信息识别失败，请重新上传清晰照片", allow_manual_input=true)
    END IF
    
    // 校验身份证号格式
    IF NOT match_regex(ocr_result.id_card_no, '^[0-9]{17}[0-9X]$') THEN
        RETURN error("身份证号格式不正确", allow_manual_input=true)
    END IF
    
    // 校验有效期
    IF ocr_result.expiry_date < CURRENT_DATE THEN
        RETURN error("身份证已过期")
    END IF
    
    RETURN success(ocr_result)
CATCH exception
    LOG error("OCR识别异常", exception)
    RETURN error("识别服务异常，请稍后重试或手动输入", allow_manual_input=true)
END TRY
``` |
| 失败处理 | 返回识别失败原因，提供手动输入选项 |
| 优先级 | P0 |

#### BR-PT-007: 号源可用性校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PT-007 |
| 规则名称 | 号源可用性校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 预约挂号(appointment)、号源(schedule) |
| 规则描述 | 预约时必须校验号源是否可用，号源被锁定后其他患者不可预约 |
| 触发时机 | 创建预约时 |
| 校验逻辑 | ```
// 伪代码
schedule = SELECT * FROM op_schedule 
           WHERE schedule_id = :schedule_id 
           AND schedule_date = :appointment_date

IF schedule IS NULL THEN
    RETURN error("该医生当日无排班")
END IF

// 使用乐观锁检查剩余号源
IF schedule.remaining_count <= 0 THEN
    other_schedules = SELECT * FROM op_schedule 
                      WHERE dept_id = schedule.dept_id 
                      AND schedule_date = :appointment_date
                      AND remaining_count > 0
    RETURN error("该医生号源已满", other_schedules=other_schedules)
END IF

// 号源时间校验
IF schedule.schedule_date < CURRENT_DATE THEN
    RETURN error("不能选择过去的日期")
END IF

IF schedule.schedule_date > CURRENT_DATE + 7 THEN
    RETURN error("预约时间不能超过7天")
END IF

RETURN success()
``` |
| 失败处理 | 返回"号源已满"，显示同科室其他可选医生 |
| 优先级 | P0 |

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-PT-013 |
| 规则名称 | 预约状态流转规则 |
| 规则类型 | FLOW(流转规则) |
| 适用对象 | 预约记录(appointment) |
| 规则描述 | 预约状态流转：已预约→已签到/已取消/已过期；已签到后转为挂号状态 |
| 触发时机 | 预约操作时 |
| 校验逻辑 | ```
// 伪代码
// 状态定义：1-已预约, 2-已签到, 3-已取消, 4-已过期

valid_transitions = {
    1: [2, 3, 4],  // 已预约 -> 已签到/已取消/已过期
    2: [],         // 已签到 -> 转为挂号
    3: [],         // 已取消 -> 不可变更
    4: []          // 已过期 -> 不可变更
}

IF target_status NOT IN valid_transitions[current_status] THEN
    RETURN error("状态流转不合法")
END IF

// 签到后创建挂号记录
IF target_status = 2 THEN
    register = CREATE_register_from_appointment(appointment)
    register.register_status = 1  // 已挂号
END IF

RETURN success()
``` |
| 失败处理 | 返回"状态流转不合法"错误 |
| 优先级 | P0 |
