## 1. 功能概述

### 1.1 功能定位

在线缴费模块是患者服务子系统的核心功能之一，为患者提供便捷的线上缴费服务。系统支持待缴费查询、在线支付、缴费记录查询、费用明细查看、退费申请等功能，实现患者"指尖缴费"，减少现场排队等待，提升患者就医体验。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 便捷目标 | 减少患者现场排队缴费 | 线上缴费占比>=60% |
| 效率目标 | 提升医院收费效率 | 平均缴费时间<=30秒 |
| 体验目标 | 提升患者就医体验 | 患者满意度>=90分 |
| 透明目标 | 费用明细清晰透明 | 费用明细查看率>=80% |

## 5. 状态机设计

### 5.1 支付状态流转

```
支付状态定义:
0 - 待支付
1 - 已支付
2 - 已取消
3 - 已退款

状态流转图:
                    ┌─────────────┐
                    │  待支付(0)  │
                    └─────────────┘
                          │
            ┌─────────────┼─────────────┐
            │             │             │
            ▼             ▼             │
    ┌─────────────┐ ┌─────────────┐     │
    │  已支付(1)  │ │  已取消(2)  │     │
    └─────────────┘ └─────────────┘     │
            │                           │
            ▼                           │
    ┌─────────────┐                     │
    │  已退款(3)  │◄────────────────────┘
    └─────────────┘

状态流转规则:
- 待支付 -> 已支付: 支付成功
- 待支付 -> 已取消: 支付超时(10分钟)/用户取消
- 已支付 -> 已退款: 退费申请审批通过
```

### 5.2 退费状态流转

```
退费状态定义:
0 - 待审核
1 - 审核通过
2 - 已退款
3 - 审核拒绝

状态流转图:
                    ┌─────────────┐
                    │  待审核(0)  │
                    └─────────────┘
                          │
            ┌─────────────┼─────────────┐
            │             │             │
            ▼             ▼             │
    ┌─────────────┐ ┌─────────────┐     │
    │ 审核通过(1) │ │ 审核拒绝(3) │     │
    └─────────────┘ └─────────────┘     │
            │                           │
            ▼                           │
    ┌─────────────┐                     │
    │  已退款(2)  │◄────────────────────┘
    └─────────────┘    (重新申请退费)

状态流转规则:
- 待审核 -> 审核通过: 退费审批通过
- 待审核 -> 审核拒绝: 退费审批拒绝
- 审核通过 -> 已退款: 退费执行完成
- 审核拒绝 -> 待审核: 重新提交退费申请
```

| BR-PT-033 | 医保接口集成 | 集成规则 | 医保结算 |

### 6.2 关键业务规则详解

#### BR-PT-010: 在线支付金额校验

在线支付时必须校验支付金额，单笔在线支付金额上限为10000元，超过上限需到医院窗口缴费。

**触发时机**: 发起支付时

**校验逻辑**:
- 支付金额必须大于0
- 支付金额必须与费用明细合计一致
- 单笔支付金额不超过10000元

**失败处理**: 返回错误提示，引导到医院窗口

#### BR-PT-014: 支付状态流转规则

支付状态流转：待支付(0)->已支付(1)/已取消(2)；已支付(1)->已退款(3)

**触发时机**: 支付操作时

**状态流转约束**:
- 待支付状态可流转为已支付、已取消
- 已支付状态可流转为已退款
- 已取消、已退款状态不可再变更
- 支付成功后自动生成电子发票并发送通知

#### BR-PT-015: 退费审批流程

退费金额超过500元需审批，审批通过后方可执行退费。

**触发时机**: 申请退费时

**处理逻辑**:
- 退费金额<=500元：直接执行退费
- 退费金额>500元：创建审批单，进入审批流程
- 审批通过后：原路退回（医保撤销+现金退回）
