## 1. 功能概述

### 1.1 功能定位

患者门户是患者服务子系统的入口模块，面向患者用户提供注册登录、实名认证、信息管理、家庭成员管理、就诊记录查询、就诊卡管理、消息通知中心等核心功能。系统支持微信小程序、H5网页等多种接入渠道，实现患者"指尖上的医院"体验，是患者使用预约挂号、报告查询、在线缴费等功能的前置条件。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 便捷目标 | 提供一站式患者服务入口 | 患者注册转化率>=90% |
| 效率目标 | 缩短患者信息录入时间 | 身份证OCR识别率>=95% |
| 安全目标 | 确保患者身份真实性 | 实名认证通过率>=85% |
| 体验目标 | 提升患者就医体验 | 患者满意度>=90分 |

## 3. 接口设计

### 3.1 患者登录接口

**接口路径**: POST /api/patient/login

**请求体**:
```json
{
  "loginType": "wechat",
  "wxCode": "xxx",
  "phone": "13800000000",
  "verifyCode": "123456"
}
```

**响应格式**:
```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "userId": 1001,
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "tokenExpire": "2026-06-20 10:00:00",
    "isNewUser": false,
    "realNameVerified": true,
    "phoneVerified": true,
    "userInfo": {
      "name": "张三",
      "phone": "138****0000",
      "avatar": "https://xxx.com/avatar.jpg"
    }
  }
}
```

### 3.2 实名认证接口

**接口路径**: POST /api/patient/realname/verify

**请求体**:
```json
{
  "userId": 1001,
  "realName": "张三",
  "idCardNo": "123456789012345678",
  "idCardImageFront": "base64...",
  "idCardImageBack": "base64...",
  "faceImage": "base64..."
}
```

**响应格式**:
```json
{
  "code": 200,
  "msg": "实名认证成功",
  "data": {
    "verified": true,
    "empiId": "P202606190001",
    "realName": "张三",
    "gender": 1,
    "birthday": "1990-01-15"
  }
}
```

## 5. 状态机设计

### 5.1 实名认证状态机

```
状态流转图:

    ┌──────────┐
    │ 未认证(0) │
    └────┬─────┘
         │ 提交认证
         ▼
    ┌──────────┐
    │ 认证中(1) │
    └────┬─────┘
         │
    ┌────┴────────────────┐
    │                     │
    ▼                     ▼
┌──────────┐        ┌──────────┐
│ 已认证(2) │        │认证失败(3)│
└──────────┘        └────┬─────┘
   (终态)                 │
                          │ 重新认证
                          ▼
                    ┌──────────┐
                    │ 认证中(1) │
                    └──────────┘

状态说明:
- 0 未认证: 新用户初始状态
- 1 认证中: 已提交认证信息，等待验证
- 2 已认证: 认证通过，可使用全部功能
- 3 认证失败: 认证未通过，可重新提交

状态转换规则:
- 0 → 1: 用户提交认证信息
- 1 → 2: 认证通过
- 1 → 3: 认证失败
- 3 → 1: 重新提交认证
- 2 → 终态: 不可变更
```

### 5.2 就诊卡绑定状态机

```
状态流转图:

    ┌──────────┐
    │   有效   │
    │ (ACTIVE) │
    └────┬─────┘
         │ 解绑操作
         ▼
    ┌───────────┐
    │   已解绑   │
    │ (INACTIVE)│
    └───────────┘
        (终态)

状态说明:
- ACTIVE 有效: 正常绑定状态
- INACTIVE 已解绑: 已解除绑定，不可恢复

状态转换规则:
- ACTIVE → INACTIVE: 用户主动解绑
- INACTIVE → 终态: 不可恢复，需重新绑定
```

### 5.3 用户账号状态机

```
状态流转图:

    ┌──────────┐
    │ 正常(1)  │◄────────────┐
    └────┬─────┘             │
         │ 违规操作           │ 申诉通过
         ▼                   │
    ┌──────────┐        ┌────┴────┐
    │ 冻结(2)  │────────→│ 正常(1) │
    └────┬─────┘ 解冻    └─────────┘
         │
         │ 注销申请
         ▼
    ┌──────────┐
    │ 注销(3)  │
    └──────────┘
       (终态)

状态说明:
- 1 正常: 账号可正常使用
- 2 冻结: 账号被限制使用，需申诉
- 3 注销: 账号已注销，不可恢复

状态转换规则:
- 1 → 2: 违规操作，系统冻结
- 2 → 1: 申诉通过，解除冻结
- 1 → 3: 用户申请注销
- 2 → 3: 严重违规，强制注销
- 3 → 终态: 不可恢复
```

### 6.2 关键业务规则详解

#### BR-PT-001: 患者实名认证校验

患者必须完成实名认证才能使用预约挂号、在线缴费等核心功能。

**触发时机**: 预约挂号、在线缴费等操作时

**校验逻辑**:
```
IF patient_user.real_name_verified != 2 THEN
    RETURN error("请先完成实名认证", redirect_to="/realname/verify")
END IF
RETURN success()
```

#### BR-PT-003: 短信验证码校验

短信验证码5分钟有效，错误5次锁定手机号30分钟。

**触发时机**: 验证码验证时

**校验逻辑**:
```
// 验证码过期检查
IF NOW() - verify_code.create_time > 300 SECONDS THEN
    RETURN error("验证码已过期，请重新获取")
END IF

// 错误次数检查
error_count = SELECT count(*) FROM verify_code_log 
              WHERE phone = :phone 
              AND create_time > NOW() - 30 MINUTES
              AND result = 'FAIL'

IF error_count >= 5 THEN
    RETURN error("验证码错误次数过多，请30分钟后再试")
END IF

// 验证码匹配
IF input.code != verify_code.code THEN
    INSERT INTO verify_code_log (phone, result) VALUES (:phone, 'FAIL')
    RETURN error("验证码错误，请重新输入")
END IF
```

#### BR-PT-022: 就诊卡绑定数量限制

每位用户最多绑定5张就诊卡。

**触发时机**: 绑定就诊卡时

**校验逻辑**:
```
bind_count = SELECT count(*) 
             FROM patient_bindcard 
             WHERE patient_user_id = :current_user_id 
             AND status = 'ACTIVE'

IF bind_count >= 5 THEN
    RETURN error("每位用户最多绑定5张就诊卡")
END IF
RETURN success()
```

#### BR-PT-023: 家庭成员数量限制

每位用户最多添加10位家庭成员。

**触发时机**: 添加家庭成员时

**校验逻辑**:
```
family_count = SELECT count(*) 
               FROM family_member 
               WHERE user_id = :current_user_id 
               AND status = 'ACTIVE'

IF family_count >= 10 THEN
    RETURN error("最多可添加10位家庭成员")
END IF
RETURN success()
```
