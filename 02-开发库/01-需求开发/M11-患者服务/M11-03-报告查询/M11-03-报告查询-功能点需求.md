## 1. 功能概述

### 1.1 功能定位

报告查询是患者服务子系统的重要功能模块，面向患者用户提供检验报告、影像报告的在线查询服务。系统支持报告详情查看、报告推送、报告解读、报告分享等功能，实现患者"指尖上的报告查看"体验，减少患者往返医院取报告的次数。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 便捷目标 | 减少患者往返医院取报告 | 线上报告查询占比≥80% |
| 效率目标 | 提升报告获取效率 | 报告发布后30分钟内可查询 |
| 体验目标 | 提升患者就医体验 | 患者满意度≥90分 |
| 数字化目标 | 建立患者报告档案 | 报告数字化存储率100% |

### 1.3 功能范围

```

│   ├── DICOM影像调阅（影像缩略图、大图查看）
│   ├── 影像操作（缩放、旋转、窗宽窗位）
│   └── 报告打印预约
├── 报告推送通知
│   ├── 报告发布自动推送（微信消息通知）
│   ├── 推送内容配置（报告类型、关键信息）
│   └── 推送记录查询
├── 报告解读
│   ├── AI辅助解读（检验报告解读）
│   ├── 异常指标分析（可能原因、健康建议）
│   └── 解读历史记录
├── 报告分享
│   ├── 分享给家属（生成分享链接）
│   ├── 分享给医生（医生专用链接）
```

### 1.4 用户角色

| 角色 | 主要职责 | 使用功能 |
|------|----------|----------|
| 患者 | 查看自己的检验/影像报告 | 报告查询、报告解读、报告分享、打印预约 |
| 家属代理 | 代老人/儿童查看报告 | 代管成员报告查询、报告分享 |
| 医生（被分享） | 查看患者分享的报告 | 通过分享链接查看完整报告 |
| 医院管理员 | 管理报告推送配置 | 推送配置、通知模板管理 |

### 2.3 字段定义

#### 2.3.1 检验报告表 (his_lab_report)

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| report_id | BIGINT | 是 | 报告ID（主键） |
| report_no | VARCHAR(30) | 是 | 报告编号，格式：LJ+日期+序号 |
| patient_id | BIGINT | 是 | 患者ID |
| patient_name | VARCHAR(50) | 是 | 患者姓名 |
| request_id | BIGINT | 是 | 检验申请ID |
| report_type | VARCHAR(50) | 是 | 报告类型（血常规、肝功能等） |
| report_date | DATE | 是 | 检验日期 |
| report_time | DATETIME | 是 | 检验时间 |
| dept_id | BIGINT | 是 | 检验科室ID |
| dept_name | VARCHAR(100) | 是 | 科室名称 |
| technician_id | BIGINT | 否 | 检验技师ID |
| technician_name | VARCHAR(50) | 否 | 检验技师姓名 |
| reviewer_id | BIGINT | 是 | 审核医师ID |
| reviewer_name | VARCHAR(50) | 是 | 审核医师姓名 |
| report_status | TINYINT | 是 | 状态：0检验中/1待审核/2已发布 |

#### BR-PT-011: 报告访问权限校验

患者只能查看自己的报告，家庭成员可查看被授权的报告，分享链接访问需验证链接有效性。

**触发时机**: 查询报告时

**校验逻辑**:
```
// 伪代码
// 本人报告
IF report.patient_id = current_patient_id THEN
    RETURN success()
END IF

// 家庭成员授权
family_member = SELECT * FROM family_member 
                WHERE user_id = :current_user_id 
                AND patient_id = :report_patient_id
                AND status = 'ACTIVE'

IF family_member IS NULL THEN
    RETURN error("无权查看该报告")
END IF

// 分享链接访问
share_link = SELECT * FROM report_share_link 
             WHERE share_code = :share_code
             AND expire_time > NOW()
             AND status = 'ACTIVE'

IF share_link IS NULL THEN
    RETURN error("分享链接已过期或不存在")
END IF

RETURN success()
```

#### BR-PT-019: 报告分享有效期管理

报告分享链接有效期为7天（家属）或30天（医生），过期后自动失效。

**触发时机**: 创建分享链接、访问分享链接时

**校验逻辑**:
```
// 伪代码
// 创建分享链接
IF share_target = 'FAMILY' THEN
    expire_days = 7
ELSE IF share_target = 'DOCTOR' THEN
    expire_days = 30
END IF

share_link = CREATE_share_link({
    'report_id': report_id,
    'share_code': GENERATE_share_code(),
    'share_target': share_target,
    'expire_time': NOW() + INTERVAL expire_days DAY,
    'status': 'ACTIVE'
})

// 访问分享链接时校验
share_link = SELECT * FROM report_share_link WHERE share_code = :share_code

IF share_link.expire_time < NOW() THEN
    share_link.status = 'EXPIRED'
    RETURN error("分享链接已过期")
END IF

RETURN success(share_link)
```

#### BR-PT-025: 报告分享次数限制

每位患者每日最多分享报告5次。

**触发时机**: 创建分享链接时

**校验逻辑**:
```
// 伪代码
today_share_count = SELECT count(*) 
                    FROM report_share_link 
                    WHERE patient_id = :patient_id 
                    AND DATE(create_time) = CURRENT_DATE

IF today_share_count >= 5 THEN
    RETURN error("每日最多分享5次")
END IF

RETURN success()
```

#### BR-PT-030: 检验结果趋势分析计算

根据历史检验数据计算指标变化趋势，识别上升/下降/稳定趋势。

**触发时机**: 查看趋势图时

**校验逻辑**:
```
// 伪代码
// 获取历史数据
history_data = SELECT result_value, report_date 
               FROM lab_result 
               WHERE patient_id = :patient_id 
               AND item_code = :item_code
               ORDER BY report_date DESC
               LIMIT 10

IF COUNT(history_data) < 2 THEN
    RETURN trend = 'INSUFFICIENT_DATA'
END IF

// 计算变化率
changes = []
FOR i = 1 TO COUNT(history_data) - 1 DO
    change_rate = (history_data[i].result_value - history_data[i-1].result_value) 
                  / history_data[i-1].result_value * 100
    changes.append(change_rate)
END FOR

avg_change = AVG(changes)

// 判断趋势
IF avg_change > 5 THEN
    trend = 'RISING'
ELSE IF avg_change < -5 THEN
    trend = 'DECLINING'
ELSE
    trend = 'STABLE'
END IF

RETURN {
    'trend': trend,
    'avg_change': avg_change,
    'data': history_data
}
```

#### BR-PT-035: AI报告解读集成规则

集成AI服务接口，提供检验报告智能解读。

**触发时机**: 请求AI解读时

**校验逻辑**:
```
// 伪代码
// 检查健康数据是否充足
health_data = SELECT * FROM health_record 
              WHERE patient_id = :patient_id

IF COUNT(health_data) < 3 THEN
    RETURN error("健康数据不足，建议完善健康档案")
END IF

// 每日解读次数限制
today_count = SELECT count(*) 
              FROM ai_interpretation_log 
              WHERE patient_id = :patient_id 
              AND DATE(create_time) = CURRENT_DATE

IF today_count >= 3 THEN
    RETURN error("今日解读次数已达上限")
END IF

TRY
    // 调用AI解读服务
    interpretation = CALL ai_service.interpret_lab_report({
        'patient_id': patient.patient_id,
        'report_id': report.report_id,
        'items': report.items,
        'patient_info': {
            'age': patient.age,
            'gender': patient.gender,
            'chronic_diseases': patient.chronic_diseases
        }
    })
    
    // 记录解读日志
    INSERT INTO ai_interpretation_log (patient_id, report_id, create_time)
    VALUES (:patient_id, :report_id, NOW())
    
    RETURN interpretation
CATCH exception
    LOG error("AI解读异常", exception)
    RETURN error("AI服务暂时不可用，请稍后再试")
END TRY
```
