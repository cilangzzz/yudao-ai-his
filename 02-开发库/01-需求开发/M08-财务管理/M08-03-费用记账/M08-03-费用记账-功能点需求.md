## 1. 功能概述

### 1.1 功能定位

费用记账是财务管理子系统的核心模块之一，负责门诊和住院费用的自动记账、费用明细管理、费用调整和退费记账处理。系统实现费用产生时自动记账，支持费用明细的多维度查询，提供费用调整（退费、补收）功能，为后续收费结算和财务报表提供准确的数据基础。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 实现费用自动记账和实时汇总 | 费用记账响应时间≤2秒 |
| 准确目标 | 保证费用计算准确无误 | 费用计算准确率100% |
| 完整目标 | 确保费用记录完整可追溯 | 费用记录完整率100% |
| 安全目标 | 费用调整需审批可追溯 | 财务操作审计覆盖率100% |

### 1.3 功能范围

```
M08-03 费用记账
├── 门诊费用记账
│   ├── 挂号费用自动记账（挂号费、诊查费）
│   ├── 处方费用自动记账（药品费）
│   ├── 检查检验费用自动记账（检查费、检验费）
│   └── 费用明细记录
├── 住院费用记账
│   ├── 床位费每日自动记账
│   ├── 护理费每日自动记账
│   ├── 医嘱执行费用记账（药品费、治疗费）
│   ├── 手术费用记账（手术费、麻醉费、材料费）
│   └── 出院前费用核对
├── 费用明细查询
│   ├── 按患者查询
│   ├── 按科室查询
│   ├── 按时间范围查询
│   ├── 按费用状态查询
│   └── 费用汇总计算
├── 费用调整
│   ├── 退费调整（已收费项目退费）
│   ├── 补收调整（漏收项目补收）
│   ├── 金额调整（费用金额修正）
│   └── 调整历史查询
└── 退费记账处理
    ├── 退费记录创建
    ├── 医保撤销处理
    └── 退费金额计算
```

## 7. 状态机设计

```
                    ┌─────────────┐
                    │   已记账    │ (1)
                    │   状态1    │
                    └──────┬──────┘
                           │
                    收费结算│
                           ↓
                    ┌─────────────┐
                    │   已收费    │ (2)
                    │   状态2    │
                    └──────┬──────┘
                           │
                    退费处理│
                           ↓
                    ┌─────────────┐
                    │   已退费    │ (3)
                    │   状态3    │
                    └─────────────┘

状态流转规则:
- 已记账(1) → 已收费(2)：收费结算成功
- 已收费(2) → 已退费(3)：退费处理成功
- 已退费(3) → 不可变更：终态
```

### 7.2 费用调整状态流转

```
    ┌─────────────┐
    │   待处理    │ (1)
    │   状态1    │
    └──────┬──────┘
           │
    执行调整│
           ↓
    ┌─────────────┐
    │   已完成    │ (2)
    │   状态2    │
    └─────────────┘
           ↑
           │
    取消调整│
           │
    ┌─────────────┐
    │   已取消    │ (3)
    │   状态3    │
    └─────────────┘

状态流转规则:
- 待处理(1) → 已完成(2)：调整执行成功
- 待处理(1) → 已取消(3)：取消调整
- 已完成(2) → 不可变更：终态
- 已取消(3) → 不可变更：终态
```

## 8. 业务规则

### 8.1 相关业务规则引用

| 规则编号 | 规则名称 | 规则类型 | 适用场景 |
|----------|----------|----------|----------|
| BR-FIN-006 | 费用金额非负校验 | VAL (校验规则) | 费用记账时 |
| BR-FIN-007 | 费用调整金额上限校验 | VAL (校验规则) | 费用调整时 |
| BR-FIN-008 | 已退费项目重复退费校验 | VAL (校验规则) | 费用退费时 |
| BR-FIN-016 | 费用明细状态流转规则 | FLOW (流转规则) | 费用状态变更时 |
| BR-FIN-020 | 费用调整审批流程 | FLOW (流转规则) | 费用调整时 |
| BR-FIN-024 | 费用调整权限控制 | PERM (权限规则) | 费用调整操作时 |
| BR-FIN-027 | 医保报销金额计算 | CALC (计算规则) | 医保患者费用记账时 |
| BR-FIN-029 | 费用汇总计算 | CALC (计算规则) | 费用明细查询时 |
| BR-FIN-032 | 退费金额计算 | CALC (计算规则) | 退费处理时 |

### 8.2 关键业务规则详解

#### BR-FIN-006: 费用金额非负校验

费用金额必须为非负数，不允许零或负数费用。

**触发时机**: 费用记账时、费用调整时

**校验逻辑**:
```
IF amount IS NULL THEN
    RETURN error("费用金额不能为空")
END IF

IF amount < 0 THEN
    RETURN error("费用金额不能为负数")
END IF

IF amount = 0 THEN
    RETURN warning("费用金额为零，请确认是否正确")
END IF
```

#### BR-FIN-008: 已退费项目重复退费校验

已退费的费用项目不允许再次退费。

**触发时机**: 费用退费调整时

**校验逻辑**:
```
fee_detail = SELECT * FROM his_charge_detail WHERE detail_id = :detail_id

IF fee_detail.charge_status = 3 THEN  // 已退费
    RETURN error("该项目已退费，不可重复退费")
END IF
```

#### BR-FIN-016: 费用明细状态流转规则

费用明细状态流转：已记账(1)→已收费(2)→已退费(3)；已收费可退费；已退费不可再操作。

**触发时机**: 费用明细状态变更时

**校验逻辑**:
```
// 状态定义：1已记账, 2已收费, 3已退费
valid_transitions = {
    1: [2],        // 已记账 -> 已收费
    2: [3],        // 已收费 -> 已退费
    3: []          // 已退费 -> 不可变更
}

IF target_status NOT IN valid_transitions[current_status] THEN
    RETURN error("费用明细状态流转不合法")
END IF
```

#### BR-FIN-020: 费用调整审批流程

费用调整金额超过500元需审批，审批通过后方可执行调整。

**触发时机**: 费用调整提交时

**校验逻辑**:
```
IF adjustment_amount > 500 THEN
    IF NOT has_approval('FEE_ADJUSTMENT', :adjustment_id) THEN
        RETURN require_approval("费用调整金额超过500元，需要审批", {
            'approval_type': 'FEE_ADJUSTMENT',
            'amount': adjustment_amount,
            'reason': adjustment_reason
        })
    END IF
END IF
```

#### BR-FIN-029: 费用汇总计算

费用汇总计算：总金额 = Σ各费用项目金额，自动按费用类型汇总。

**触发时机**: 费用明细查询时、结算时

**计算逻辑**:
```
total_amount = 0
type_summary = {}

FOR each detail IN details DO
    total_amount = total_amount + detail.amount
    
    IF type_summary[detail.item_type] EXISTS THEN
        type_summary[detail.item_type] = type_summary[detail.item_type] + detail.amount
    ELSE
        type_summary[detail.item_type] = detail.amount
    END IF
END FOR

RETURN {
    'total_amount': total_amount,
    'type_summary': type_summary
}
```

#### BR-FIN-032: 退费金额计算

退费金额需分别计算医保退回和现金退回，按原支付渠道退款。

**触发时机**: 退费处理时

**计算逻辑**:
```
insurance_refund = 0
cash_refund = 0

FOR each item IN refund.items DO
    detail = SELECT * FROM his_charge_detail WHERE detail_id = item.detail_id
    
    IF detail.insurance_amount > 0 THEN
        // 该项目有医保支付，需医保撤销
        insurance_refund = insurance_refund + detail.insurance_amount
    END IF
    
    IF detail.personal_amount > 0 THEN
        // 该项目有个人支付，需原路退回
        cash_refund = cash_refund + detail.personal_amount
    END IF
END FOR

RETURN {
    'insurance_refund': insurance_refund,
    'cash_refund': cash_refund,
    'total_refund': insurance_refund + cash_refund
}
```
