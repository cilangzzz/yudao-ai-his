## 1. 功能概述

### 1.1 功能定位

医保结算是财务管理子系统的核心模块之一，负责与医保平台的实时对接，实现医保身份验证、医保费用结算、医保对账、医保撤销处理等功能。系统支持城镇职工医保、城乡居民医保等多种医保类型，实现医保目录对照和费用自动拆分，确保医保结算的准确性和合规性。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 实现医保实时结算，缩短结算等待时间 | 医保结算响应时间<=3秒 |
| 合规目标 | 符合医保结算规范，确保结算成功率 | 医保实时结算成功率>=98% |
| 准确目标 | 保证费用拆分和报销计算准确 | 费用计算准确率100% |
| 安全目标 | 保证医保数据安全传输 | 医保接口数据加密传输率100% |

## 3. 用户故事

### US-M08-02-001 医保身份验证

**故事卡片**
```
作为 收费员
我想要 验证患者的医保身份和资格
以便于 确认患者是否可以享受医保报销
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 医保身份验证

  Scenario: 医保身份验证成功
    Given 患者医保类型为"城镇职工医保"
    And 患者医保卡状态正常
    When 收费员读取医保卡信息
    Then 系统调用医保接口验证身份
    And 显示患者医保信息（医保类型、累计金额、限额）
    And 显示医保状态"有效"

  Scenario: 医保卡读取失败
    Given 收费员尝试医保结算
    When 医保卡读卡失败
    Then 系统提示"医保卡读取失败，请重试"
    And 提供"手动输入医保卡号"选项

  Scenario: 医保卡已停保
    Given 收费员读取医保卡
    When 医保接口返回"已停保"
    Then 系统提示"医保卡状态异常：已停保"
    And 显示停保原因
    And 提供"转为自费结算"选项

  Scenario: 年度额度查询
    Given 患者医保身份验证成功
    When 系统显示医保信息
    Then 显示本年度累计金额
    And 显示年度限额
    And 显示剩余可用额度

  Scenario Exception: 医保接口异常
    Given 收费员正在验证医保身份
    When 医保接口调用超时或异常
    Then 系统提示"医保系统异常，请稍后重试或选择自费结算"
    And 系统记录异常日志
    And 提供"重试"和"自费结算"选项

  Scenario Boundary: 医保卡号格式校验
    Given 收费员手动输入医保卡号
    When 输入的医保卡号格式不正确
    Then 系统提示"医保卡号格式不正确"
    And 提供正确格式提示
```

### US-M08-02-002 医保费用结算

**故事卡片**
```
作为 收费员
我想要 为医保患者办理医保费用结算
以便于 患者享受医保报销政策
```

**优先级**: P0（MVP必需）

**故事点**: 13

**验收标准 (Gherkin)**
```gherkin
Feature: 医保费用结算

  Scenario: 医保预结算成功
    Given 患者医保身份验证成功
    And 患者有待缴费项目
    When 收费员点击"医保结算"
    Then 系统调用医保预结算接口
    And 显示费用明细拆分（甲类/乙类/丙类）
    And 显示医保报销金额
    And 显示个人自付金额
    And 显示起付线、共付段、自费段明细

  Scenario: 医保正式结算
    Given 医保预结算成功
    And 患者确认结算信息
    When 收费员点击"确认结算"
    Then 系统调用医保正式结算接口
    And 生成医保结算记录
    And 获取医保流水号
    And 医保报销金额入账
    And 个人自付部分进行收费

  Scenario: 医保目录对照自动拆分
    Given 患者费用包含多个项目
    And 部分项目已建立医保对照
    When 系统进行医保预结算
```

### 5.2 关键业务规则详解

#### BR-FIN-015: 医保结算状态流转规则

医保结算状态流转：待结算(1) -> 已结算(2) -> 已撤销(3)；已结算可撤销；已撤销不可再操作。

**状态流转约束**:
- 待结算状态可流转为已结算
- 已结算状态可流转为已撤销
- 已撤销状态不可再变更
- 撤销后需更新费用明细

#### BR-FIN-027: 医保报销金额计算

医保报销金额根据医保类别（甲/乙/丙类）和报销比例自动计算。

**计算逻辑**:
```
insurance_amount = 0
personal_amount = 0

FOR each item IN settlement.items DO
    IF mapping.insurance_type = 1 THEN  // 甲类
        insurance_amount = insurance_amount + item.amount
    ELSE IF mapping.insurance_type = 2 THEN  // 乙类
        covered = item.amount * mapping.coverage_ratio / 100
        insurance_amount = insurance_amount + covered
        personal_amount = personal_amount + (item.amount - covered)
    ELSE  // 丙类、自费
        personal_amount = personal_amount + item.amount
    END IF
END FOR
```

#### BR-FIN-028: 医保起付线计算

医保结算需扣除起付线，累计年度达到起付线后可享受报销。

**计算逻辑**:
```
// 获取起付线标准
deductible_standard = SELECT deductible FROM dict_insurance_policy 
                      WHERE insurance_type = :insurance_type

// 获取年度累计
year_accumulated = SELECT SUM(amount) FROM his_charge_detail 
                   WHERE patient_id = :patient_id
                   AND year = :current_year
                   AND insurance_type IN (1, 2)

// 计算是否达到起付线
IF year_accumulated >= deductible_standard THEN
    // 已达到起付线，本次可报销
    deductible = 0
ELSE
    // 未达到起付线
    IF year_accumulated + insurance_amount >= deductible_standard THEN
        // 本次达到起付线，超过部分可报销
        deductible = deductible_standard - year_accumulated
        insurance_amount = insurance_amount - deductible
    ELSE
        // 本次仍未达到起付线
        deductible = insurance_amount
        insurance_amount = 0
    END IF
END IF
```

#### BR-FIN-042: 医保接口超时重试规则

医保接口调用超时时自动重试3次，重试间隔递增（1秒、2秒、3秒），重试失败后转为人工处理。

**重试逻辑**:
```
retry_count = 0
max_retry = 3
retry_interval = [1, 2, 3]  // 秒

WHILE retry_count < max_retry DO
    TRY
        result = CALL medical_insurance_api.operation(params)
        IF result.status = 'SUCCESS' THEN
            RETURN success(result)
        END IF
    CATCH timeout_exception
        retry_count = retry_count + 1
        IF retry_count < max_retry THEN
            SLEEP(retry_interval[retry_count - 1])
            CONTINUE
        ELSE
            RETURN error("医保系统响应超时，请稍后重试或人工处理")
        END IF
    END TRY
END WHILE
```

## 7. 交互流程

### 7.1 医保结算完整流程

```
患者缴费
    |
    v
判断是否医保患者 --> 否 --> 全额自费结算
    |
    v 是
医保身份验证（调用医保接口）
    |
    +-> 验证失败 --> 提示失败原因 --> 询问是否自费
    |
    v 验证成功
医保预结算（调用医保接口）
    |
    v
医保目录对照
    |
    +-- 甲类项目 --> 全额纳入报销
    +-- 乙类项目 --> 按比例纳入报销（自付比例）
    +-- 丙类项目 --> 全额自费
    |
    v
计算报销金额（BR-FIN-027）
    |
    v
计算起付线（BR-FIN-028）
    |
    v
计算个人自付（自付段+自费段）
    |
    v
显示结算明细，等待确认
    |
    v
医保正式结算（调用医保接口）
    |
    +-> 结算失败 --> 提示错误 --> 询问是否转自费
    |
    v 结算成功
生成医保结算记录
    |
    v
获取医保流水号
    |
    v
医保报销金额入账
    |
    v
个人自付部分收费
    |
    v
结算完成
```

### 7.2 医保撤销流程

```
医保退费申请
    |
    v
查询医保结算记录
    |
    +-> 状态非"已结算" --> 提示不可撤销
    |
    v 状态为"已结算"
填写撤销原因
    |
    v
调用医保撤销接口
    |
    +-> 撤销失败 --> 提示失败，转人工处理
    |
    v 撤销成功
更新结算状态为"已撤销"
    |
    v
更新费用明细（医保金额归零，转为自费）
    |
    v
继续退费流程
```

### 7.3 医保对账流程

```
医保专员登录系统
    |
    v
选择对账日期
    |
    v
查询本地医保结算汇总
    |
    +-- 统计结算笔数、金额
    +-- 统计撤销笔数、金额
    |
    v
点击"与医保平台对账"
    |

调用医保对账接口
    |

    |

获取医保平台数据
    |
    v
对比本地数据与平台数据
    |
    +-- 一致 --> 显示"对账一致"
    |
    +-- 差异 --> 显示差异明细
    |              +-- 提供差异处理入口
    |
    v
生成对账记录
    |
    v
导出对账报表（可选）
```

## 8. 状态机设计

### 8.1 医保结算状态机

```
                    +-------------+
                    |   待结算    |
                    |   (1)       |
                    +------+------+
                           | 正式结算成功
                           v
                    +-------------+
           撤销失败 |   已结算    | 撤销成功
          +--------|   (2)       |--------+
          |        +------+------+
          |               |
          |               | 撤销操作
          |               v
          |        +-------------+
          +------->|   已撤销    |
                   |   (3)       |
                   +-------------+
                         |
                         | 终态，不可变更
                         v
                   +-------------+
                   |  结束       |
                   +-------------+

状态说明：
- 1 待结算：医保预结算完成，等待正式结算
- 2 已结算：医保正式结算完成
- 3 已撤销：医保结算已撤销

状态流转规则：
- 待结算(1) -> 已结算(2)：正式结算成功
- 已结算(2) -> 已撤销(3)：撤销操作成功
- 已撤销(3)：终态，不可再变更
```

### 8.2 医保身份验证状态机

```
         +-------------+
         |   未验证    |
         +------+------+
                | 发起验证
                v
         +-------------+
         |   验证中    |
         +------+------+
                |
       +--------+--------+
       |        |        |
       v        v        v
+----------+ +----------+ +----------+
|  成功    | |  失败    | |  异常    |
|  SUCCESS | |  FAIL    | |  ERROR   |
+----------+ +----------+ +----------+
     |            |            |
     v            v            v
 可进行结算    提示原因     提供重试选项
```
