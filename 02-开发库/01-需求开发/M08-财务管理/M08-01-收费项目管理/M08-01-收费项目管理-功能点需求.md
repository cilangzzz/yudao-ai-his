## 1. 功能概述

### 1.1 功能定位

收费项目管理是财务管理子系统的基础模块，负责全院收费项目的统一管理，包括收费项目维护、价格管理、医保目录对照等功能。收费项目涵盖诊疗类、药品类、检查类、服务类、材料类等多种类型，是门诊收费、住院结算、医保结算的费用数据来源，为全院费用管理提供基础支撑。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 实现收费项目快速查询和维护 | 项目查询响应≤1秒 |
| 准确目标 | 保证价格计算准确无误 | 费用计算准确率100% |
| 合规目标 | 符合医保目录对照规范 | 医保对照覆盖率≥95% |
| 安全目标 | 价格调整需审批流程 | 价格调整审批率100% |

### 1.3 功能范围

```
M08-01 收费项目管理
```

## 3. 用户故事

### US-M08-01-001 收费项目维护

**故事卡片**
```
作为 财务管理员
我想要 维护医院收费项目信息
以便于 统一管理全院收费标准并确保计价准确
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 收费项目维护

  Scenario: 新增收费项目
    Given 财务管理员已登录系统
    And 具有收费项目管理权限
    When 管理员点击"新增收费项目"
    And 填写项目编码"ZX003"
    And 填写项目名称"特需诊查费"
    And 选择项目类型"诊疗类"
    And 填写单价"100.00"
    And 选择医保类别"自费"
    And 点击"保存"
    Then 系统创建收费项目记录
    And 项目状态为"启用"
    And 系统记录创建人和创建时间

  Scenario: 修改收费项目
    Given 收费项目"ZX001"已存在
    When 管理员修改项目名称为"门诊诊查费(普通)"
    And 修改项目说明为"普通门诊诊查服务"
    And 点击"保存"
    Then 系统更新项目信息
    And 系统记录修改人和修改时间

  Scenario: 停用收费项目
    Given 收费项目"ZX001"状态为"启用"
    When 管理员点击"停用"
    And 确认停用操作
    Then 项目状态变更为"停用"
    And 项目在收费时不再显示

  Scenario Boundary: 项目编码唯一性校验
    Given 收费项目编码"ZX001"已存在
    When 管理员新增项目编码为"ZX001"
    Then 系统提示"项目编码已存在"
    And 新增操作被阻止

  Scenario Boundary: 项目编码格式校验
    Given 管理员正在新增收费项目
    When 管理员输入项目编码包含特殊字符（如中文、空格）
    Then 系统提示"项目编码只能包含字母、数字和下划线"
```

### 5.2 关键业务规则详解

#### BR-FIN-001: 收费项目编码唯一性校验

收费项目编码必须全局唯一，不允许重复创建相同编码的项目。

**触发时机**: 创建收费项目时

**校验逻辑**:
```
existing_item = SELECT * FROM his_charge_item_def 
                WHERE item_code = :item_code
                AND deleted = 0

IF existing_item EXISTS THEN
    RETURN error("项目编码已存在: " + existing_item.item_name)
END IF

RETURN success()
```

**失败处理**: 返回错误"项目编码已存在"，显示已存在项目名称，阻止创建

---

#### BR-FIN-002: 收费项目价格格式校验

收费项目价格必须为正数，精度不超过2位小数，不允许零或负数价格。

**触发时机**: 创建/修改收费项目时、价格调整时

**校验逻辑**:
```
IF price IS NULL THEN
    RETURN error("价格不能为空")
END IF

IF price <= 0 THEN
    RETURN error("价格必须为正数")
END IF

// 精度校验
IF price * 100 != FLOOR(price * 100) THEN
    RETURN error("价格精度不能超过2位小数")
END IF

RETURN success()
```

**失败处理**: 返回错误"价格必须为正数且精度不超过2位小数"，阻止操作

---

#### BR-FIN-003: 医保编码唯一性校验

医保编码在系统中必须唯一对应一个收费项目，不允许同一医保编码对照多个项目。

**触发时机**: 建立医保目录对照时

**校验逻辑**:
```
IF insurance_code IS NULL OR insurance_code = '' THEN
    RETURN error("医保编码不能为空")
END IF

existing_mapping = SELECT * FROM his_insurance_mapping 
                   WHERE insurance_code = :insurance_code
                   AND deleted = 0

IF existing_mapping EXISTS AND existing_mapping.item_id != :item_id THEN
    RETURN error("医保编码已与[" + existing_mapping.item_name + "]对照")
END IF

RETURN success()
```

**失败处理**: 返回错误"医保编码已与其他项目对照"，显示已对照项目名称，阻止对照

---

#### BR-FIN-013: 收费项目状态流转规则

收费项目状态流转：草稿(0)→启用(1)→停用(2)；启用项目不可删除，只能停用；停用项目可重新启用。

**触发时机**: 项目状态变更时

**状态流转约束**:
```
valid_transitions = {
    0: [1],        // 草稿 -> 启用
    1: [2],        // 启用 -> 停用
    2: [1]         // 停用 -> 启用
}

IF target_status NOT IN valid_transitions[current_status] THEN
    RETURN error("状态流转不合法")
END IF

// 启用状态不允许删除
IF operation = 'DELETE' AND current_status = 1 THEN
    RETURN error("启用项目不可删除，请先停用")
END IF

RETURN success()
```

---

#### BR-FIN-014: 价格调整状态流转规则

价格调整状态流转：待审批(0)→待生效(1)→已生效(2)；待生效可取消；已生效不可修改。

**触发时机**: 价格调整状态变更时

**状态流转约束**:
```
valid_transitions = {
    0: [1, 3],     // 待审批 -> 待生效/已取消
    1: [2, 3],     // 待生效 -> 已生效/已取消
    2: [],         // 已生效 -> 不可变更
    3: []          // 已取消 -> 不可变更
}

IF target_status NOT IN valid_transitions[current_status] THEN
    RETURN error("价格调整状态流转不合法")
END IF

RETURN success()
```

## 7. 状态机设计

### 7.1 收费项目状态机

```
           ┌──────────────┐
           │    草稿(0)   │
           └──────────────┘
                  │
                  │ 新增完成
                  ↓
           ┌──────────────┐
      停用 │    启用(1)   │ ←──── 重新启用
           └──────────────┘           ↑
                  │                    │
                  │ 停用操作           │
                  ↓                    │
           ┌──────────────┐           │
           │    偗用(2)   │ ──────────┘
           └──────────────┘

状态说明：
- 草稿(0): 项目创建初始状态，信息不完整
- 启用(1): 项目可正常使用，在收费时可见
- 嗗用(2): 项目停止使用，在收费时不可见，但历史数据保留

状态流转规则：
- 草稿 → 启用: 信息完整后可启用
- 启用 → 偗用: 停用操作
- 嗗用 → 启用: 重新启用
- 启用状态不可删除，只能停用
```

### 7.2 价格调整状态机

```
           ┌──────────────┐
           │  待审批(0)   │
           └──────────────┘
                  │
                  │ 审批通过    ─────→ 审批拒绝 ──→ 已取消(3)
                  ↓
           ┌──────────────┐
      取消 │  待生效(1)   │ ←──── 定时生效设置
           └──────────────┘           ↑
                  │                    │
                  │ 生效日期到达       │ 取消操作
                  │ 或立即生效         │
                  ↓                    │
           ┌──────────────┐           │
           │  已生效(2)   │           │
           └──────────────┘           │
                                  │
           ┌──────────────┐           │
           │  已取消(3)   │ ←─────────┘
           └──────────────┘

状态说明：
- 待审批(0): 价格调整提交，等待审批
- 待生效(1): 定时生效，等待生效日期
- 已生效(2): 价格已更新生效
- 已取消(3): 调整被取消或审批拒绝

状态流转规则：
- 待审批 → 待生效: 审批通过（如需审批）
- 待审批 → 已取消: 审批拒绝
- 待生效 → 已生效: 生效日期到达
- 待生效 → 已取消: 手动取消
- 已生效、已取消状态不可变更
```
