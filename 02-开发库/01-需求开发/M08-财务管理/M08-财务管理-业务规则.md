### 3.2 FLOW (流转规则)

#### BR-FIN-013: 收费项目状态流转规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-FIN-013 |
| 规则名称 | 收费项目状态流转规则 |
| 规则类型 | FLOW (流转规则) |
| 适用对象 | 收费项目(charge_item) |
| 规则描述 | 收费项目状态流转：草稿→启用→停用；启用项目不可删除，只能停用；停用项目可重新启用 |
| 触发时机 | 项目状态变更时 |
| 校验逻辑 | ```
// 伪代码
// 状态定义：0草稿, 1启用, 2停用

valid_transitions = {
    0: [1],        // 草稿 -> 启用
    1: [2],        // 启用 -> 停用
    2: [1]         // 停用 -> 启用
}

IF target_status NOT IN valid_transitions[current_status] THEN
    RETURN error("状态流转不合法")
END IF

// 启用状态不允许删除
IF operation = 'DELETE' AND current_status = 1 THEN
    RETURN error("启用项目不可删除，请先停用")
END IF

RETURN success()
``` |
| 失败处理 | 返回错误"状态流转不合法"或"启用项目不可删除" |
| 优先级 | P0 |

// 伪代码
retry_count = 0

    TRY
        result = CALL medical_insurance_api.operation(params)
        IF result.status = 'SUCCESS' THEN
            RETURN success(result)
        END IF
    CATCH timeout_exception
        retry_count = retry_count + 1
        IF retry_count < max_retry THEN
            SLEEP(retry_interval[retry_count - 1])
            CONTINUE
        ELSE
            LOG error("医保接口超时，重试失败")
            RETURN error("医保系统响应超时，请稍后重试或人工处理")
        END IF
    END TRY
END WHILE
