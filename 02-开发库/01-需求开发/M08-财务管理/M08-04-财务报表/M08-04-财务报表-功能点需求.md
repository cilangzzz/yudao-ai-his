## 1. 功能概述

### 1.1 功能定位

财务报表是财务管理子系统的报表输出模块，负责财务数据的汇总统计和报表生成。系统支持收费员日结、科室日结、月结报表、收入统计等多维度统计分析，为医院财务管理和决策提供数据支撑。报表数据来源于收费结算、费用记账、医保结算等业务数据，实现财务数据的可视化呈现和归档管理。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提高财务报表生成效率 | 日结报表生成时间≤10秒 |
| 准确目标 | 保证报表数据准确无误 | 报表数据准确率100% |
| 合规目标 | 符合财务制度和审计要求 | 报表数据可追溯，审计覆盖率100% |
| 决策目标 | 为医院经营决策提供数据支撑 | 报表数据实时更新，多维度统计 |

## 7. 状态机设计

### 7.1 日结报表状态流转

```
状态定义：
┌─────────┐     确认日结      ┌─────────┐
│  待确认 │ ───────────────→ │  已确认 │
│   (1)   │                  │   (2)   │
└─────────┘                  └─────────┘
     │                            │
     │ 取消日结                    │ 不可变更
     │（仅数据错误时）             │
     ↓                            ↓
  回退处理                    锁定归档

状态流转规则：
1. 待确认(1) → 已确认(2)：收费员确认日结后流转
2. 已确认(2)：当日收费数据锁定，不可修改
3. 已确认状态不可回退，如数据错误需特殊审批流程

流转触发条件：
- 待确认 → 已确认：收费员点击"确认日结"，且无未完成交易
```

### 7.2 月结报表状态流转

```
状态定义：
┌─────────┐     确认月结      ┌─────────┐
│  待确认 │ ───────────────→ │  已确认 │
│   (1)   │                  │   (2)   │
└─────────┘                  └─────────┘
     │                            │
     │ 重新生成                    │ 不可变更
     │（数据有误时）               │
     ↓                            ↓
  更新数据                    锁定归档

状态流转规则：
1. 待确认(1) → 已确认(2)：财务会计确认月结后流转
2. 已确认(2)：当月收费数据锁定，不可修改
3. 待确认状态可重新生成报表（当数据有误时）

流转触发条件：
- 待确认 → 已确认：当月所有日结已完成，财务会计点击"确认月结"
```

### 8.2 关键业务规则详解

#### BR-FIN-009: 日结前未完成交易校验

日结前必须检查是否存在未完成的收费交易，存在未完成交易则不允许日结。

**触发时机**: 收费员日结确认时

**校验逻辑**:
```pseudo
pending_transactions = SELECT * FROM his_payment 
                       WHERE cashier_id = :cashier_id
                       AND payment_date = :report_date
                       AND payment_status = 0  // 进行中

IF COUNT(pending_transactions) > 0 THEN
    RETURN error("存在未完成的交易，请先处理", pending_transactions)
END IF

RETURN success()
```

**失败处理**: 返回错误"存在未完成的交易，请先处理"，显示未完成交易列表

---

#### BR-FIN-010: 月结前未日结校验

月结前必须检查当月所有日期是否已完成日结，存在未日结日期则不允许月结。

**触发时机**: 月结生成时

**校验逻辑**:
```pseudo
month_start = :report_month + '-01'
month_end = LAST_DAY(:report_month)

unsettled_dates = SELECT DISTINCT payment_date 
                   FROM his_payment 
                   WHERE payment_date BETWEEN month_start AND month_end
                   AND NOT EXISTS (
                       SELECT * FROM his_daily_report 
                       WHERE report_date = payment_date
                       AND status = 2  // 已确认
                   )

IF COUNT(unsettled_dates) > 0 THEN
    RETURN error("存在未日结的日期", unsettled_dates)
END IF

RETURN success()
```

**失败处理**: 返回错误"存在未日结的日期"，显示未日结日期列表

---

#### BR-FIN-030: 日结报表金额计算

日结报表金额：实收金额 = 收费总额 - 退费总额；按支付方式分类统计。

**触发时机**: 日结生成时

**计算逻辑**:
```pseudo
// 收费统计
collection_amount = SELECT SUM(amount), pay_type 
                    FROM his_payment 
                    WHERE cashier_id = :cashier_id
                    AND payment_date = :report_date
                    AND payment_status = 2  // 已完成
                    GROUP BY pay_type

// 退费统计
refund_amount = SELECT SUM(amount), pay_type 
                FROM his_refund 
                WHERE cashier_id = :cashier_id
                AND refund_date = :report_date
                AND refund_status = 2  // 已完成
                GROUP BY pay_type

// 计算实收金额
FOR each pay_type IN ['CASH', 'WECHAT', 'ALIPAY', 'INSURANCE'] DO
    report[pay_type + '_collection'] = collection_amount[pay_type] OR 0
    report[pay_type + '_refund'] = refund_amount[pay_type] OR 0
    report[pay_type + '_net'] = report[pay_type + '_collection'] - report[pay_type + '_refund']
END FOR

report.total_collection = SUM(collection_amount)
report.total_refund = SUM(refund_amount)
report.net_amount = report.total_collection - report.total_refund

RETURN report
```

---

#### BR-FIN-031: 月结报表金额计算

月结报表金额：汇总当月所有日结数据，按科室、支付方式分类统计。

**触发时机**: 月结生成时

**计算逻辑**:
```pseudo
month_start = :report_month + '-01'
month_end = LAST_DAY(:report_month)

// 汇总日结数据
daily_reports = SELECT * FROM his_daily_report 
                WHERE report_date BETWEEN month_start AND month_end

// 汇总各支付方式
FOR each report IN daily_reports DO
    monthly.cash_income = monthly.cash_income + report.cash_net
    monthly.wechat_income = monthly.wechat_income + report.wechat_net
    monthly.alipay_income = monthly.alipay_income + report.alipay_net
    monthly.insurance_income = monthly.insurance_income + report.insurance_net
END FOR

// 汇总科室收入
dept_income = SELECT dept_id, SUM(amount) 
              FROM his_charge_detail 
              WHERE charge_time BETWEEN month_start AND month_end
              AND charge_status = 2  // 已收费
              GROUP BY dept_id

monthly.total_income = SUM(daily_reports.net_amount)
monthly.dept_income = dept_income

RETURN monthly
```

---

#### BR-FIN-033: 科室收入占比计算

科室收入按费用类型计算占比，用于收入结构分析。

**触发时机**: 科室收入统计查询时

**计算逻辑**:
```pseudo
total_amount = SUM(dept_income.type_amount)

FOR each type IN dept_income DO
    type_ratio = type.amount / total_amount * 100
    type.ratio = type_ratio
END FOR

RETURN dept_income
```
