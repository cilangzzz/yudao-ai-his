## 1. 功能概述

### 1.1 功能定位

日志管理模块记录系统操作日志、登录日志，支持日志查询、导出，满足等保三级审计要求。日志是系统安全审计的重要依据，记录用户的操作行为，为问题排查和安全审计提供数据支持。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 合规目标 | 满足等保三级审计要求 | 日志保留≥3年 |
| 安全目标 | 完整记录用户操作 | 操作覆盖率100% |
| 运维目标 | 快速问题定位 | 日志查询≤3秒 |

### 2.3 字段定义

#### 操作日志字段定义

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| log_id | BIGINT | 是 | 日志ID（主键） |
| user_id | BIGINT | 是 | 操作用户ID |
| username | VARCHAR | 是 | 用户名 |
| operation_type | TINYINT | 是 | 操作类型：1查询/2新增/3修改/4删除/5导出/6打印 |
| module_code | VARCHAR | 是 | 模块编码 |
| module_name | VARCHAR | 是 | 模块名称 |
| operation_desc | VARCHAR | 否 | 操作描述 |
| request_url | VARCHAR | 否 | 请求URL |
| request_method | VARCHAR | 否 | 请求方法 |
| request_params | TEXT | 否 | 请求参数（脱敏） |
| response_code | INT | 否 | 响应状态码 |
| response_msg | VARCHAR | 否 | 响应消息 |
| ip_address | VARCHAR | 否 | 操作IP地址 |
| location | VARCHAR | 否 | 操作地点 |
| user_agent | VARCHAR | 否 | 用户代理 |
| browser | VARCHAR | 否 | 浏览器 |
| os | VARCHAR | 否 | 操作系统 |
| operation_time | DATETIME | 是 | 操作时间 |
| execution_time | INT | 否 | 执行时长(ms) |
| status | TINYINT | 是 | 状态：1成功/2失败 |
| error_msg | TEXT | 否 | 错误信息 |

### 3.2 故事详情

#### US-M09-018: 查看操作日志

**故事卡片**:
```
作为 系统管理员
我想要 查看用户的操作日志（按时间、用户、模块筛选）
以便于 追溯用户操作行为
```

**优先级**: P0 | **故事点**: 3 | **Sprint**: Sprint 2

**验收标准**:

```gherkin
Scenario: 查看操作日志列表
  Given 系统管理员已登录
  When 管理员进入操作日志页面
  Then 显示操作日志列表（默认显示最近1000条）
  And 每条日志包含：模块、操作人、IP、时间、操作类型、状态

Scenario: 筛选操作日志
  Given 管理员在操作日志页面
  When 管理员输入筛选条件：
    - 操作人 "doctor01"
    - 操作类型 "新增"
    - 时间范围 "2026-06-01 至 2026-06-30"
  Then 显示符合条件的日志列表

Scenario: 查看日志详情
  Given 管理员在操作日志列表
  When 管理员点击某条日志的"查看详情"
  Then 显示日志详情弹窗
  And 包含：请求URL、请求方法、请求参数（脱敏）、响应状态、执行时长

Scenario: 日志不可删除修改
  Given 管理员查看操作日志
  Then 不显示删除和修改按钮
  And 数据库禁止UPDATE和DELETE操作
```
