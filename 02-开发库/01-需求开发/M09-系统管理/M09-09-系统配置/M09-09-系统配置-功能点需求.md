## 1. 功能概述

### 1.1 功能定位

系统配置模块实现系统运行参数的统一管理。支持参数分类、参数配置、参数缓存、参数版本历史。系统配置为各业务模块提供可配置化能力，实现系统行为的灵活调整。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 配置目标 | 提供统一的参数管理 | 配置项覆盖率100% |
| 灵活目标 | 配置实时生效无需重启 | 配置生效时间≤5秒 |
| 安全目标 | 配置变更可追溯 | 变更历史100%记录 |

### 3.2 故事详情

#### US-M09-091: 创建系统参数

**故事卡片**:
```
作为 系统管理员
我想要 创建系统参数
以便于 配置系统运行参数
```

**优先级**: P0 | **故事点**: 3 | **Sprint**: Sprint 1

**验收标准**:

```gherkin
Scenario: 创建参数成功
  Given 系统管理员已登录
  And 在系统配置页面
  When 输入参数信息：
    | 字段 | 值 |
    | 参数名称 | 门诊号源上限 |
    | 参数键名 | op.reg.source.limit |
    | 参数分类 | 医疗配置 |
    | 参数值 | 100 |
    | 参数类型 | 数字 |
  And 点击保存
  Then 参数创建成功
  And 参数缓存自动更新

Scenario: 参数键名唯一性校验
  Given 系统管理员创建参数
  When 输入已存在的参数键名
  Then 提示 "参数键名已存在"
  And 无法保存
```

## 6. 数据模型

### 6.1 参数配置表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 参数ID |
| name | varchar(100) | 参数名称 |
| key | varchar(100) | 参数键名（唯一） |
| value | varchar(500) | 参数值 |
| type | tinyint | 参数类型(1字符串 2数字 3布尔) |
| category | varchar(50) | 参数分类 |
| remark | varchar(500) | 参数说明 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |
| deleted | tinyint | 删除标记 |

### 6.2 参数变更历史表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 记录ID |
| param_id | bigint | 参数ID |
| param_key | varchar(100) | 参数键名 |
| old_value | varchar(500) | 旧值 |
| new_value | varchar(500) | 新值 |
| operator | varchar(50) | 操作人 |
| operate_time | datetime | 操作时间 |
| operate_type | tinyint | 操作类型(1新增 2修改 3删除) |
