## 1. 用户管理验收标准

### AC-M09-001: 用户创建

```gherkin
Feature: 用户创建

  Scenario: 创建用户成功
    Given 系统管理员已登录系统
    And 拥有"system:user:add"权限
    When 管理员进入用户管理页面
    And 点击"新建用户"
    And 填写用户名 "doctor01"
    And 填写姓名 "张医生"
    And 填写工号 "D001"
    And 选择科室 "内科"
    And 选择角色 "门诊医生"
    And 点击"保存"
    Then 系统创建用户成功
    And 自动生成初始密码
    And 显示初始密码供管理员告知用户

  Scenario Outline: 用户名格式校验
    Given 管理员在新建用户页面
    When 输入用户名 "<username>"
    Then 校验结果为 "<result>"

    Examples:
      | username        | result                       |
      | ab              | 无效（少于3位）              |
      | abc             | 有效                         |
      | abcdefghij12345 | 有效                         |
      | admin@#$        | 无效（含特殊字符）            |
      | 1abc            | 无效（不以字母开头）          |

  Scenario: 用户名唯一性校验
    Given 系统中已存在用户名 "admin"
    When 管理员输入用户名 "admin"
    Then 系统提示 "用户名已存在"
    And 用户名字段标红

  Scenario: 工号唯一性校验
    Given 系统中已存在工号 "D001"
    When 管理员输入工号 "D001"
    Then 系统提示 "工号已存在"

  Scenario: 必填字段校验
    Given 管理员在新建用户页面
    When 管理员未填写用户名点击保存
    Then 提示 "用户名不能为空"
    When 管理员未填写姓名点击保存
    Then 提示 "姓名不能为空"
    When 管理员未选择科室点击保存
    Then 提示 "请选择所属科室"

  Scenario: 手机号格式校验
    Given 管理员在新建用户页面
    When 输入手机号 "12345"
    Then 提示 "手机号格式不正确"
    When 输入手机号 "13800138000"
    Then 校验通过

  Scenario: 邮箱格式校验
    Given 管理员在新建用户页面
    When 输入邮箱 "invalid-email"
    Then 提示 "邮箱格式不正确"
    When 输入邮箱 "test@example.com"
    Then 校验通过
```

### AC-M09-002: 用户登录

```gherkin
Feature: 用户登录

  Scenario: 登录成功
    Given 用户在登录页面
    And 用户 "doctor01" 状态为"正常"
    When 输入用户名 "doctor01"
    And 输入密码 "Abc12345!"
    And 点击"登录"
    Then 登录成功
    And 返回Token
    And 记录登录日志（状态：成功）
    And 清空失败计数
    And 跳转首页

  Scenario: 用户名不存在
    Given 用户在登录页面
    When 输入不存在的用户名 "notexist"
    And 输入任意密码
    And 点击"登录"
    Then 提示 "用户名或密码错误"

  Scenario: 密码错误
    Given 用户在登录页面
    And 用户 "doctor01" 存在
    When 输入错误密码
    And 点击"登录"
    Then 提示 "用户名或密码错误"
    And 失败计数+1
    And 记录登录日志（状态：失败）

  Scenario: 连续失败锁定
    Given 用户 "doctor01" 已连续失败4次
    When 再次输入错误密码登录
    Then 失败计数达到5次
    And 账号锁定30分钟
    And 提示 "登录失败次数过多，账户已锁定30分钟"

  Scenario: 锁定中尝试登录
    Given 用户账号已锁定（锁定时间开始30分钟内）
    When 尝试登录
    Then 提示 "账户已锁定，请等待X分钟后再尝试"
    And 登录按钮禁用

  Scenario: 自动解锁
    Given 用户账号已锁定
    And 当前时间超过锁定时间30分钟
    When 输入正确密码登录
    Then 登录成功
    And 失败计数清空
    And 锁定状态解除

  Scenario: 账号停用
    Given 用户 "doctor01" 状态为"停用"
    When 尝试登录
    Then 提示 "账号已停用，请联系管理员"

  Scenario Outline: 密码复杂度校验
    Given 用户在修改密码页面
    When 输入新密码 "<password>"
    Then 校验结果为 "<result>"

    Examples:
      | password     | result                         |
      | 123456       | 无效（缺少大小写和特殊字符）   |
      | Abc12345     | 无效（缺少特殊字符）           |
      | Abc12345!    | 有效                           |
      | Ab1!         | 无效（少于8位）                |

  Scenario: 密码修改后需重新登录
    Given 用户已登录
    When 修改密码成功
    Then Token失效
    And 跳转登录页面
    And 提示 "密码已修改，请重新登录"
```

### AC-M09-003: 用户状态管理

```gherkin
Feature: 用户状态管理

  Scenario: 停用用户
    Given 用户 "doctor01" 状态为"正常"
    When 管理员将状态改为"停用"
    Then 用户状态更新为"停用"
    And 用户无法登录
    And 已登录用户Token失效

  Scenario: 启用用户
    Given 用户 "doctor01" 状态为"停用"
    When 管理员将状态改为"正常"
    Then 用户状态更新为"正常"
    And 用户可以正常登录

  Scenario: 删除用户
    Given 用户 "test01" 无关联业务数据
    When 管理员点击"删除"
    And 确认删除
    Then 用户逻辑删除（is_deleted=1）
    And 列表中不再显示

  Scenario: 删除有关联数据的用户
    Given 用户 "doctor01" 有处方数据
    When 管理员尝试删除
    Then 提示 "该用户有关联业务数据，建议停用账号"
    And 删除操作被阻止
```

## 2. 角色权限验收标准

### AC-M09-005: 角色创建与权限分配

```gherkin
Feature: 角色创建与权限分配

  Scenario: 创建角色并分配菜单权限
    Given 管理员已登录
    And 拥有"system:role:add"权限
    When 进入角色管理页面
    And 点击"新建角色"
    And 填写角色名称 "门诊医生"
    And 填写角色标识 "op_doctor"
    And 勾选菜单权限：
      | 菜单         | 是否选中 |
      | 门诊管理     | ☑        |
      | 医生工作站   | ☑        |
      | 挂号管理     | □        |
    And 点击"保存"
    Then 角色创建成功

  Scenario: 分配按钮权限
    Given 管理员在角色编辑页面
    When 勾选按钮权限：
      | 按钮         | 是否选中 |
      | 接诊         | ☑        |
      | 开立处方     | ☑        |
      | 导出         | □        |
    Then 按钮权限分配成功

  Scenario: 分配数据权限
    Given 管理员在角色编辑页面
    When 选择数据权限 "本部门数据"
    Then 数据权限设置成功
    And 用户查询数据时自动过滤为本科室

  Scenario: 角色标识唯一性
    Given 系统中已存在角色标识 "admin"
    When 管理员输入角色标识 "admin"
    Then 提示 "角色标识已存在"
```

### AC-M09-006: 权限生效验证

```gherkin
Feature: 权限生效验证

  Scenario: 菜单权限控制
    Given 用户 "doctor01" 角色为"门诊医生"
    And 该角色菜单权限：门诊管理、医生工作站
    When 用户登录系统
    Then 显示授权菜单
    And 隐藏未授权菜单（挂号管理、收费管理等）

  Scenario: 按钮权限控制
    Given 用户 "doctor01" 角色为"门诊医生"
    And 该角色有"接诊"权限，无"导出"权限
    When 进入医生工作站
    Then 显示"接诊"按钮
    And 隐藏"导出"按钮

  Scenario: 越权操作拦截
    Given 用户 "doctor01" 无"用户管理"权限
    When 直接访问用户管理URL "/system/user"
    Then 返回403错误
    And 显示 "无权访问该功能"

  Scenario: 数据权限过滤
    Given 用户 "doctor01" 数据权限为"本部门数据"
    And 用户所属科室为"内科"
    When 查询患者列表
    Then 只显示内科患者数据

  Scenario: 权限变更实时生效
    Given 用户 "doctor01" 已登录
    And 角色有"导出"权限
    When 管理员移除该角色的"导出"权限
    And 用户刷新页面
    Then 用户无法看到"导出"按钮
```

## 3. 组织架构验收标准

### AC-M09-007: 科室管理

```gherkin
Feature: 科室管理

  Scenario: 创建科室成功
    Given 管理员已登录
    When 进入科室管理页面
```

## 5. 日志审计验收标准

### AC-M09-009: 操作日志

```gherkin
Feature: 操作日志

  Scenario: 查看操作日志
    Given 管理员已登录
    When 进入操作日志页面
    Then 显示日志列表
    And 每条日志包含：模块、操作人、IP、时间、操作类型、状态

  Scenario: 筛选操作日志
    Given 管理员在操作日志页面
    When 输入筛选条件：
      | 条件     | 值                  |
      | 操作人   | doctor01            |
      | 操作类型 | 新增                |
      | 时间范围 | 2026-06-01至06-30   |
    Then 显示符合条件的日志

  Scenario: 查看日志详情
    Given 管理员在日志列表
    When 点击"查看详情"
    Then 显示完整日志信息：
      - 请求URL
      - 请求方法
      - 请求参数（脱敏）
      - 响应状态码
      - 执行时长

  Scenario: 日志不可删除修改
    Given 管理员查看操作日志
    Then 不显示删除和修改按钮
    And 数据库禁止UPDATE和DELETE操作

  Scenario: 导出日志
    Given 管理员在操作日志页面
    When 点击"导出"
    Then 导出Excel文件
    And 文件名格式：操作日志_2026-06-16.xlsx
```

## 6. 非功能验收标准

### AC-M09-NFR: 性能与安全

```gherkin
Feature: 性能与安全

  Scenario: 性能验收
    Given 系统正常运行
    When 查询用户列表（1000条数据）
    Then 响应时间 ≤ 2秒
    When 创建用户
    Then 响应时间 ≤ 1秒
    When 用户登录
    Then 响应时间 ≤ 1秒

  Scenario: 密码安全
    Given 用户设置密码
    Then 密码使用BCrypt加密存储
    And 密码不可逆解密

  Scenario: Token有效期
    Given 用户登录成功
    When Token超过2小时未使用
    Then Token自动失效
    And 需重新登录

  Scenario: 审计合规
    Given 系统运行中
    Then 所有用户操作记录审计日志
    And 日志保留期限 ≥ 3年
    And 日志不可删除修改
```

| 角色创建 | 创建成功，分配权限 | 功能测试 | ☐ |
| 菜单权限 | 未授权菜单隐藏 | 功能测试 | ☐ |
