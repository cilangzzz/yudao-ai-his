## 1. 子系统概述

### 1.1 子系统定位

系统管理子系统是YUDAO-AI-HIS的基础支撑模块，为所有业务子系统提供用户认证、权限控制、数据字典、组织架构、系统配置、审计日志等基础服务。作为MVP第一期的首要开发模块，所有其他子系统均依赖本模块。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 安全目标 | 实现基于RBAC的细粒度权限控制 | 权限控制覆盖率100% |
| 合规目标 | 满足等保三级审计要求 | 审计日志保留≥3年 |
| 效率目标 | 支持快速用户管理和权限配置 | 用户创建≤30秒 |
| 运维目标 | 提供完整的系统监控和配置能力 | 接口监控实时告警 |

#### 2.1.4 交互流程

##### 创建用户流程

```
用户操作                    系统响应
─────────────────────────────────────────────
点击"新建用户"        →    打开新建页面
填写用户名            →    实时校验格式和唯一性
填写姓名、工号        →    校验必填和格式
选择科室              →    加载科室下拉列表
选择角色              →    加载角色复选框列表
点击"保存"            →    
                      ←    校验所有字段
                      ←    校验通过：创建用户
                      ←    自动生成初始密码
                      ←    显示成功提示
                      ←    返回列表页
                      ←    校验失败：显示错误提示
```

##### 用户登录流程

```
用户操作                    系统响应
─────────────────────────────────────────────
输入用户名            →    
输入密码              →    
点击"登录"            →    
                      ←    校验用户名是否存在
                      ←    校验账号状态（是否停用/锁定）
                      ←    校验密码是否正确
                      ←    校验通过：
                           - 清空失败计数
                           - 记录登录日志
                           - 创建Session
                           - 跳转首页
                      ←    校验失败：
                           - 增加失败计数
                           - 记录失败日志
                           - 失败≥5次：锁定30分钟
                           - 显示错误提示
```

##### 密码重置流程

```
管理员操作                  系统响应
─────────────────────────────────────────────
点击用户"重置密码"    →    
                      ←    弹出确认对话框
确认重置              →    
                      ←    生成新的初始密码
                      ←    更新密码字段
                      ←    清空失败计数
                      ←    解除锁定
                      ←    显示新密码（供管理员告知用户）
                      ←    发送短信/邮件通知用户
```

### 2.3 M09-03 权限管理

#### 2.3.1 功能概述

权限管理模块实现菜单权限、按钮权限、数据权限的统一管理。支持权限标识定义、权限树展示、权限分配。

#### 2.3.2 权限类型定义

| 权限类型 | 标识格式 | 作用范围 | 示例 |
|----------|----------|----------|------|
| 菜单权限 | 无标识 | 页面可见性 | 用户管理菜单 |
| 按钮权限 | module:sub:action | 操作可用性 | system:user:add |
| 数据权限 | 数据范围规则 | 数据可见范围 | 本科室数据 |

#### 2.3.3 按钮权限标识规范

```
权限标识格式: {模块}:{子模块}:{操作}

示例：
- system:user:list    → 用户列表查询
- system:user:add     → 用户新增
- system:user:edit    → 用户修改
- system:user:remove  → 用户删除
- system:user:export  → 用户导出
- op:doctor:visit     → 门诊接诊
- op:doctor:prescribe → 门诊开方
- op:doctor:view      → 查看患者信息
- ip:nurse:medicate   → 给药执行
- pharm:audit:pass    → 处方审核通过
```

#### 2.3.4 数据权限规则

| 数据范围 | 标识值 | SQL条件 | 说明 |
|----------|--------|---------|------|
| 全部数据 | 1 | 无限制 | 可查看全院所有数据 |
| 自定义数据 | 2 | dept_id IN (自定义科室列表) | 查看指定科室数据 |
| 本部门数据 | 3 | dept_id = 当前用户科室 | 仅查看本科室数据 |
| 本部门及下级 | 4 | dept_id IN (当前科室及下级) | 查看本科室及下属科室数据 |
| 仅本人数据 | 5 | create_by = 当前用户 | 仅查看自己创建的数据 |

|------|----------|
