#### BR-SUR-001: 手术申请完整性校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-SUR-001 |
| 规则名称 | 手术申请完整性校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 手术申请(surgery_request) |
| 规则描述 | 手术申请必须包含完整的患者信息、诊断、手术名称、申请科室、申请医生，缺少必要信息时不可接收 |
| 触发时机 | 接收手术申请时 |
| 校验逻辑 | ```
// 伪代码
required_fields = [
    'patient_id',        // 患者ID
    'patient_name',      // 患者姓名
    'diagnosis_code',    // 术前诊断编码
    'surgery_code',      // 手术编码(ICD-9-CM-3)
    'surgery_name',      // 手术名称
    'apply_dept_id',     // 申请科室
    'apply_doctor_id',   // 申请医生
    'admission_id'       // 住院记录ID
]

missing_fields = []
FOR each field IN required_fields DO
    IF request[field] IS NULL OR request[field] = '' THEN
        missing_fields.add(field)
    END IF
END FOR

IF COUNT(missing_fields) > 0 THEN
    RETURN error("申请信息不完整，缺少：" + JOIN(missing_fields, ', '))
END IF

RETURN success()
``` |
| 失败处理 | 返回错误提示"申请信息不完整，缺少：[字段列表]"，阻止申请接收 |
| 优先级 | P0 |

#### BR-SUR-003: 手术分级权限校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-SUR-003 |
| 规则名称 | 手术分级权限校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 手术申请(surgery_request)、医生(doctor) |
| 规则描述 | 手术分级与医生职称需匹配：一级手术住院医师可主刀、二级手术主治医师可主刀、三级手术副主任医师可主刀、四级手术主任医师可主刀 |
| 触发时机 | 审核手术申请时 |
| 校验逻辑 | ```
// 伪代码
surgery_level = GET_surgery_level(request.surgery_code)
doctor_title = GET_doctor_title(request.surgeon_id)

// 手术分级权限对照表
level_permission = {
    'LEVEL_1': ['住院医师', '主治医师', '副主任医师', '主任医师'],
    'LEVEL_2': ['主治医师', '副主任医师', '主任医师'],
    'LEVEL_3': ['副主任医师', '主任医师'],
    'LEVEL_4': ['主任医师']
}

IF doctor_title NOT IN level_permission[surgery_level] THEN
    // 检查是否有专项授权
    IF HAS_special_authorization(request.surgeon_id, request.surgery_code) THEN
        RETURN success_with_note("通过专项授权")
    END IF
    
    RETURN error("手术分级与医生权限不匹配，" + surgery_level + "需" + 
                 JOIN(level_permission[surgery_level], '或') + "主刀")
END IF

RETURN success()
``` |
| 失败处理 | 返回错误提示"手术分级与医生权限不匹配"，提供上级医师选择或专项授权申请 |
| 优先级 | P0 |

#### BR-SUR-006: 手术室可用性校验

|------|------|
| 规则编号 | BR-SUR-006 |
| 规则名称 | 手术室可用性校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 手术排期(surgery_schedule)、手术室(operating_room) |
| 规则描述 | 排期时需校验手术室在指定时段是否可用，状态为"维护"或"消毒"的手术室不可排期 |
| 触发时机 | 创建/修改手术排期时 |
| 校验逻辑 | ```
// 伪代码
room = SELECT * FROM operating_room WHERE room_id = :room_id

IF room.status NOT IN ('AVAILABLE', 'OCCUPIED') THEN
    RETURN error("手术室当前状态为" + room.status + "，不可排期")
END IF

// 检查时段冲突
existing_schedule = SELECT * FROM surgery_schedule 
                     WHERE room_id = :room_id 
                       AND schedule_date = :schedule_date
                       AND time_slot OVERLAPS :time_slot
                       AND status NOT IN ('CANCELLED', 'COMPLETED')

IF COUNT(existing_schedule) > 0 THEN
    RETURN error("手术室该时段已有排期")
END IF

// 检查维护计划
maintenance = SELECT * FROM room_maintenance 
               WHERE room_id = :room_id 
                 AND maintenance_time OVERLAPS :schedule_date + :time_slot

IF COUNT(maintenance) > 0 THEN
    RETURN error("手术室该时段有维护计划")
END IF

RETURN success()
``` |
| 失败处理 | 返回错误提示，显示可用时段建议 |
| 优先级 | P0 |

// 伪代码
warnings = []

    })

        'level': 'WARNING',

    })

    warnings.add({
        'type': 'SPO2_LOW',
        'level': 'CRITICAL',
        'message': '血氧过低：' + vital_sign.spo2 + '%'
    })
END IF

IF COUNT(warnings) > 0 THEN
    SEND_alert(warnings)
END IF

RETURN success_with_warnings(warnings)
``` |
