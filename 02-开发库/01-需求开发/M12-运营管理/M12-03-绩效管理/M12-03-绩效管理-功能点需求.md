## 1. 功能概述

### 1.1 功能定位

绩效管理模块是运营管理子系统的核心考核模块，负责绩效指标配置、绩效评分自动计算、绩效报表生成与发布。系统整合门诊、住院、病历质控、患者满意度等多维度数据，实现医护人员工作绩效的量化考核，为医院绩效分配和人才管理提供数据支撑。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提升绩效考核效率 | 绩效计算自动化率≥95% |
| 公平目标 | 实现绩效量化管理 | 绩效数据可追溯率100% |
| 决策目标 | 为绩效分配提供依据 | 绩效报表准确率≥99% |
| 透明目标 | 绩效结果公开透明 | 绩效申诉处理率100% |

### 1.3 功能范围

```
M12-03 绩效管理
├── 绩效指标配置
│   ├── 指标创建与编辑

│   ├── 数据自动采集
│   ├── 工作量指标计算
│   ├── 质量指标计算
│   ├── 服务指标计算
│   ├── 效率指标计算

│   ├── 绩效排名生成
│   └── 异常数据处理

    ├── 绩效结果发布

```

### 1.4 用户角色

| 角色 | 主要职责 | 使用功能 |
|------|----------|----------|
| 绩效专员 | 配置绩效指标、执行绩效计算、发布绩效结果 | 绩效指标配置、绩效评分计算、绩效报表生成 |

## 4. 字段定义

### 4.1 绩效指标表 (perf_indicator)

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| indicator_id | BIGINT | 是 | 指标ID（主键） |
| indicator_code | VARCHAR(30) | 是 | 指标编码，格式：IND+类型+序号 |
| indicator_name | VARCHAR(100) | 是 | 指标名称 |
| indicator_type | TINYINT | 是 | 指标类型：1工作量/2质量/3服务/4效率 |
| scheme_id | BIGINT | 是 | 所属方案ID |
| weight | DECIMAL(5,2) | 是 | 权重百分比，如30.00表示30% |
| target_value | DECIMAL(10,2) | 是 | 目标值 |
| target_unit | VARCHAR(20) | 是 | 目标值单位，如人次、%、天 |
| target_type | TINYINT | 是 | 目标类型：1绝对值/2百分比/3相对值 |
| calc_period | TINYINT | 是 | 计算周期：1日/2周/3月/4季/5年 |
| calc_rule | VARCHAR(500) | 否 | 计算规则JSON |
| bonus_ratio | DECIMAL(5,2) | 否 | 超额奖励比例 |
| max_bonus | DECIMAL(5,2) | 否 | 最高奖励分数 |
| completion_factor | DECIMAL(3,2) | 否 | 完成系数，默认0.80 |
| is_negative | TINYINT | 否 | 是否负向指标：0否/1是 |
| dept_scope | VARCHAR(500) | 否 | 适用科室ID列表，逗号分隔 |
| data_source | VARCHAR(200) | 是 | 数据来源模块 |
| sort_no | INT | 否 | 排序号 |
| status | TINYINT | 是 | 状态：1启用/2停用 |
| create_time | DATETIME | 是 | 创建时间 |
| create_by | VARCHAR(50) | 是 | 创建人 |
| update_time | DATETIME | 否 | 更新时间 |
| update_by | VARCHAR(50) | 否 | 更新人 |

## 8. 业务规则

### 8.1 校验规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OPS-003 | 绩效指标权重校验 | 同一绩效考核方案下，所有指标的权重总和必须等于100% |
| BR-OPS-004 | 绩效指标目标值校验 | 目标值必须合理，不能为负数；工作量指标目标值应为正整数，质量指标目标值不超过100% |

### 8.2 流转规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OPS-007 | 绩效考核流程状态流转 | 绩效周期状态流转：初始化→数据采集→计算中→待审核→已发布→已归档 |
| BR-OPS-008 | 绩效申诉流程 | 绩效申诉流程：申诉提交→待审核→审核通过/审核驳回→重新计算(如通过) |

### 8.3 权限规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OPS-012 | 绩效配置权限 | 只有绩效专员和人力资源管理员可以配置绩效指标，科室主任可以查看但不能修改 |
| BR-OPS-011 | 运营数据查看权限 | 不同角色只能查看对应范围的运营数据：院长看全院，科室主任看本科室，普通员工看个人 |

### 8.4 计算规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OPS-019 | 绩效得分计算 | 绩效总分 = Σ(指标得分 × 指标权重)，指标得分根据实际值与目标值对比计算 |
| BR-OPS-020 | 绩效排名计算 | 按绩效总分降序排名，同分时按工号排序，支持按科室分组排名 |

#### BR-OPS-019 绩效得分计算详细规则

```
绩效总分 = Σ(指标得分 × 指标权重)

指标得分计算：
- 达到目标值：得分 = 满分
- 超过目标值：得分 = 满分 × (1 + 超额比例 × 奖励系数)
- 未达目标值：得分 = 满分 × (实际值 / 目标值) × 完成系数

特殊规则：
- 投诉次数为负向指标，超过目标扣分
- 危急值未及时处理按次扣分
- 医疗事故一票否决
```

### 8.5 绩效指标类型定义

| 指标类型 | 指标名称 | 数据来源 | 计算方式 | 权重范围 |
|----------|----------|----------|----------|----------|
| 工作量 | 门诊接诊人次 | M01门诊管理 | COUNT(接诊记录) | 20-40% |
| 工作量 | 住院管理人次 | M02住院管理 | COUNT(住院记录) | 15-30% |
| 工作量 | 手术台次 | M07手术麻醉 | COUNT(手术记录) | 10-20% |
| 质量 | 病历书写合格率 | M03电子病历 | 合格病历数/总病历数 | 10-20% |
| 质量 | 处方合格率 | M06药品管理 | 合格处方数/总处方数 | 5-15% |
| 质量 | 危急值处理及时率 | M04检验管理 | 及时处理数/总数 | 5-10% |
| 服务 | 患者满意度 | M11患者服务 | 满意评分均值 | 10-20% |
| 服务 | 投诉次数 | M09系统管理 | COUNT(投诉记录) | 5-10% |
| 效率 | 平均住院日控制 | M02住院管理 | AVG(住院天数) | 5-15% |
| 效率 | 床位周转次数 | M02住院管理 | 出院人次/平均开放床位 | 5-10% |
