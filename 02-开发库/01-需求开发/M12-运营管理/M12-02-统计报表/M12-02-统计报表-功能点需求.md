## 1. 功能概述

### 1.1 功能定位

统计报表模块是运营管理子系统的核心数据分析模块，为医院管理者和统计人员提供多维度的业务统计分析能力。模块整合门诊、住院、药品、手术等业务数据，生成科室工作量、医生工作量、药品消耗、手术统计等标准化报表，支持自定义报表和多种格式导出，为医院经营决策提供数据支撑。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提升报表生成效率 | 日报生成≤30秒，月报生成≤60秒 |
| 管理目标 | 实现业务数据量化分析 | 报表数据准确率≥99.9% |
| 决策目标 | 为资源配置提供依据 | 支持多维度交叉分析 |
| 合规目标 | 报表格式符合卫健委规范 | 报表格式100%合规 |

### 1.3 功能范围

```
M12-02 统计报表
├── 科室工作量统计
│   ├── 科室工作量日报（门诊人次、住院人次、手术台次）
│   ├── 科室工作量月报（日均门诊量、平均住院日）
│   ├── 科室收入统计（门诊收入、住院收入、药品收入）
│   └── 科室筛选与导出
├── 医生工作量统计
│   ├── 医生工作量日报（接诊人次、处方数、检查申请数）
│   ├── 医生工作量月报（接诊人次、手术台次、会诊次数）
│   ├── 医生排班统计（出勤天数、加班时长）
│   └── 个人工作量详情查看
├── 药品消耗统计
│   ├── 药品消耗日报（发药量、销售金额、库存量）
│   ├── 药品消耗月报（消耗量、消耗金额、周转率）
│   ├── 抗菌药物专项统计（使用量、使用率、DDD值）
│   └── 科室药品消耗分析
└── 手术统计报表
    ├── 手术量日报（手术台次、手术时长）
    ├── 手术量月报（手术台次、三四级手术占比）
    ├── 手术分级统计（一至四级手术台次、占比）
    └── 手术时长分析
```

### 1.4 用户角色

| 角色 | 主要职责 | 使用功能 |
|------|----------|----------|
| 统计人员 | 数据统计分析、报表生成 | 科室工作量统计、医生工作量统计、药品消耗统计 |

| 财务人员 | 收入数据分析 | 科室收入统计 |

## 4. 字段定义

### 4.1 科室工作量报表字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| report_id | BIGINT | 是 | 报表ID（主键） |
| report_type | VARCHAR(20) | 是 | 报表类型：RPT-DEPT-001/002/003 |
| report_name | VARCHAR(100) | 是 | 报表名称 |
| stat_date_start | DATE | 是 | 统计开始日期 |
| stat_date_end | DATE | 是 | 统计结束日期 |
| dept_id | BIGINT | 是 | 科室ID |
| dept_name | VARCHAR(100) | 是 | 科室名称 |
| outpatient_count | INT | 是 | 门诊人次 |
| inpatient_count | INT | 是 | 住院人次 |
| surgery_count | INT | 否 | 手术台次 |
| admission_count | INT | 否 | 入院人数（月报） |
| discharge_count | INT | 否 | 出院人数（月报） |
| avg_stay_days | DECIMAL(5,1) | 否 | 平均住院日（月报） |
| avg_daily_outpatient | DECIMAL(10,1) | 否 | 日均门诊量（月报） |
| outpatient_revenue | DECIMAL(12,2) | 否 | 门诊收入（收入报表） |
| inpatient_revenue | DECIMAL(12,2) | 否 | 住院收入（收入报表） |
| drug_revenue | DECIMAL(12,2) | 否 | 药品收入（收入报表） |
| total_revenue | DECIMAL(12,2) | 是 | 总收入 |

### 8.2 关键业务规则详解

#### BR-OPS-001: 数据时间范围校验

报表生成时必须校验时间范围有效性：
- 结束日期不能晚于当前日期
- 历史数据查询不能超过365天
- 开始日期不能晚于结束日期
- 月报查询时间范围不能超过31天

**触发时机**: 生成报表时

**校验逻辑**:
```
IF end_date > CURRENT_DATE THEN
    RETURN error("结束日期不能晚于当前日期")
END IF

IF start_date < CURRENT_DATE - 365 THEN
    RETURN error("查询时间范围不能超过1年")
END IF

IF start_date > end_date THEN
    RETURN error("开始日期不能晚于结束日期")
END IF

IF report_type = 'MONTHLY' AND (end_date - start_date) > 31 THEN
    RETURN error("月报查询时间范围不能超过31天")
END IF
```

#### BR-OPS-021: 科室工作量汇总计算

科室工作量统计的计算逻辑：

**门诊人次**: COUNT(挂号记录) WHERE visit_date IN period AND dept_id = target_dept

**住院人次**: COUNT(住院记录) WHERE admission_date IN period AND dept_id = target_dept

**手术台次**: COUNT(手术记录) WHERE surgery_date IN period AND dept_id = target_dept

**总收入**: SUM(收费金额) WHERE payment_date IN period AND dept_id = target_dept

#### BR-OPS-022: 药品周转率计算

药品周转率计算逻辑：
```
turnover_rate = period_consumption / average_stock
turnover_days = period_days / turnover_rate

// 周转率>=3表示周转良好
// 周转率<1表示周转缓慢，需关注库存管理
```
