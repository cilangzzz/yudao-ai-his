## 1. 功能概述

### 1.1 功能定位

运营看板是运营管理子系统的核心数据可视化模块，为医院管理者提供全院运营数据的实时展示和分析能力。系统整合门诊、住院、床位、收入等多维业务数据，通过可视化图表和核心指标展示，帮助管理者快速掌握医院运营状况，支持数据驱动的管理决策。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提升医院运营决策效率 | 数据查询响应时间<=5秒 |
| 可视化目标 | 运营数据直观展示 | 核心指标实时更新 |
| 预警目标 | 床位紧张及时预警 | 预警响应时间<=1分钟 |
| 决策目标 | 为经营决策提供数据支撑 | 关键指标数据准确性>=99.9% |

### 2.3 数据字段定义

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| id | BIGINT | 是 | 主键ID |
| indicator_code | VARCHAR(50) | 是 | 指标编码 |
| indicator_name | VARCHAR(100) | 是 | 指标名称 |
| indicator_value | DECIMAL(15,2) | 是 | 指标值 |
| unit | VARCHAR(20) | 否 | 单位 |
| trend | VARCHAR(10) | 否 | 趋势：UP/DOWN/FLAT |
| change_rate | DECIMAL(10,2) | 否 | 变化率(%) |
| data_type | VARCHAR(20) | 是 | 数据类型：OUTPATIENT/INPATIENT/BED/REVENUE |
| stat_date | DATE | 是 | 统计日期 |
| stat_time | DATETIME | 是 | 统计时间 |
| dept_id | BIGINT | 否 | 科室ID（科室维度统计时） |
| dept_name | VARCHAR(100) | 否 | 科室名称 |
| create_time | DATETIME | 是 | 创建时间 |
| update_time | DATETIME | 否 | 更新时间 |

| BR-OPS-026 | 住院数据同步 | 集成规则 | 数据采集 |
| BR-OPS-027 | 财务数据同步 | 集成规则 | 数据采集 |

### 5.2 关键业务规则详解

#### BR-OPS-015: 床位使用率计算

床位使用率 = 实际占用床位数 / 开放床位数 x 100%

**触发时机**: 实时计算、定时汇总时

**计算逻辑**:
1. 获取开放床位数：SELECT COUNT(*) FROM bed WHERE status = 'OPEN'
2. 获取占用床位数：SELECT COUNT(*) FROM bed WHERE status = 'OCCUPIED'
3. 计算使用率：(occupied_beds / open_beds) * 100，保留1位小数
4. 使用率>=95%时，触发床位紧张预警

#### BR-OPS-017: 门诊量增长率计算

门诊增长率 = (今日门诊量 - 昨日门诊量) / 昨日门诊量 x 100%

**触发时机**: 实时计算时

**计算逻辑**:
- 今日门诊量：COUNT(挂号记录) WHERE register_date = TODAY
- 昨日门诊量：COUNT(挂号记录) WHERE register_date = YESTERDAY
- 环比增长率：(today_count - yesterday_count) / yesterday_count * 100
- 同比增长率：(today_count - last_week_same_day) / last_week_same_day * 100

#### BR-OPS-018: 收入构成占比计算

各项收入占比 = 各项收入金额 / 总收入 x 100%

**触发时机**: 收入分析时

**计算逻辑**:
- 门诊收入：SUM(门诊收费) WHERE payment_date = TODAY
- 住院收入：SUM(住院收费) WHERE payment_date = TODAY
- 药品收入：SUM(药品收费) WHERE payment_date = TODAY
- 总收入 = 门诊收入 + 住院收入 + 药品收入 + 其他收入
- 各项占比：(各项收入 / 总收入) * 100，保留1位小数
