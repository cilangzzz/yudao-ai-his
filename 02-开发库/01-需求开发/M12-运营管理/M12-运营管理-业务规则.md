### 3.1 校验规则 (VAL)

#### BR-OPS-001: 数据时间范围校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-001 |
| 规则名称 | 数据时间范围校验 |
| 规则类型 | VAL (校验规则) |
| 适用对象 | 统计报表、数据分析 |
| 规则描述 | 统计报表和数据分析的时间范围必须在有效期内，不能查询未来的数据，历史数据查询有时间限制 |
| 触发时机 | 生成报表、查询数据时 |
| 校验逻辑 | ```
// 伪代码
today = CURRENT_DATE

// 不能查询未来数据
IF end_date > today THEN
    RETURN error("结束日期不能晚于当前日期")
END IF

// 历史数据查询限制
max_history_days = 365  // 最多查询1年历史数据
IF start_date < today - max_history_days THEN
    RETURN error("查询时间范围不能超过1年")
END IF

// 时间范围合理性
IF start_date > end_date THEN
    RETURN error("开始日期不能晚于结束日期")
END IF

// 月报特殊校验
IF report_type = 'MONTHLY' THEN
    IF end_date - start_date > 31 THEN
        RETURN error("月报查询时间范围不能超过31天")
    END IF
END IF

RETURN success()
``` |
| 失败处理 | 返回错误提示，阻止查询操作 |
| 优先级 | P0 |

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-002 |
| 规则名称 | 科室数据权限校验 |
| 规则类型 | VAL (校验规则) |
| 适用对象 | 所有数据查询、报表生成 |
| 规则描述 | 用户只能查看有权限的科室数据，科室主任只能查看本科室数据，院长可查看全院数据 |

// 伪代码
user = CURRENT_USER

// 管理员角色可查看全院
IF user.has_role('ADMIN') OR user.has_role('DEAN') THEN
    RETURN success(all_depts)
END IF

// 科室主任角色只能查看本科室
IF user.has_role('DEPT_DIRECTOR') THEN
    user_dept = user.dept_id

allowed_depts = SELECT dept_id FROM user_data_permission WHERE user_id = user.user_id

END IF

#### BR-OPS-003: 绩效指标权重校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-003 |
| 规则名称 | 绩效指标权重校验 |
| 规则类型 | VAL (校验规则) |
| 适用对象 | 绩效指标配置 |
| 规则描述 | 同一绩效考核方案下，所有指标的权重总和必须等于100% |
| 触发时机 | 保存绩效指标配置时 |
| 校验逻辑 | ```
// 伪代码
// 查询同一方案下的所有指标
indicators = SELECT * FROM perf_indicator WHERE scheme_id = :scheme_id

// 计算权重总和
total_weight = SUM(indicator.weight FOR indicator IN indicators)

// 校验权重总和
IF total_weight != 100 THEN
    RETURN error("权重总和必须等于100%，当前总和为" + total_weight + "%")
END IF

// 单个权重范围校验
FOR each indicator IN indicators DO
    IF indicator.weight < 0 OR indicator.weight > 100 THEN
        RETURN error("指标权重必须在0-100之间")
    END IF
END FOR

RETURN success()
``` |
| 失败处理 | 返回错误"权重总和必须等于100%"，阻止保存操作 |
| 优先级 | P0 |

#### BR-OPS-004: 绩效指标目标值校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-004 |
| 规则名称 | 绩效指标目标值校验 |
| 规则类型 | VAL (校验规则) |
| 适用对象 | 绩效指标配置 |
| 规则描述 | 绩效指标目标值必须合理，不能为负数或超出合理范围 |
| 触发时机 | 保存绩效指标配置时 |
| 校验逻辑 | ```
// 伪代码
indicator = CURRENT_INDICATOR

// 基本校验
IF indicator.target_value IS NULL THEN
    RETURN error("目标值不能为空")
END IF

IF indicator.target_value < 0 THEN
    RETURN error("目标值不能为负数")
END IF

// 指标类型校验
CASE indicator.indicator_type
    WHEN 'WORKLOAD' THEN
        // 工作量指标，目标值应为正整数
        IF indicator.target_value != FLOOR(indicator.target_value) THEN
            RETURN error("工作量指标目标值必须为整数")
        END IF
    WHEN 'QUALITY' THEN
        // 质量指标，目标值应为百分比
        IF indicator.target_value > 100 THEN
            RETURN error("质量指标目标值不能超过100%")
        END IF
    WHEN 'SERVICE' THEN
        // 服务指标，满意度目标值应为百分比
        IF indicator.target_value > 100 THEN
            RETURN error("满意度目标值不能超过100%")
        END IF
END CASE

RETURN success()
``` |
| 失败处理 | 返回具体校验失败原因，阻止保存操作 |
| 优先级 | P0 |

missing_data = []

    missing_data.add('门诊数据')
END IF

IF inpatient_data IS NULL OR COUNT(inpatient_data) = 0 THEN
    missing_data.add('住院数据')
END IF

IF finance_data IS NULL OR COUNT(finance_data) = 0 THEN
    missing_data.add('财务数据')

    ELSE
        // 全部缺失阻止生成

    END IF

    END IF

### 3.2 流转规则 (FLOW)

#### BR-OPS-007: 绩效考核流程状态流转

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-007 |
| 规则名称 | 绩效考核流程状态流转 |
| 规则类型 | FLOW (流转规则) |
| 适用对象 | 绩效考核周期(perf_cycle) |
| 规则描述 | 绩效考核周期状态流转：初始化→数据采集→计算中→待审核→已发布→已归档 |
| 触发时机 | 绩效考核流程操作时 |
| 校验逻辑 | ```
// 伪代码
// 状态定义：1-初始化, 2-数据采集, 3-计算中, 4-待审核, 5-已发布, 6-已归档

valid_transitions = {
    1: [2],        // 初始化 -> 数据采集
    2: [3],        // 数据采集 -> 计算中
    3: [4],        // 计算中 -> 待审核
    4: [5, 3],     // 待审核 -> 已发布 或 退回计算中
    5: [6],        // 已发布 -> 已归档
    6: []          // 已归档 -> 终态
}

IF target_status NOT IN valid_transitions[current_status] THEN
    RETURN error("状态流转不合法：当前状态[" + current_status + "]不可变更为[" + target_status + "]")
END IF

// 状态变更操作
CASE target_status
    WHEN 2 THEN  // 数据采集
        START_DATA_COLLECTION()
    WHEN 3 THEN  // 计算中
        START_PERFORMANCE_CALCULATION()
    WHEN 4 THEN  // 待审核
        SEND_NOTIFICATION_TO_APPROVERS()
    WHEN 5 THEN  // 已发布
        SEND_NOTIFICATION_TO_STAFF()
        OPEN_APPEAL_CHANNEL()
    WHEN 6 THEN  // 已归档
        ARCHIVE_PERFORMANCE_DATA()
END CASE

RETURN success()
``` |
| 失败处理 | 返回错误"状态流转不合法" |
| 优先级 | P0 |

#### BR-OPS-008: 绩效申诉流程

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-008 |
| 规则名称 | 绩效申诉流程 |
| 规则类型 | FLOW (流转规则) |
| 适用对象 | 绩效申诉(perf_appeal) |
| 规则描述 | 绩效申诉流程：申诉提交→待审核→审核通过/审核驳回→重新计算(如通过) |
| 触发时机 | 绩效申诉操作时 |
| 校验逻辑 | ```
// 伪代码
// 申诉状态：1-待审核, 2-审核通过, 3-审核驳回

// 申诉提交条件校验
IF perf_cycle.status != '已发布' THEN
    RETURN error("绩效结果未发布，不可申诉")
END IF

// 申诉时间窗口
appeal_deadline = perf_cycle.publish_time + 7 DAYS
IF CURRENT_DATE > appeal_deadline THEN
    RETURN error("已过申诉截止时间")
END IF

// 创建申诉
appeal = CREATE_appeal({
    'user_id': current_user.user_id,
    'cycle_id': perf_cycle.cycle_id,
    'appeal_reason': appeal_reason,
    'status': 1  // 待审核
})

// 发送通知给审核人
SEND_NOTIFICATION_TO_APPROVERS("有绩效申诉待审核")

RETURN success(appeal)

// 申诉审核处理
IF appeal_decision = 'APPROVE' THEN
    appeal.status = 2  // 审核通过
    // 触发重新计算
    TRIGGER_RECALCULATION(appeal.user_id)
ELSE
    appeal.status = 3  // 审核驳回
END IF
``` |
| 失败处理 | 返回具体错误原因 |
| 优先级 | P0 |

#### BR-OPS-019: 绩效得分计算

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-019 |
| 规则名称 | 绩效得分计算 |
| 规则类型 | CALC (计算规则) |
| 适用对象 | 绩效评分(performance_score) |
| 规则描述 | 绩效总分 = Σ(指标得分 × 指标权重)，指标得分根据实际值与目标值对比计算 |
| 触发时机 | 绩效计算执行时 |
| 校验逻辑 | ```
// 伪代码
user_id = TARGET_USER.user_id
period = CURRENT_PERF_PERIOD

total_score = 0
indicator_scores = []

// 获取用户的绩效指标配置
indicators = SELECT * FROM perf_indicator 
             WHERE scheme_id = period.scheme_id 
               AND (dept_id = user.dept_id OR dept_id IS NULL)

FOR each indicator IN indicators DO
    // 获取指标原始数据值
    actual_value = GET_INDICATOR_DATA(indicator.code, user_id, period)
    
    // 计算指标得分
    target_value = indicator.target_value
    full_score = 100  // 单项满分
    
    IF actual_value >= target_value THEN
        // 达到目标值
        IF indicator.has_bonus THEN
            // 超额奖励
            exceed_ratio = (actual_value - target_value) / target_value
            bonus_score = MIN(exceed_ratio * indicator.bonus_ratio * full_score, indicator.max_bonus)
            indicator_score = full_score + bonus_score
        ELSE
            indicator_score = full_score
        END IF
    ELSE
        // 未达目标值
        completion_ratio = actual_value / target_value
        indicator_score = full_score * completion_ratio * indicator.completion_factor
    END IF
    
    // 负向指标处理
    IF indicator.is_negative THEN
        IF actual_value > target_value THEN
            indicator_score = full_score * (1 - (actual_value - target_value) / target_value)
        ELSE
            indicator_score = full_score
        END IF
    END IF
    
    indicator_score = MAX(0, indicator_score)  // 得分不能为负
    
    // 加权计算
    weighted_score = indicator_score * (indicator.weight / 100)
    total_score = total_score + weighted_score
    
    indicator_scores.add({
        'indicator_name': indicator.name,
        'actual_value': actual_value,
        'target_value': target_value,
        'indicator_score': indicator_score,
        'weight': indicator.weight,
        'weighted_score': weighted_score
    })
END FOR

total_score = ROUND(total_score, 1)

RETURN {
    'total_score': total_score,
    'indicator_scores': indicator_scores
}
``` |
| 失败处理 | - |
| 优先级 | P0 |

// 伪代码
sync_mode = CONFIG.sync_mode  // 'REALTIME' or 'SCHEDULED'

IF sync_mode = 'REALTIME' THEN
    // 实时同步：监听消息队列
    MESSAGE_QUEUE.subscribe('OUTPATIENT_EVENTS', (event) => {
        SWITCH event.type
            CASE 'REGISTER':
                SYNC_REGISTER_DATA(event.data)
            CASE 'VISIT':
                SYNC_VISIT_DATA(event.data)
            CASE 'PAYMENT':
                SYNC_PAYMENT_DATA(event.data)
        END SWITCH
        
        // 更新看板缓存
        REFRESH_DASHBOARD_CACHE()
    })
ELSE
    // 定时同步：每小时执行
    SCHEDULE_TASK('0 * * * *', () => {
        last_sync_time = GET_LAST_SYNC_TIME('OUTPATIENT')
        
        // 批量同步增量数据
        new_registers = SELECT * FROM op_register WHERE create_time > last_sync_time
        new_visits = SELECT * FROM op_visit WHERE create_time > last_sync_time
        
        INSERT INTO ops_register_stats SELECT ... FROM new_registers
        INSERT INTO ops_visit_stats SELECT ... FROM new_visits
        
        UPDATE_SYNC_TIME('OUTPATIENT', NOW())
    })
END IF

RETURN success()

// 伪代码
// 实时同步住院事件
MESSAGE_QUEUE.subscribe('INPATIENT_EVENTS', (event) => {
    SWITCH event.type
        CASE 'ADMISSION':
            // 入院事件
            UPDATE_INPATIENT_COUNT(+1)
            UPDATE_BED_STATUS(event.data.bed_id, 'OCCUPIED')
            SYNC_ADMISSION_DATA(event.data)
            
        CASE 'DISCHARGE':
            // 出院事件
            UPDATE_INPATIENT_COUNT(-1)
            UPDATE_BED_STATUS(event.data.bed_id, 'OPEN')

            UPDATE_BED_STATUS(event.data.bed_id, event.data.new_status)
    END SWITCH
    
    // 更新看板

})

#### BR-OPS-027: 财务数据同步

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-OPS-027 |
| 规则名称 | 财务数据同步 |
| 规则类型 | INT (集成规则) |
| 适用对象 | 收入统计、经营分析 |
| 规则描述 | 从财务管理系统同步收入、成本数据，采用定时批量同步模式 |
| 触发时机 | 定时任务（每日凌晨） |
| 校验逻辑 | ```
// 伪代码
// 每日凌晨2点执行
SCHEDULE_TASK('0 2 * * *', () => {
    yesterday = CURRENT_DATE - 1 DAY
    
    // 同步收入数据
    income_data = CALL_FINANCE_API('getIncomeByDate', { date: yesterday })
    FOR each item IN income_data DO
        INSERT INTO ops_income_stats (
            stat_date, dept_id, income_type, amount
        ) VALUES (
            item.date, item.dept_id, item.income_type, item.amount
        )
    END FOR
    
    // 同步成本数据
    cost_data = CALL_FINANCE_API('getCostByDate', { date: yesterday })
    FOR each item IN cost_data DO
        INSERT INTO ops_cost_stats (
            stat_date, dept_id, cost_type, amount
        ) VALUES (
            item.date, item.dept_id, item.cost_type, item.amount
        )
    END FOR
    
    // 数据校验
    total_income = SUM(item.amount FOR item IN income_data)
    total_cost = SUM(item.amount FOR item IN cost_data)
    
    LOG sync_result = {
        'sync_date': yesterday,
        'income_records': COUNT(income_data),
        'cost_records': COUNT(cost_data),
        'total_income': total_income,
        'total_cost': total_cost
    }
    
    // 发送同步结果通知
    SEND_SYNC_NOTIFICATION(sync_result)
})
``` |
| 失败处理 | 失败后重试3次，记录错误日志 |
