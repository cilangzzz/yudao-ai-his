## 1. 功能概述

### 1.1 功能定位

EMPI（Enterprise Master Patient Index）患者主索引管理模块是集成平台的核心子模块，为全院各业务系统提供统一的患者唯一标识服务。系统实现患者主索引创建、重复患者检测、患者信息合并、EMPI查询服务、主索引同步等核心功能，遵循HL7 FHIR R4标准，确保跨系统患者信息的准确匹配和数据融合。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 数据一致目标 | 患者主索引准确匹配 | EMPI匹配准确率>=99% |
| 互操作性目标 | 支持标准数据交换 | FHIR接口标准化率>=95% |
| 效率目标 | 查询响应实时快速 | EMPI查询响应<=200ms |
| 合规目标 | 敏感信息安全存储 | 身份证/手机号加密存储 |

### 1.3 功能范围

```
M10-01 EMPI管理
├── EMPI主索引创建
│   ├── 患者信息录入（姓名、性别、出生日期）
│   ├── EMPI编号生成（E+年月日+流水号）
│   ├── 敏感信息加密（身份证、手机号AES加密）
│   ├── 关联系统记录（来源系统、本地ID）
│   └── 自动匹配检测（创建前检测重复）
├── 重复患者检测
│   ├── 身份证号精确匹配（置信度=1.00）
│   ├── 姓名出生日期匹配（置信度>=0.85）
│   ├── 手机号辅助匹配（置信度>=0.70）
│   ├── 多维度综合匹配（加权计算置信度）
│   ├── 匹配阈值配置（精确/高度/可能匹配阈值）
│   └── 候选结果排序（按置信度降序）
├── 患者信息合并
│   ├── 合并预览功能（信息对比显示）
│   ├── 信息冲突处理（继承非空/较新值）
│   ├── 关联系统迁移（合并后关联转移）
│   ├── 合并历史记录（保留合并操作记录）
│   ├── 合并撤销功能（24小时内可撤销）
│   └── 合并审批流程（有活跃就诊时需审批）
├── EMPI查询服务
│   ├── EMPI编号查询（GET /api/empi/{id}）
│   ├── 身份证号查询（按身份证查询）
│   ├── 批量查询服务（支持100个编号批量查询）
│   ├── FHIR Patient资源（符合FHIR R4规范）
│   ├── 关联系统列表查询（查询EMPI关联系统）
│   └── 查询频率限制（同一IP限100次/分钟）
└── 主索引同步
    ├── EMPI信息分发（变更推送订阅系统）
    ├── 外部系统同步（区域平台、医保等）
    ├── 同步状态追踪（记录同步成功/失败）
    └── 同步失败重试（自动重试机制）
```

## 4. 字段定义

### 4.1 EMPI主索引表 (empi)

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| empi_id | BIGINT | 是 | EMPI主键ID（系统生成） |
| empi_no | VARCHAR(30) | 是 | EMPI编号（对外唯一标识），格式：E+年月日+4位流水号 |
| patient_name | VARCHAR(50) | 是 | 患者姓名 |
| gender | TINYINT | 是 | 性别：1男/2女/9未知 |
| birth_date | DATE | 是 | 出生日期 |
| id_card_no | VARCHAR(100) | 否 | 身份证号（AES-256加密存储） |
| phone | VARCHAR(100) | 否 | 手机号（AES-256加密存储） |
| address | VARCHAR(200) | 否 | 地址 |
| match_score | DECIMAL(5,2) | 否 | 匹配置信度分数 |
| empi_status | TINYINT | 是 | 状态：1有效/2已合并/3已注销 |
| merged_to | BIGINT | 否 | 合并目标EMPI_ID（已合并时指向目标） |
| merge_count | INT | 否 | 累计被合并次数（上限2次） |
| create_time | DATETIME | 是 | 创建时间 |
| update_time | DATETIME | 否 | 更新时间 |
| create_by | VARCHAR(50) | 是 | 创建人 |
| update_by | VARCHAR(50) | 否 | 更新人 |

  "birthDate": "1990-01-15",

## 6. 交互流程

### 6.1 EMPI创建流程

```
流程步骤：

1. 业务系统接收患者信息
   ↓
2. 调用EMPI匹配检测接口 (POST /api/empi/match)
   ↓
3. 判断匹配结果
   ├── 精确匹配（置信度=1.00）
   │   ↓
   │   返回已有EMPI编号，建议关联
   │   ↓
   │   业务系统选择关联已有EMPI
   │
   ├── 高度匹配（置信度>=0.85）
   │   ↓
   │   返回候选列表，需人工确认
   │   ↓
   │   数据管理员确认后关联或创建新EMPI
   │
   ├── 可能匹配（置信度>=0.70）
   │   ↓
   │   返回候选列表，供参考
   │   ↓
   │   数据管理员判断后决定
   │
   └── 无匹配（置信度<0.70）
       ↓
       调用EMPI创建接口 (POST /api/empi)
       ↓
       系统生成唯一EMPI编号
       ↓
       加密存储敏感信息
       ↓
       记录关联系统关系
       ↓
       返回创建成功结果
       ↓
       分发EMPI信息到订阅系统
```

### 6.2 患者合并流程

```
流程步骤：

1. 数据管理员选择两个患者记录
   ↓
2. 点击"合并预览"按钮
   ↓
3. 系统显示合并预览页面
   ├── 显示两个患者信息对比
   ├── 显示合并后预览结果
   ├── 高亮显示冲突字段
   └── 提供字段选择选项
   ↓
4. 数据管理员选择合并目标
   ↓
5. 系统执行合并前校验
   ├── 校验目标EMPI状态是否为有效
   ├── 校验合并次数是否超过上限
   ├── 检查是否有活跃就诊记录
   │   ├── 有活跃就诊
   │   │   ↓
   │   │   创建合并审批工单
   │   │   ↓
   │   │   通知审批人
   │   │   ↓
   │   │   等待审批结果
   │   │   ↓
   │   │   审批通过后继续合并
   │   └── 无活跃就诊
   │       ↓
   │       直接继续合并
   ↓
6. 执行合并操作
   ├── 源EMPI状态变更为"已合并"
   ├── merged_to指向目标EMPI
   ├── 目标EMPI继承关联系统
   ├── 处理信息冲突（继承非空/较新值）
   ├── 记录合并历史
   └── 记录操作日志
   ↓
7. 返回合并成功结果
   ↓
8. 分发合并通知到订阅系统
```

### 6.3 合并撤销流程

```
流程步骤：

1. 数据管理员查看合并历史
   ↓
2. 选择需要撤销的合并记录
   ↓
3. 系统校验撤销时限（24小时）
   ├── 超过时限
   │   ↓
   │   提示"撤销时限已过，需联系管理员"
   │
   └── 未超过时限
       ↓
       执行撤销操作
       ├── 源EMPI状态恢复为"有效"
       ├── 清空merged_to字段
       ├── 恢复原有关联关系
       ├── 记录撤销操作日志
       └── 更新合并记录状态
       ↓
       返回撤销成功结果
       ↓
       分发撤销通知到订阅系统
```

### 8.2 关键业务规则详解

#### BR-INT-CALC-001: EMPI编号生成算法

EMPI编号格式为E+年月日+4位流水号，当日超过9999则扩展为5位。

**触发时机**: EMPI创建时

**计算逻辑**:
- date_part = 当前日期（YYYYMMDD格式）
- sequence = 获取当日流水号序列
- 若 sequence <= 9999：sequence_part = 4位流水号（如0101）
- 若 sequence > 9999：sequence_part = 5位流水号（如10001）
- empi_no = 'E' + date_part + sequence_part

**示例**: E202606190001（2026年6月19日第1个EMPI）

#### BR-INT-CALC-002: 匹配置信度计算算法

综合置信度=姓名权重*姓名相似度+出生日期权重*出生日期匹配+手机权重*手机匹配+地址权重*地址相似度

**权重配置**:
- 身份证号精确匹配：置信度直接为1.00
- 姓名：权重0.4
- 出生日期：权重0.3
- 手机号：权重0.2
- 地址：权重0.1

**阈值判定**:
- 置信度=1.00：精确匹配（EXACT）
- 置信度>=0.85：高度匹配（MATCHED）
- 置信度>=0.70：可能匹配（POSSIBLE）
- 置信度<0.70：无匹配（NO_MATCH）

#### BR-INT-INT-001: EMPI敏感信息加密规则

身份证号、手机号等敏感信息使用AES-256加密存储。

**触发时机**: EMPI数据保存时

**加密逻辑**:
- 身份证号：AES_ENCRYPT(id_card_no, encryption_key)
- 手机号：AES_ENCRYPT(phone, encryption_key)
- 加密后存储，明文清空
- 查询时解密显示

#### BR-INT-FLOW-002: 合并撤销时限规则

合并操作完成后24小时内可撤销，超过时限需联系管理员。

**触发时机**: 合并撤销时

**校验逻辑**:
- 获取合并时间
- 计算当前时间与合并时间的差值（小时）
- 若差值>24小时：返回错误"撤销时限已过(24小时)，需联系管理员"
- 若差值<=24小时：允许撤销操作
