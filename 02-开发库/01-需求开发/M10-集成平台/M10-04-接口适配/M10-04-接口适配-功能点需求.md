## 1. 功能概述

### 1.1 功能定位

接口适配模块是集成平台子系统的外部对接门户，提供外部系统对接的适配器管理能力。系统实现接口配置管理、接口日志记录、接口监控告警等功能，支持多种协议和格式的接口适配，为区域平台、医保系统、银行支付等外部系统的对接提供统一管理入口。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 互操作性目标 | 实现多协议接口适配 | 支持≥4种协议类型 |
| 可靠性目标 | 接口调用可追溯 | 日志完整率≥99% |
| 运维目标 | 异常及时发现处理 | 告警响应时间≤5分钟 |
| 可用性目标 | 接口高可用运行 | 成功率≥95% |

### 1.3 功能范围

```
M10-04 接口适配
├── 接口配置管理
│   ├── 外部系统管理（区域平台、医保、银行等）
│   ├── 接口类型配置（REST/SOAP/HL7/DICOM）
│   ├── 请求模板配置（支持变量替换）
│   ├── 响应映射配置（字段映射规则）
│   ├── 接口测试连接
│   └── 接口启用/禁用管理
├── 接口日志管理
│   ├── 调用日志记录（请求/响应/耗时）
│   ├── 日志查询（按接口/时间/状态）
│   ├── 日志详情查看（完整请求响应）
│   ├── 日志导出（CSV格式）
│   └── 日志自动清理（保留90天）
├── 接口监控告警
│   ├── 实时监控面板（状态/调用量/成功率）
│   ├── 性能指标监控（响应时间阈值）
│   ├── 可用性监控（成功率阈值）
│   ├── 异常告警（错误次数阈值）
│   ├── 告警通知渠道（邮件/短信/企业微信）
│   └── 告警风暴抑制
└── 外部系统对接
    ├── 区域平台对接（HL7 CDA）
    ├── 医保系统对接（医保接口）
    ├── 银行支付对接（银行接口）
    └── 自助设备对接（设备接口）
```

## 6. 状态机设计

### 6.1 接口配置状态机

```
接口状态定义：
- 启用（1）：接口可正常调用
- 禁用（2）：接口不可调用

状态流转图：

        ┌─────────────────────────────────────┐
        │                                      │
        ▼                                      │
    ┌────────┐                           ┌────────┐
    │  启用  │ ◄─────────────────────── │  禁用  │
    │ (1)   │                           │ (2)   │
    └────────┘ ────────────────────────► └────────┘
        │                                      ▲
        │                                      │
        └──────────────────────────────────────┘
              管理员点击"启用"/"禁用"

状态流转规则：
- 启用 <-> 禁用：双向流转，需管理员权限
- 禁用状态下接口不可被调用
- 启用前可检查是否有正在进行的调用
```

### 6.2 告警状态机

```
告警状态定义：
- 未处理（1）：告警待处理
- 已处理（2）：告警已处理
- 已忽略（3）：告警已忽略

状态流转图：

           创建告警
              │
              ▼
         ┌────────┐
         │ 未处理 │ ◄───────┐
         │  (1)   │         │
         └────────┘         │
              │              │
    ┌─────────┼─────────┐    │
    ▼         ▼         │    │
┌────────┐ ┌────────┐   │    │
│ 已处理 │ │ 已忽略 │   │    │
│  (2)   │ │  (3)   │   │    │
└────────┘ └────────┘   │    │
                         │    │
                         └────┘
                     相同告警合并

状态流转规则：
- 未处理 -> 已处理：运维人员处理告警
- 未处理 -> 已忽略：运维人员忽略告警
- 未处理状态下可合并相同告警
```

### 7.2 关键业务规则详解

#### BR-INT-VAL-009: 接口URL格式校验

接口URL必须为有效的HTTP/HTTPS地址，格式校验包括协议、域名、端口、路径等。

**触发时机**: 接口配置保存时

**校验逻辑**:
- URL必须以http://或https://开头
- URL格式必须符合标准URL规范
- 建议使用HTTPS协议保证传输安全

#### BR-INT-VAL-010: 接口编码唯一性校验

接口编码必须全局唯一，格式为IF-XXX-XXX（X为数字）。

**触发时机**: 接口配置保存时

**校验逻辑**:
- 接口编码格式必须为IF-XXX-XXX
- 同一接口编码不可重复
- 编码前三位为系统编码，后三位为序号

#### BR-INT-INT-005: 接口超时配置规则

接口超时时间默认30秒，最大不超过120秒。

**触发时机**: 接口调用时

**配置规则**:
- 连接超时为总超时的1/3
- 读取超时等于总超时
- 写入超时为总超时的1/2

#### BR-INT-INT-006: 日志保留期限规则

接口调用日志保留90天，监控原始数据保留30天后聚合。

**触发时机**: 定期清理任务

**保留规则**:
- 接口调用日志保留90天
- 死信队列日志保留30天
- 监控原始数据保留30天后聚合为小时级数据
- 聚合数据保留90天
