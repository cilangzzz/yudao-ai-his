### 3.1 校验规则 (VAL)

#### BR-RIS-001: 影像申请接收校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-001 |
| 规则名称 | 影像申请接收校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 影像申请(imaging_request) |
| 规则描述 | 影像申请接收时必须校验申请来源有效性：门诊申请需已缴费，住院申请需医嘱已确认 |
| 触发时机 | 申请接收时 |
| 校验逻辑 | ```
// 伪代码
IF request.source = 'OUTPATIENT' THEN
    // 门诊申请：检查缴费状态
    IF request.payment_status != 'PAID' THEN
        RETURN error("患者未缴费，请引导患者先缴费")
    END IF
ELSE IF request.source = 'INPATIENT' THEN
    // 住院申请：检查医嘱执行状态
    order = SELECT * FROM ip_order WHERE order_id = request.order_id
    IF order.order_status NOT IN ('CONFIRMED', 'EXECUTING') THEN
        RETURN error("医嘱未确认，暂不可接收")
    END IF
END IF

// 检查申请信息完整性
IF request.exam_type_code IS NULL THEN
    RETURN error("检查类型不能为空")
END IF
IF request.exam_part IS NULL THEN
    RETURN error("检查部位不能为空")
END IF
IF request.patient_id IS NULL THEN
    RETURN error("患者信息不能为空")
END IF

RETURN success()
``` |
| 失败处理 | 返回具体校验失败原因，阻止申请接收 |
| 优先级 | P0 |

#### BR-RIS-002: 检查排期冲突校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-002 |
| 规则名称 | 检查排期时段冲突校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 检查排期(imaging_schedule) |
| 规则描述 | 检查排期时必须校验时段是否已被占用，同一设备同一时间段只能安排一次检查 |
| 触发时机 | 排期创建/修改时 |
| 校验逻辑 | ```
// 伪代码
// 检查时段冲突
conflict_count = SELECT count(*) 
                 FROM imaging_schedule 
                 WHERE room_id = :room_id 
                   AND schedule_date = :schedule_date
                   AND time_slot OVERLAPS (:start_time, :end_time)
                   AND schedule_status NOT IN ('CANCELLED', 'COMPLETED')

IF conflict_count > 0 THEN
    // 查询可用时段
    available_slots = SELECT * FROM imaging_time_slot 
                      WHERE room_id = :room_id 
                        AND schedule_date = :schedule_date
                        AND slot_status = 'AVAILABLE'
    RETURN error("该时段已被占用", available_slots)
END IF

// 检查设备可用性
equipment = SELECT * FROM imaging_equipment WHERE equipment_id = :equipment_id
IF equipment.equipment_status != 'NORMAL' THEN
    RETURN error("检查设备状态异常：" + equipment.equipment_status)
END IF

RETURN success()
``` |
| 失败处理 | 返回错误"该时段已被占用"，显示其他可用时段 |
| 优先级 | P0 |

#### BR-RIS-006: DICOM标准合规校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-006 |
| 规则名称 | DICOM影像标准合规校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | DICOM影像(dicom_image) |
| 规则描述 | 影像上传时必须校验DICOM 3.0标准合规性，校验DICOM Tag完整性 |
| 触发时机 | 影像上传时 |
| 校验逻辑 | ```
// 伪代码
// DICOM Tag完整性校验
required_tags = [
    (0010,0020),  // Patient ID
    (0010,0010),  // Patient Name
    (0010,0030),  // Patient Birth Date
    (0010,0040),  // Patient Sex
    (0020,000D),  // Study Instance UID
    (0008,0060),  // Modality
    (0008,0020),  // Study Date
    (0008,0030)   // Study Time
]

FOR each tag IN required_tags DO
    IF NOT dicom_file.has_tag(tag) THEN
        RETURN error("DICOM文件缺少必要Tag：" + tag)
    END IF
END FOR

// UID唯一性校验
study_uid = dicom_file.get_tag(0020,000D)
existing_uid = SELECT * FROM imaging_study WHERE study_instance_uid = :study_uid
IF existing_uid EXISTS THEN
    RETURN error("Study Instance UID已存在，影像重复")
END IF

// 文件完整性校验
IF dicom_file.size = 0 OR dicom_file.corrupted THEN
    RETURN error("DICOM文件损坏或不完整")
END IF

RETURN success()
``` |
| 失败处理 | 返回具体校验失败原因，提示重新采集 |
| 优先级 | P0 |

#### BR-RIS-008: 影像质量校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-008 |
| 规则名称 | 影像质量合格校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | DICOM影像(dicom_image) |
| 规则描述 | 影像上传后自动进行质量检查，检查影像清晰度、伪影、运动模糊等质量问题 |
| 触发时机 | 影像上传后 |
| 校验逻辑 | ```
// 伪代码
quality_score = 0

// 分辨率检查
IF dicom_image.resolution >= MIN_RESOLUTION THEN
    quality_score = quality_score + 25
ELSE
    quality_issues.add("分辨率不足")
END IF

// 信噪比检查
IF dicom_image.signal_noise_ratio >= MIN_SNR THEN
    quality_score = quality_score + 25
ELSE
    quality_issues.add("信噪比偏低")
END IF

// 运动模糊检查
IF dicom_image.motion_artifact_level < MAX_MOTION_ARTIFACT THEN
    quality_score = quality_score + 25
ELSE
    quality_issues.add("存在运动伪影")
END IF

// 对比度检查
IF dicom_image.contrast_ratio >= MIN_CONTRAST THEN
    quality_score = quality_score + 25
ELSE
    quality_issues.add("对比度不足")
END IF

IF quality_score < 75 THEN
    RETURN warning("影像质量不合格：" + quality_issues, require_recheck=true)
END IF

RETURN success()
``` |
| 失败处理 | 返回警告"影像质量不合格"，显示具体问题，可标记需重新检查 |
| 优先级 | P0 |

#### BR-RIS-016: 影像申请状态流转

| 属性 | 内容 |

| 规则编号 | BR-RIS-016 |
| 规则名称 | 影像申请状态流转规则 |

| 适用对象 | 影像申请(imaging_request) |
| 规则描述 | 申请状态流转：待缴费→待排期→已排期→已到检→检查中→已完成→已取消；状态流转有顺序约束 |

| 适用对象 | 影像存储(imaging_storage) |
| 规则描述 | 影像在线存储3个月后归档到近线存储，近线存储1年后迁移到离线存储 |
| 触发时机 | 定时归档任务 |
| 校验逻辑 | ```
// 伪代码
// 存储保留期限配置
retention_policy = {
    'XR': {online: 3_MONTHS, nearline: 1_YEAR, offline: PERMANENT},
    'CT': {online: 3_MONTHS, nearline: 2_YEAR, offline: PERMANENT},
    'MRI': {online: 3_MONTHS, nearline: 2_YEAR, offline: PERMANENT},
    'US': {online: 3_MONTHS, nearline: 1_YEAR, offline: PERMANENT},
    'NM': {online: 3_MONTHS, nearline: 2_YEAR, offline: PERMANENT}
}

// 每日执行归档任务
FOR each study IN SELECT * FROM imaging_study 
                  WHERE storage_location != 'OFFLINE' DO
    
    policy = retention_policy[study.modality]
    age_months = MONTHS(CURRENT_DATE - study.study_date)
    
    IF study.storage_location = 'ONLINE' AND age_months >= policy.online THEN
        // 迁移到近线存储
        CALL storage_migration(study, 'ONLINE', 'NEARLINE')
    ELSE IF study.storage_location = 'NEARLINE' AND age_months >= policy.nearline THEN
        // 迁移到离线存储
        CALL storage_migration(study, 'NEARLINE', 'OFFLINE')
    END IF
END FOR
``` |
| 失败处理 | 迁移失败时记录日志，下次任务自动重试 |
| 优先级 | P0 |

#### BR-RIS-045: 存储容量预警时间

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-045 |
| 规则名称 | 存储容量预警规则 |
| 规则类型 | TIME(时间规则) |
| 适用对象 | 影像存储(imaging_storage) |
| 规则描述 | 存储容量使用率超过80%时发送预警，超过95%时阻止上传并告警 |
| 触发时机 | 实时监控 |
| 校验逻辑 | ```
// 伪代码
// 实时监控存储使用率
storage = SELECT * FROM imaging_storage WHERE storage_type = 'ONLINE'

usage_ratio = storage.used_size / storage.total_size

IF usage_ratio >= 0.95 THEN
    // 严重告警
    SEND_alert({
        'level': 'CRITICAL',
        'recipient': 'ADMIN',
        'message': '在线存储空间严重不足，使用率95%+'
    })
    // 阻止新影像上传
    SET upload_blocked = true
    
ELSE IF usage_ratio >= 0.90 THEN
    // 高级预警
    SEND_alert({
        'level': 'HIGH',
        'recipient': 'ADMIN',
        'message': '在线存储空间紧张，使用率90%+'
    })
    
ELSE IF usage_ratio >= 0.80 THEN
    // 中级预警
    SEND_alert({
        'level': 'MEDIUM',
        'recipient': 'ADMIN',
        'message': '在线存储空间预警，使用率80%+'
    })
END IF

RETURN usage_ratio
``` |
| 失败处理 | - |
| 优先级 | P0 |
