## 1. 影像申请管理验收标准

### 1.1 影像申请接收

#### Feature: 影像申请接收功能

```gherkin
Feature: 影像申请接收
  作为影像科技师
  我想要接收来自门诊和住院的影像检查申请
  以便于安排患者的检查时间

  # ========== 正常流程场景 ==========

  Scenario: 正常接收门诊申请
    Given 门诊医生已为患者"张三"开立"胸部CT"检查申请
    And 患者已完成缴费
    When 影像科技师查询待接收申请
    Then 系统显示该申请，状态为"待排期"
    And 系统显示患者基本信息、检查类型、检查部位
    And 系统显示临床诊断和检查目的

  Scenario: 正常接收住院申请
    Given 住院医生已为患者"李四"开立"腹部B超"检查申请
    And 医嘱已确认执行
    When 影像科技师查询待接收申请
    Then 系统显示该申请，状态为"待排期"
    And 申请关联住院信息

  Scenario: 申请查询过滤
    Given 影像科有多条检查申请
    When 技师选择查询条件：检查类型="CT"、日期="今日"
    Then 系统仅显示符合条件的CT申请
    And 申请按时间倒序排列

  # ========== 异常流程场景 ==========

  Scenario: 未缴费申请提示
    Given 门诊患者"王五"有检查申请
    And 患者尚未缴费
    When 技师查询该申请
    Then 系统显示申请状态为"待缴费"
    And 系统提示"患者未缴费，请引导患者先缴费"

  Scenario: 申请查询失败
    Given 技师正在查询申请列表
    When 查询接口发生异常
    Then 系统显示"查询失败，请重试"
    And 提供刷新按钮

  # ========== 边界条件场景 ==========

  Scenario: 申请日期范围查询
    Given 技师正在查询申请列表
    When 技师选择日期范围超过31天
    Then 系统提示"查询日期范围不能超过31天"

  Scenario: 申请列表为空
    Given 指定条件下无检查申请
    When 技师查询申请列表
    Then 系统显示"暂无符合条件的申请"
    And 列表为空
```

### 1.2 检查排期

#### Feature: 检查排期功能

```gherkin
Feature: 检查排期
  作为影像科技师
  我想要为患者安排检查时间
  以便于有序安排检查工作，减少等待

  # ========== 正常流程场景 ==========

  Scenario: 正常排期流程
    Given 患者申请状态为"待排期"
    And 今日CT室有空闲时段
    When 技师选择排期时间"14:00-14:30"
    And 技师选择检查房间"CT室1"
    And 技师确认排期
    Then 申请状态变更为"已排期"
    And 系统生成排期单
    And 系统记录排期时间、检查房间

  Scenario: 查看可用时段
    Given 技师正在为CT检查排期
    When 技师选择排期日期"2026-06-20"
    Then 系统显示该日各时段预约情况
    And 空闲时段标记为"可预约"
    And 已约满时段标记为"已满"

  Scenario: 急诊优先排期
    Given 患者申请紧急程度为"急诊"
    When 技师进行排期
    Then 系统优先安排最近的可用时段
    And 系统标记为"急诊优先"

  Scenario: 修改排期
    Given 患者已有排期记录
    And 患者尚未到检
    When 技师修改排期时间
    Then 系统更新排期信息
    And 系统记录修改记录

  # ========== 异常流程场景 ==========

  Scenario: 排期冲突检测
    Given 技师选择的时间段"14:00-14:30"已被占用
    When 技师尝试排期
    Then 系统提示"该时段已被占用"
    And 系统显示其他可用时段

  Scenario: 排期保存失败
    Given 技师正在保存排期信息
    When 保存接口发生异常
    Then 系统提示"排期保存失败，请重试"
    And 排期信息不丢失

  # ========== 边界条件场景 ==========

  Scenario: 排期时间限制
    Given 技师正在排期
    When 技师选择排期时间为"过去的时间"
    Then 系统提示"不能选择过去的时间"

  Scenario: 排期时段限制
    Given 技师正在排期
    When 技师选择的时段不在工作时间内
    Then 系统提示"该时段不在工作时间内"
```

## 2. 影像检查执行验收标准

### 2.1 检查登记

#### Feature: 检查登记功能

```gherkin
Feature: 检查登记
  作为影像科技师
  我想要为到检患者办理检查登记
  以便于开始影像检查工作

  # ========== 正常流程场景 ==========

  Scenario: 正常登记流程
    Given 患者已按排期到达影像科
    And 患者申请状态为"已排期"
    When 技师扫描申请单或输入申请编号
    And 技师确认患者身份信息
    And 技师选择检查设备
    Then 系统完成登记
    And 申请状态变更为"已到检"
    And 系统生成检查号

  Scenario: 患者身份核对
    Given 技师正在为患者登记
    When 技师扫描患者腕带或输入患者编号
    Then 系统显示患者基本信息
    And 技师核对姓名、年龄、检查类型

  Scenario: 检查设备选择
    Given 技师正在登记
    And 科室有多台同类型设备
    When 技师选择检查设备
    Then 系统显示设备状态（空闲/使用中）
    And 技师选择空闲设备

  # ========== 异常流程场景 ==========

  Scenario: 未缴费不可登记
    Given 患者申请状态为"待缴费"
    When 技师尝试登记
    Then 系统提示"患者未缴费，请先引导患者缴费"
    And 登记操作被阻止

  Scenario: 申请查询失败
    Given 技师输入申请编号
    When 查询不到对应申请
    Then 系统提示"未找到对应申请，请检查编号"

  # ========== 边界条件场景 ==========

  Scenario: 过期排期处理
    Given 患者排期时间为"昨日"
    And 患者未到检
    When 技师尝试登记
    Then 系统提示"排期已过期，需要重新排期"

  Scenario: 未到排期时间提示
    Given 患者排期时间为"14:00"
    And 当前时间为"10:00"
    When 技师尝试提前登记
    Then 系统提示"未到排期时间，是否提前检查？"
    And 技师确认后可继续
```

### 2.2 DICOM影像采集

#### Feature: DICOM影像采集功能

```gherkin
Feature: DICOM影像采集
  作为影像科技师
  我想要采集DICOM标准影像并自动上传
  以便于医生能够查看影像进行诊断

  # ========== 正常流程场景 ==========

  Scenario: 正常影像采集
    Given 患者已完成检查登记
    And 检查设备已就绪
    When 技师执行影像采集
    Then 设备生成DICOM影像
    And 影像自动上传至PACS服务器
    And 影像与检查记录关联

  Scenario: DICOM信息关联
    Given 技师正在采集影像
    When 影像生成时
    Then DICOM头信息自动填充患者信息
    And 填充检查类型、检查部位
    And 填充机构信息

  Scenario: 影像质量检查
    Given 影像已上传
    When 系统执行质量检查
    Then 系统校验DICOM标准合规性
    And 系统校验影像完整性
    And 质量合格时自动通过

  Scenario: 多序列影像采集
    Given 患者需要进行增强CT检查
    When 技师完成平扫和增强扫描
    Then 系统生成多个影像序列
    And 各序列独立存储
    And 统一关联到同一次检查

  # ========== 异常流程场景 ==========

  Scenario: 影像上传失败
    Given 影像已采集
    When PACS服务器不可用
    Then 系统本地暂存影像
    And 系统提示"影像上传失败，正在重试"
    And 网络恢复后自动上传

  Scenario: DICOM校验失败
    Given 影像已采集
    When DICOM标准校验失败
    Then 系统提示"影像格式不符合DICOM标准"
    And 技师可选择重新采集

  # ========== 边界条件场景 ==========

  Scenario: 影像大小限制
    Given 技师正在采集影像
    When 单次检查影像超过5GB
    Then 系统提示"影像数据量较大，确认是否继续"
```

## 3. 影像存储管理验收标准

### 3.1 影像存储管理

#### Feature: 影像存储管理功能

```gherkin
Feature: 影像存储管理
  作为系统管理员
  我想要管理影像存储并自动迁移归档
  以便于优化存储资源使用

  # ========== 正常流程场景 ==========

  Scenario: 影像在线存储
    Given 影像已成功上传
    When 系统存储影像
    Then 影像存储到在线存储设备
    And 系统记录存储位置
    And 影像可快速调阅

  Scenario: 自动归档迁移
    Given 影像存储已超过3个月
    When 系统执行定时归档任务
    Then 影像自动迁移到近线存储
    And 数据库更新存储位置标记
    And 释放在线存储空间

  Scenario: 归档影像调阅
    Given 影像已归档到近线存储
    When 医生请求调阅该影像
    Then 系统从近线存储读取影像
    And 调阅响应时间在5秒内

  Scenario: 存储容量监控
    Given 管理员查看存储状态
    When 系统计算存储使用率
    Then 系统显示各层级存储使用量
    And 使用率超过80%时发送告警

  Scenario: 影像备份
    Given 每日凌晨备份任务
    When 系统执行备份
    Then 增量备份当日新增影像
    And 备份数据写入备份介质
    And 系统记录备份日志

  # ========== 异常流程场景 ==========

  Scenario: 归档迁移失败
    Given 系统正在执行归档迁移
    When 近线存储设备不可用
    Then 系统记录迁移失败日志
    And 下次任务自动重试

  # ========== 边界条件场景 ==========

  Scenario: 存储空间不足
    Given 在线存储使用率超过95%
    When 新影像上传请求
    Then 系统提示"存储空间不足"
    And 系统通知管理员处理
```

## 4. 影像查看验收标准

### 4.1 影像调阅

#### Feature: 影像调阅功能

```gherkin
Feature: 影像调阅
  作为影像科医生
  我想要快速调阅患者的影像资料
  以便于进行影像诊断

  # ========== 正常流程场景 ==========

  Scenario: 正常调阅影像
    Given 医生选择某检查记录
    When 医生点击"查看影像"
    Then 系统加载影像查看器
    And 显示所有影像序列
    And 默认显示第一个序列的首张影像
    And 加载时间不超过3秒

  Scenario: 序列切换
    Given 影像已加载
    And 检查包含多个序列
    When 医生点击其他序列
    Then 系统切换显示对应序列影像
    And 保持当前窗宽窗位设置

  Scenario: 历史影像对比
    Given 患者有多次同类检查
    When 医生点击"历史对比"
    Then 系统显示历史检查列表
    And 医生选择历史检查后同时显示

  Scenario: 影像加载进度
    Given 影像数据量较大
    When 医生请求调阅
    Then 系统显示加载进度条
    And 优先加载首张影像供查看

  # ========== 异常流程场景 ==========

  Scenario: 影像不存在
    Given 医生请求调阅影像
    When 影像文件损坏或丢失
    Then 系统提示"影像不可用，请联系管理员"
    And 显示错误详情

  Scenario: 影像加载超时
    Given 医生请求调阅影像
    When 影像加载超过30秒
    Then 系统提示"加载超时，请重试"
    And 提供重新加载选项

  # ========== 边界条件场景 ==========

  Scenario: 并发调阅限制
    Given 系统并发调阅用户达到上限
    When 新用户请求调阅
    Then 系统提示"系统繁忙，请稍后再试"
```

```gherkin
Feature: 测量标注

    When 医生保存报告
    Then 标注信息随报告保存

    Given 医生正在进行距离测量

    Then 系统显示测量精度受限提示
```

## 5. 影像报告验收标准

```gherkin
Feature: 报告书写
  作为影像科医生
  我想要书写影像检查报告
  以便于记录影像诊断结果

  # ========== 正常流程场景 ==========

    Given 医生打开待报告检查
    When 医生填写影像所见
    And 医生填写诊断意见
    And 医生点击"保存"
    Then 系统保存报告草稿

  Scenario: 使用报告模板

    When 医生选择"胸部CT正常模板"
    Then 系统自动填充模板内容
    And 医生可修改模板内容

  Scenario: 引用历史报告
    Given 患者有历史检查报告
```

### 5.2 报告审核发布

#### Feature: 报告审核发布功能

```gherkin
Feature: 报告审核发布
  作为审核医师
  我想要审核并发布影像报告
  以便于确保报告质量后提供给临床

  # ========== 正常流程场景 ==========

  Scenario: 审核通过发布
    Given 报告医生已提交报告
    And 报告状态为"待审核"
    When 审核医师查看报告
    And 确认报告内容无误
    And 点击"审核通过"
    Then 报告状态变更为"已发布"
    And 系统记录审核人和审核时间
    And 系统通知临床医生

  Scenario: 审核退回修改
    Given 审核医师发现报告有误
    When 审核医师点击"退回修改"
    And 输入退回原因
    Then 报告状态变更为"草稿"
    And 系统通知报告医生
    And 报告医生收到退回原因

  Scenario: 双签审核
    Given 报告需要双签审核
    When 主审医师审核通过
    Then 系统等待副审医师签字
    And 双签完成后报告发布

  Scenario: 已发布报告修改
    Given 报告已发布
    When 发现报告有误需要修改
    Then 审核医师撤回报告
    And 报告状态变更为"已撤回"
    And 报告医生重新编辑

  # ========== 异常流程场景 ==========

  Scenario: 审核权限不足
    Given 用户无报告审核权限
    When 用户尝试审核报告
    Then 系统提示"无审核权限"
    And 操作被阻止

  # ========== 边界条件场景 ==========

  Scenario: 审核时限
    Given 报告提交时间已超过24小时
    When 系统检查待审核报告
    Then 系统发送催办通知给审核医师
```
