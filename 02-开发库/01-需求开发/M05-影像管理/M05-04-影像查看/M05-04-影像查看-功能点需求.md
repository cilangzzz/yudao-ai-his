## 1. 功能概述

### 1.1 功能定位

影像查看模块是影像管理子系统的核心诊断功能模块，为影像科医生和临床医生提供专业的DICOM影像查看器。系统遵循DICOM 3.0标准，支持窗宽窗位调整、测量标注、序列浏览、多序列对比等专业功能，满足影像诊断和临床诊断需求。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短影像调阅响应时间 | 在线影像调阅≤3秒，近线影像调阅≤5秒 |
| 质量目标 | 提供专业影像诊断工具 | 影像诊断工具完备率100% |
| 标准目标 | 遵循DICOM标准 | DICOM标准合规率100% |
| 便捷目标 | 支持历史影像对比 | 历史对比响应时间≤5秒 |

## 3. 用户故事

### US-M05-008 影像调阅

**故事卡片**
```
作为 影像科医生
我想要 快速调阅患者的影像资料
以便于 进行影像诊断
```

**优先级**: P0（MVP必需）

**故事点**: 8

**验收标准 (Gherkin)**
```gherkin
Feature: 影像调阅

  Scenario: 正常调阅影像
    Given 医生选择某检查记录
    When 医生点击"查看影像"
    Then 系统加载影像查看器
    And 显示所有影像序列
    And 默认显示第一个序列的首张影像
    And 加载时间不超过3秒

  Scenario: 序列切换
    Given 影像已加载
    And 检查包含多个序列
    When 医生点击其他序列
    Then 系统切换显示对应序列影像
    And 保持当前窗宽窗位设置

  Scenario: 历史影像对比
    Given 患者有多次同类检查
    When 医生点击"历史对比"
    Then 系统显示历史检查列表
    And 医生选择历史检查后同时显示

  Scenario: 影像加载进度
    Given 影像数据量较大
    When 医生请求调阅
    Then 系统显示加载进度条
    And 优先加载首张影像供查看

  Scenario Exception: 影像不存在
    Given 医生请求调阅影像
    When 影像文件损坏或丢失
    Then 系统提示"影像不可用，请联系管理员"
    And 显示错误详情

  Scenario Boundary: 并发调阅限制
    Given 系统并发调阅用户达到上限
    When 新用户请求调阅
    Then 系统提示"系统繁忙，请稍后再试"
```

| 规则名称 | 影像调阅并发限制校验 |

// 伪代码
concurrent_users = SELECT count(*) FROM imaging_view_session WHERE session_status = 'ACTIVE'

IF concurrent_users >= MAX_CONCURRENT_VIEWERS THEN
    RETURN error("系统繁忙，请稍后再试")
END IF

RETURN success()
``` |

#### BR-RIS-029: 影像调阅权限控制

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-029 |
| 规则名称 | 影像调阅权限控制 |
| 规则类型 | PERM(权限规则) |
| 适用对象 | 影像调阅(imaging_view) |
| 规则描述 | 影像调阅需要医生权限，患者只能通过患者门户查看自己的影像报告（非原始影像） |
| 触发时机 | 影像调阅请求时 |
| 校验逻辑 | ```
// 伪代码
IF viewer_type = 'PATIENT' THEN
    // 患者：只能查看自己的报告摘要
    IF patient.patient_id != current_patient.patient_id THEN
        RETURN error("只能查看本人的影像报告")
    END IF
    
    // 患者不可查看原始影像
    IF request.view_type = 'RAW_IMAGE' THEN
        RETURN error("患者不可查看原始影像，请查看报告")
    END IF
    
ELSE IF viewer_type = 'DOCTOR' THEN
    // 医生：可查看患者影像
    IF NOT has_permission(current_user, 'IMAGING_VIEW') THEN
        RETURN error("无影像调阅权限")
    END IF
    
    // 检查是否为患者主治医生
    IF NOT is_patient_doctor(current_user.user_id, patient.patient_id) THEN
        IF NOT has_permission(current_user, 'IMAGING_VIEW_ALL') THEN
            RETURN error("非患者主治医生，无调阅权限")
        END IF
    END IF
END IF

RETURN success()
``` |
| 失败处理 | 返回具体权限不足原因 |
| 优先级 | P0 |

#### BR-RIS-037: DICOM影像调阅接口集成

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-037 |
| 规则名称 | DICOM影像调阅接口集成 |
| 规则类型 | INT(集成规则) |
| 适用对象 | PACS服务器(pacs_server) |
| 规则描述 | 影像调阅通过DICOM C-FIND/C-MOVE接口，从PACS服务器获取影像数据 |
| 触发时机 | 影像调阅请求时 |
| 校验逻辑 | ```
// 伪代码
// DICOM C-FIND查询影像
study_uid = request.study_instance_uid

find_result = CALL dicom_c_find({
    'query': {
        (0020,000D): study_uid
    },
    'target': pacs_server
})

IF find_result.status = 'SUCCESS' THEN
    // DICOM C-MOVE获取影像
    move_result = CALL dicom_c_move({
        'study_uid': study_uid,
        'destination': viewer_client
    })
    
    IF move_result.status = 'SUCCESS' THEN
        RETURN success(images)
    ELSE
        RETURN error("影像获取失败：" + move_result.error_message)
    END IF
ELSE
    RETURN error("影像查询失败")
END IF
``` |
| 失败处理 | 获取失败时提示"影像不可用" |
| 优先级 | P0 |

## 10. 交互流程

### 10.1 影像调阅流程

```
步骤1: 医生选择检查记录
    │
    ↓
步骤2: 点击"查看影像"按钮
    │
    ↓
步骤3: 系统校验调阅权限
    │
    ├── 无权限 → 提示"无影像调阅权限"
    │
    ├── 并发限制 → 提示"系统繁忙，请稍后再试"
    │
    └── 有权限 → 继续
            │
            ↓
步骤4: 查询检查记录信息
    │
    ↓
步骤5: 调用DICOM接口获取影像
    │
    ├── 影像不存在 → 提示"影像不可用"
    │
    └── 影像存在 → 继续
            │
            ↓
步骤6: 加载影像序列列表
    │
    ↓
步骤7: 加载第一个序列首张影像
    │
    ↓
步骤8: 显示影像查看器界面
    │
    ↓
步骤9: 后台加载剩余影像
    │
    ↓
步骤10: 加载历史标注信息（如有）
```

### 10.2 测量标注流程

```
步骤1: 医生选择测量工具
    │
    ↓
步骤2: 在影像上点击起始点
    │
    ↓
步骤3: 拖动鼠标到终点
    │
    ↓
步骤4: 系统计算测量值
    │
    ├── 距离测量: 计算两点间距离
    ├── CT值测量: 获取像素CT值
    ├── 角度测量: 计算三点角度
    └── 面积测量: 计算封闭区域面积和CT值
    │
    ↓
步骤5: 在影像上绘制测量线和数值
    │
    ↓
步骤6: 保存测量记录（临时）
    │
    ↓
步骤7: 保存报告时持久化标注数据
```
