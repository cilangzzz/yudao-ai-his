## 1. 功能概述

### 1.1 功能定位

影像申请管理是影像管理子系统(RIS/PACS)的入口模块，负责接收来自门诊/住院的影像检查申请，支持检查排期、申请查询、状态追踪、申请取消/修改等核心功能。系统与门诊管理系统、住院管理系统无缝集成，实现影像检查申请的全流程管理。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短影像申请处理时间 | 申请接收至排期时间≤30分钟 |
| 流程目标 | 实现申请状态全程追踪 | 状态实时更新率100% |
| 管理目标 | 优化检查资源分配 | 排期冲突率≤5% |
| 体验目标 | 减少患者等待时间 | 急诊检查等待时间≤15分钟 |

## 3. 接口设计

### 3.1 影像申请接收接口

**接口说明**: 接收门诊或住院的影像检查申请

**接口路径**: `POST /api/ris/request`

**请求参数**:
```json
{
  "encounterId": 1001,
  "encounterType": 1,
  "patientId": 100,
  "examTypeCode": "CT_CHEST",
  "examTypeName": "胸部CT",
  "examPart": "胸部",
  "clinicalDiagnosis": "肺炎",
  "examPurpose": "排除肺部占位",
  "urgencyLevel": 1,
  "requestDeptId": 10,
  "requestDoctorId": 100,
  "orderId": null
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "申请接收成功",
  "data": {
    "requestId": 10001,
    "requestNo": "IA202606190001",
    "requestStatus": 2,
    "paymentStatus": 1,
    "requestTime": "2026-06-19 08:30:00"
  }
}
```

**业务规则**:
- 门诊申请需校验缴费状态，已缴费才能接收
- 住院申请需校验医嘱执行状态，医嘱已确认才能接收
- 检查类型、检查部位、患者信息不能为空

### 3.2 检查排期接口

**接口说明**: 为影像申请安排检查时间和房间

**接口路径**: `POST /api/ris/request/schedule`

**请求参数**:
```json
{
  "requestId": 10001,
  "scheduleDate": "2026-06-20",
  "timeSlotStart": "08:00",
  "timeSlotEnd": "08:30",
  "roomId": 1,
  "equipmentId": 1,
  "remark": "患者要求上午检查"
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "排期成功",
  "data": {
    "scheduleId": 5001,
    "scheduleNo": "SC202606200001",
    "queueNo": 1,
    "scheduleDate": "2026-06-20",
    "timeSlot": "08:00-08:30",
    "roomName": "CT室1"
  }
}
```

**业务规则**:
- 排期时间必须在未来
- 排期时段不能与已有排期冲突
- 急诊申请优先安排最近可用时段

## 4. 用户故事

### US-M05-001 影像申请接收

**故事卡片**
```
作为 影像科技师
我想要 接收来自门诊和住院的影像检查申请
以便于 安排患者的检查时间
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 影像申请接收

  Scenario: 正常接收门诊申请
    Given 门诊医生已为患者"张三"开立"胸部CT"检查申请
    And 患者已完成缴费
    When 影像科技师查询待接收申请
    Then 系统显示该申请，状态为"待排期"
    And 系统显示患者基本信息、检查类型、检查部位
    And 系统显示临床诊断和检查目的

  Scenario: 正常接收住院申请
    Given 住院医生已为患者"李四"开立"腹部B超"检查申请
    And 医嘱已确认执行
    When 影像科技师查询待接收申请
    Then 系统显示该申请，状态为"待排期"
    And 申请关联住院信息

  Scenario: 未缴费申请提示
    Given 门诊患者"王五"有检查申请
    And 患者尚未缴费
```

## 6. 状态机设计

### 6.1 影像申请状态流转

```
状态定义：
1 - 待缴费
2 - 待排期
3 - 已排期
4 - 已到检
5 - 检查中
6 - 已完成
7 - 已取消

状态流转图：
                    ┌─────────────────────────────────────────┐
                    │                                         │
                    ▼                                         │
┌──────┐        ┌──────┐        ┌──────┐        ┌──────┐        ┌──────┐
│待缴费│ ───→ │待排期│ ───→ │已排期│ ───→ │已到检│ ───→ │检查中│
│  1  │        │  2  │        │  3  │        │  4  │        │  5  │
└──────┘        └──────┘        └──────┘        └──────┘        └──────┘
    │               │               │                               │
    │               │               │                               │
    │               │               │                               ▼
    │               │               │                           ┌──────┐
    │               │               │                           │已完成│
    │               │               │                           │  6  │
    │               │               │                           └──────┘
    │               │               │
    └───────────────┴───────────────┴───→ ┌──────┐
                                           │已取消│
                                           │  7  │
                                           └──────┘

状态流转规则：
- 待缴费(1) → 待排期(2)：门诊患者缴费成功后自动流转
- 待缴费(1) → 已取消(7)：患者取消申请
- 待排期(2) → 已排期(3)：技师完成排期
- 待排期(2) → 已取消(7)：患者取消申请
- 已排期(3) → 已到检(4)：患者到检登记
- 已排期(3) → 已取消(7)：患者取消排期
- 已到检(4) → 检查中(5)：开始检查
- 检查中(5) → 已完成(6)：检查完成确认
- 已完成(6)：终态，不可变更
- 已取消(7)：终态，不可变更
```

### 6.2 检查排期状态流转

```
状态定义：
1 - 已排期
2 - 已到检
3 - 已过期
4 - 已取消

状态流转图：
┌──────┐        ┌──────┐        ┌──────┐
│已排期│ ───→ │已到检│ ───→ │检查中│
│  1  │        │  2  │        │      │
└──────┘        └──────┘        └──────┘
    │
    ├───→ ┌──────┐
    │     │已过期│ (排期时间过后30分钟未到检)
    │     │  3  │
    │     └──────┘
    │         │
    │         └───→ 重新排期 ───→ 已排期(1)
    │
    └───→ ┌──────┐
          │已取消│
          │  4  │
          └──────┘

状态流转规则：
- 已排期(1) → 已到检(2)：患者到检登记
- 已排期(1) → 已过期(3)：排期时间过后30分钟未到检
- 已排期(1) → 已取消(4)：取消排期
- 已过期(3) → 已排期(1)：重新排期
```

## 7. 业务规则

### 7.1 校验规则

#### BR-RIS-001: 影像申请接收校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-001 |
| 规则名称 | 影像申请接收校验 |
| 规则类型 | 校验规则 |
| 适用对象 | 影像申请(imaging_request) |
| 规则描述 | 影像申请接收时必须校验申请来源有效性：门诊申请需已缴费，住院申请需医嘱已确认 |
| 触发时机 | 申请接收时 |
| 校验逻辑 | 门诊申请检查payment_status='PAID'；住院申请检查医嘱状态为CONFIRMED或EXECUTING |
| 失败处理 | 返回具体校验失败原因，阻止申请接收 |
| 优先级 | P0 |

#### BR-RIS-002: 检查排期冲突校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-002 |
| 规则名称 | 检查排期时段冲突校验 |
| 规则类型 | 校验规则 |
| 适用对象 | 检查排期(imaging_schedule) |
| 规则描述 | 检查排期时必须校验时段是否已被占用，同一设备同一时间段只能安排一次检查 |
| 触发时机 | 排期创建/修改时 |
| 校验逻辑 | 查询同一房间、同一时间段内是否有未取消的排期记录 |
| 失败处理 | 返回错误"该时段已被占用"，显示其他可用时段 |
| 优先级 | P0 |

#### BR-RIS-003: 排期时间有效性校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-003 |

| 规则类型 | 校验规则 |
| 适用对象 | 检查排期(imaging_schedule) |

| 触发时机 | 排期创建时 |
| 校验逻辑 | schedule_time >= CURRENT_TIME；时间在08:00-17:30范围内；不在12:00-13:00午休时间 |
| 失败处理 | 返回具体校验失败原因 |
| 优先级 | P0 |
