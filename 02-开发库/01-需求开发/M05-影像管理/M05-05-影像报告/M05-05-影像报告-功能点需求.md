## 1. 功能概述

### 1.1 功能定位

影像报告模块是影像管理子系统(RIS/PACS)的核心诊断输出模块，负责影像检查报告的书写、审核、发布、打印和管理。系统支持结构化报告和自由文本报告两种模式，提供报告模板管理功能，实现报告质量控制（单签/双签审核），确保诊断报告的准确性和规范性，为临床诊疗提供重要依据。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短报告出具时间 | 平均报告出具时间≤24小时 |
| 质量目标 | 确保报告准确性 | 报告审核通过率≥98% |
| 标准目标 | 提升报告规范性 | 结构化报告占比≥80% |
| 安全目标 | 保障报告数据安全 | 报告保存期限≥30年 |

### 5.2 关键业务规则详解

#### BR-RIS-011: 报告必填项校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-011 |
| 规则名称 | 影像报告必填项校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 影像报告(imaging_report) |
| 规则描述 | 影像报告必须有影像所见和诊断意见，报告内容不能为空 |
| 触发时机 | 报告提交/发布时 |

**校验逻辑**:
- 影像所见不能为空
- 诊断意见不能为空
- 影像所见字数限制≤5000字
- 诊断意见字数限制≤2000字

#### BR-RIS-018: 报告状态流转

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-018 |
| 规则名称 | 影像报告状态流转规则 |
| 规则类型 | FLOW(流转规则) |
| 适用对象 | 影像报告(imaging_report) |
| 规则描述 | 报告状态流转：草稿→待审核→已发布/已撤回；审核退回回到草稿状态 |
| 触发时机 | 报告状态变更时 |

**状态定义**: 1-草稿, 2-待审核, 3-已发布, 4-已撤回

**状态流转约束**:
- 草稿(1) → 待审核(2)：提交审核
- 待审核(2) → 已发布(3)：审核通过
- 待审核(2) → 草稿(1)：审核退回
- 已发布(3) → 已撤回(4)：报告撤回
- 已撤回(4) → 草稿(1)：重新编辑

#### BR-RIS-021: 报告审核发布流程

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-021 |
| 规则名称 | 报告审核发布流程 |
| 规则类型 | FLOW(流转规则) |
| 适用对象 | 影像报告审核(report_audit) |
| 规则描述 | 报告提交审核后进入待审核状态，审核通过后发布并通知临床，审核退回通知报告医生 |
| 触发时机 | 报告审核时 |

**流程逻辑**:
1. 审核通过：报告状态变更为"已发布"，记录审核人和时间，通知临床医生
2. 审核退回：报告状态变更为"草稿"，记录退回原因，通知报告医生

#### BR-RIS-028: 影像报告书写权限控制

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-028 |
| 规则名称 | 影像报告书写权限控制 |
| 规则类型 | PERM(权限规则) |
| 适用对象 | 影像报告(imaging_report) |
| 规则描述 | 影像报告书写需要报告医师权限，审核报告需要审核医师权限 |
| 触发时机 | 报告操作时 |

**权限检查逻辑**:
- 报告书写：需要IMAGING_REPORT_WRITE权限，用户角色为REPORT_DOCTOR或ATTENDING_DOCTOR
- 报告审核：需要IMAGING_REPORT_AUDIT权限，用户角色为SENIOR_DOCTOR或CHIEF_DOCTOR

#### BR-RIS-043: 报告审核时限

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-RIS-043 |
| 规则名称 | 报告审核时限规则 |
| 规则类型 | TIME(时间规则) |
| 适用对象 | 影像报告(imaging_report) |
| 规则描述 | 报告提交后24小时内需完成审核，超时系统自动催办 |
| 触发时机 | 定时检查时 |

**时限逻辑**:
- 提交后20小时：发送即将超时提醒
- 提交后24小时：发送超时催办通知

## 7. 交互流程

### 7.1 报告书写流程

```
步骤1: 报告医生登录影像报告模块
    │
    ↓
步骤2: 查看待报告检查列表
    │
    ↓
步骤3: 选择待报告检查，打开报告书写页面
    │
    ├──→ 左侧显示影像预览
    ├──→ 右侧显示报告编辑区域
    │
    ↓
步骤4: 查看影像，填写影像所见
    │
    ├──→ 可选择报告模板快速填充
    ├──→ 可引用历史报告内容
    │
    ↓
步骤5: 填写诊断意见
    │
    ↓
步骤6: 保存报告草稿（状态：草稿）
    │
    ↓
步骤7: 确认内容完整，点击提交审核
    │
    ↓
步骤8: 系统校验必填项
    │
    ├──→ 校验通过：报告状态变更为待审核
    ├──→ 校验失败：提示补充内容
    │
    ↓
步骤9: 发送审核通知给审核医师
```

### 7.2 报告审核发布流程

```
步骤1: 审核医师登录审核模块
    │
    ↓
步骤2: 查看待审核报告列表
    │
    ↓
步骤3: 选择待审核报告，查看报告详情
    │
    ├──→ 左侧显示影像预览
    ├──→ 右侧显示报告内容
    │
    ↓
步骤4: 查看影像与报告内容对照
    │
    ↓
步骤5: 判断报告内容是否准确
    │
    ├──→ 准确：点击审核通过
    │       │
    │       ↓
    │       报告状态变更为已发布
    │       │
    │       ↓
    │       发送报告发布通知给临床医生
    │
    └──→ 不准确：点击退回修改
            │
            ↓
            输入退回原因
            │
            ↓
            报告状态变更为草稿
            │
            ↓
            发送退回通知给报告医生
```

### 7.3 双签审核流程

```
步骤1: 特殊检查报告提交审核
    │
    ↓
步骤2: 系统判断需要双签审核
    │
    ↓
步骤3: 主审医师审核通过
    │
    ├──→ 记录主审医师签名
    ├──→ 报告状态保持待审核
    │
    ↓
步骤4: 系统等待副审医师签字
    │
    ↓
步骤5: 副审医师审核通过
    │
    ├──→ 校验副审医师与主审医师不同
    ├──→ 记录副审医师签名
    │
    ↓
步骤6: 双签完成，报告发布
    │
    ↓
步骤7: 发送报告发布通知
```

## 8. 状态机设计

### 8.1 报告状态流转图

```
                    ┌───────────┐
                    │   草稿    │
                    │  (DRAFT)  │
                    │    [1]    │
                    └───────────┘
                     ↑    │    ↑
                     │    │    │
            撤回重新  │    │    │ 退回修改

            (EDIT)   │ 审核│    │
                     │    ↓    │
                     │ ┌───────────┐
                     │ │  待审核   │
                     │ │(PENDING)  │
                     │ │    [2]    │
                     │ └───────────┘
                     │    │    │
                     │    │    │ 撤回
                     │    │    │ (WITHDRAW)
                     │    ↓    ↓
                     │ ┌───────────┐
                     │ │  已发布   │
                     │ │(PUBLISHED)│
                     │ │    [3]    │
                     │ └───────────┘
                     │    │
                     │    ↓
                     └─ ┌───────────┐
                       │  已撤回   │
                       │(WITHDRAWN)│
                       │    [4]    │
                       └───────────┘
```

### 8.2 状态定义说明

| 状态码 | 状态名称 | 英文名称 | 说明 | 可流转状态 |

| 4 | 已撤回 | WITHDRAWN | 已发布报告被撤回 | 1（重新编辑） |
