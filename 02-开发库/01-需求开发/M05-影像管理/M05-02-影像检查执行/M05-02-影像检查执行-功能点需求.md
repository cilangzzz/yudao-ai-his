## 1. 功能概述

### 1.1 功能定位

影像检查执行模块是影像管理子系统(RIS/PACS)的核心执行模块，负责检查登记、DICOM影像采集、影像上传、影像质量检查、检查完成确认等功能。该模块是RIS与PACS的核心连接点，实现影像设备与信息系统的无缝对接，遵循DICOM 3.0国际标准。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短影像检查执行时间 | 平均检查执行时间≤30分钟 |
| 标准目标 | 实现DICOM标准对接 | DICOM设备对接成功率100% |
| 质量目标 | 提升影像采集质量 | 影像质量合格率≥98% |
| 流程目标 | 实现检查流程闭环 | 检查完成确认及时率≥95% |

### 1.3 功能范围

## 6. 交互流程

### 6.1 检查登记流程

```
步骤1: 技师进入检查登记页面
    │
    ↓
步骤2: 输入申请编号或扫描患者腕带
    │
    ↓
步骤3: 系统查询并显示申请信息和患者信息
    │
    ├── 若申请状态为"待缴费" → 提示"患者未缴费，请先引导患者缴费"
    │
    ├── 若排期已过期 → 提示"排期已过期，需要重新排期"
    │
    └── 若申请状态为"已排期" → 继续下一步
    │
    ↓
步骤4: 技师核对患者身份信息（姓名、年龄、检查类型）
    │
    ↓
步骤5: 技师选择检查设备（显示设备状态）
    │
    ├── 显示空闲设备列表
    │
    └── 技师选择空闲设备
    │
    ↓
步骤6: 技师填写备注信息（可选）
    │
    ↓
步骤7: 技师点击"确认登记"
    │
    ↓
```

## 7. 状态机设计

### 7.1 检查登记状态机

```
检查登记状态流转：

    [待排期] ─────────→ [已到检] ─────────→ [检查中] ─────────→ [已完成]
        │                  │                    │                    │
        │                  │                    │                    │
        ↓                  ↓                    ↓                    │
    [已取消]          [过期/重新排期]      [异常终止]           [不可变更]


状态说明：
- 待排期(2)：申请已接收，等待排期
- 已到检(4)：患者已完成登记
- 检查中(5)：影像正在采集
- 已完成(6)：检查完成，等待报告
- 已取消(7)：申请已取消

状态变更触发条件：
- 待排期 → 已到检：技师完成登记
- 已到检 → 检查中：开始影像采集
- 检查中 → 已完成：技师确认完成
- 任意状态 → 已取消：申请取消
```

### 7.2 影像上传状态机

```
影像上传状态流转：

    [待上传] ──────→ [上传中] ──────→ [已完成]
        │               │                  │
        │               │                  │
        ↓               ↓                  │
    [取消]         [失败] ────→ [重试] ────→ [已完成]
                        │
                        ↓
                    [本地暂存] ────→ [网络恢复] ────→ [上传中]


状态说明：
- 待上传(1)：影像已生成，等待上传
- 上传中(2)：正在上传至PACS服务器
- 已完成(3)：上传成功完成
- 失败(4)：上传失败
- 本地暂存(5)：网络故障时本地暂存

状态变更触发条件：
- 待上传 → 上传中：开始上传
- 上传中 → 已完成：上传成功
- 上传中 → 失败：网络故障或服务器异常
- 失败 → 本地暂存：自动暂存机制
- 本地暂存 → 上传中：网络恢复后自动重试
```

### 7.3 质量检查状态机

```
质量检查状态流转：

    [待检查] ──────→ [检查中] ──────→ [通过] ──────→ [不可变更]
        │               │                  │
        │               │                  │
        ↓               ↓                  ↓
    [取消]         [不合格] ────→ [重新采集] ────→ [待检查]


状态说明：
- 待检查(1)：影像已上传，等待质量检查
- 检查中(2)：正在进行质量检查
- 通过(3)：质量检查通过
- 不合格(3)：质量检查不合格

状态变更触发条件：
- 待检查 → 检查中：开始质量检查
- 检查中 → 通过：质量评分>=75分
- 检查中 → 不合格：质量评分<75分
- 不合格 → 重新采集：技师选择重新采集
```

### 8.2 关键业务规则详解

#### BR-RIS-004: 检查登记患者身份核对

检查登记时必须核对患者身份（姓名+患者编号），确保检查对象正确，防止错检。

**触发时机**: 检查登记时

**校验逻辑**:
- 患者姓名必须与申请信息匹配
- 患者编号必须与申请信息匹配
- 申请与患者信息必须一致

#### BR-RIS-006: DICOM标准合规校验

影像上传时必须校验DICOM 3.0标准合规性，校验DICOM Tag完整性。

**触发时机**: 影像上传时

**校验逻辑**:
- DICOM文件必须包含必要Tag：
  - (0010,0020) Patient ID
  - (0010,0010) Patient Name
  - (0010,0030) Patient Birth Date
  - (0010,0040) Patient Sex
  - (0020,000D) Study Instance UID
  - (0008,0060) Modality
  - (0008,0020) Study Date
  - (0008,0030) Study Time
- Study Instance UID必须唯一，不可重复
- DICOM文件必须完整，不可损坏

#### BR-RIS-008: 影像质量合格校验

影像上传后自动进行质量检查，检查影像清晰度、伪影、运动模糊等质量问题。

**触发时机**: 影像上传后

**校验逻辑**:
- 分辨率检查（>=MIN_RESOLUTION）
- 信噪比检查（>=MIN_SNR）
- 运动模糊检查（<MAX_MOTION_ARTIFACT）
- 对比度检查（>=MIN_CONTRAST）
- 质量评分计算（满分100分，合格>=75分）

#### BR-RIS-012: 影像上传完成校验

检查完成确认前必须确保影像已成功上传，未上传完成不可结束检查。

**触发时机**: 检查完成确认时

**校验逻辑**:
- 影像检查记录必须存在
- 影像上传状态必须为"已完成"
- 影像质量检查状态不可为"不合格"（若不合格需标记需重新检查）

```gherkin
Feature: 检查登记功能

  Scenario: 正常登记流程
    Given 患者已按排期到达影像科

  Scenario: 未缴费不可登记
    Given 患者申请状态为"待缴费"
    When 技师尝试登记

    And 登记操作被阻止

    Given 患者排期时间为"昨日"
    And 患者未到检
    When 技师尝试登记
    Then 系统提示"排期已过期，需要重新排期"
```

### 10.2 DICOM影像采集验收标准

```gherkin
Feature: DICOM影像采集功能

  Scenario: 正常影像采集
    Given 患者已完成检查登记
    And 检查设备已就绪
    When 技师执行影像采集
    Then 设备生成DICOM影像
    And 影像自动上传至PACS服务器
    And 影像与检查记录关联

  Scenario: DICOM信息关联
    Given 技师正在采集影像

    Then DICOM头信息自动填充患者信息

    Given 影像已采集

    And 系统提示"影像上传失败，正在重试"
    And 网络恢复后自动上传
```
