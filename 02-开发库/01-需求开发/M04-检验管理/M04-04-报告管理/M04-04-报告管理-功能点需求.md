## 1. 功能概述

### 1.1 功能定位

报告管理是检验管理子系统(LIS)的核心输出模块，负责检验报告的生成、审核、发布、打印、查询等全生命周期管理。模块重点实现危急值15分钟通报机制，确保危及患者生命的检验结果能够及时通知临床医生，保障患者安全。符合HIMSS EMRAM Stage 5+标准要求。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 安全目标 | 实现危急值15分钟通报机制 | 危急值通报时间≤15分钟 |
| 质量目标 | 检验报告审核发布 | 报告审核覆盖率100% |
| 效率目标 | 报告生成与发布效率 | 报告生成时间≤5秒 |
| 闭环目标 | 危急值确认闭环管理 | 危急值确认率100% |

## 6. 状态机设计

### 6.1 报告状态流转

```
                    ┌─────────────────┐
                    │   1-待生成      │
                    └────────┬────────┘
                             │ 生成报告
                             ↓
                    ┌─────────────────┐
                    │   2-已生成      │
                    └────────┬────────┘
                             │ 发布报告
                             ↓
                    ┌─────────────────┐
          ┌─────────│   3-已发布      │←─────────┐
          │         └─────────────────┘          │
          │ 撤回报告                    重新发布 │
          ↓                                        │
    ┌─────────────────┐                            │
    │   4-已撤回      │────────────────────────────┘
    └─────────────────┘

状态说明：
- 1-待生成：检验结果已审核，等待生成报告
- 2-已生成：报告已生成，等待发布
- 3-已发布：报告已发布，临床可见
- 4-已撤回：报告已撤回，临床不可见

状态流转规则：
- 待生成 → 已生成：检验医师生成报告
- 已生成 → 已发布：检验医师发布报告
- 已发布 → 已撤回：有权限用户撤回报告
- 已撤回 → 已发布：重新发布报告（需审批）
```

### 6.2 危急值状态流转

```
    ┌─────────────────┐
    │   1-待确认      │
    └────────┬────────┘
             │ 检验医师确认
             ↓
    ┌─────────────────┐
    │   2-待通报      │
    └────────┬────────┘
             │ 执行通报
             ↓
    ┌─────────────────┐
    │   3-已通报      │
    └────────┬────────┘
             │ 临床确认
             ↓
    ┌─────────────────┐
    │   4-已确认      │
    └────────┬────────┘
             │ 闭环完成
             ↓
    ┌─────────────────┐
    │   5-已闭环      │
    └─────────────────┘

状态说明：
- 1-待确认：危急值已识别，等待复核确认
- 2-待通报：已确认危急值，等待通报
- 3-已通报：已通报临床，等待确认
- 4-已确认：临床已确认处理
- 5-已闭环：危急值处理完成

时效要求：
- 待确认 → 待通报：建议≤5分钟
- 待通报 → 已通报：必须≤15分钟
- 已通报 → 已确认：建议≤10分钟
- 超时自动升级通知
```

### 7.2 关键业务规则详解

#### BR-LIS-008: 危急值阈值校验

检验结果超出危急值阈值时自动识别为危急值。

**触发时机**: 检验结果录入/保存时

**校验逻辑**:
- 获取患者的年龄、性别
- 查询对应检验项目的危急值阈值
- 检查结果是否超出阈值（高于高限或低于低限）
- 标记为危急值并触发警报

#### BR-LIS-017: 危急值处理流程

危急值处理流程：发现→确认→通报→临床确认→闭环完成，全程必须15分钟内完成。

**触发时机**: 危急值处理各环节

**流程要求**:
- 危急值发现后必须5分钟内确认
- 确认后必须15分钟内完成通报
- 通报后建议10分钟内临床确认
- 超时自动升级通知

#### BR-LIS-028: 危急值通报时效计算

计算危急值从发现到通报、从通报到确认的时间。

**触发时机**: 危急值状态变更时

**计算逻辑**:
- 通报耗时 = 通报时间 - 发现时间
- 确认耗时 = 确认时间 - 通报时间
- 总耗时 = 确认时间 - 发现时间
- 超过15分钟标记为超时
