## 1. 功能概述

### 1.1 功能定位

检验执行是检验管理子系统(LIS)的核心业务模块，负责检验任务的分配、执行、结果录入和审核。系统支持检验仪器自动数据采集和手工录入两种方式，实现检验结果的自动异常判断、危急值识别，为检验报告生成提供数据基础。本模块是连接标本管理与报告管理的关键环节。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提高检验执行效率，缩短检验周转时间 | 检验TAT(周转时间)≤60分钟 |
| 质量目标 | 确保检验结果准确性 | 结果审核覆盖率100% |
| 安全目标 | 实现危急值及时识别 | 危急值识别准确率100% |
| 集成目标 | 实现检验仪器数据自动采集 | 仪器对接覆盖率≥80% |

## 6. 交互流程

### 6.1 检验执行流程

```
标本接收完成
    │
    ↓
系统自动分配任务
    │
    ├── 根据项目类型分配到对应小组
    ├── 分配给当前值班的检验技师
    │
    ↓
技师查看待检验任务
    │
    ├── 急诊任务优先显示
    ├── 按时间顺序排列
    │
    ↓
技师执行检验
    │
    ├── 方式1: 仪器自动采集
    │   ├── 仪器发送检测结果
    │   ├── 系统自动解析并保存
    │   └── 自动关联对应标本
    │
    └── 方式2: 手工录入
        ├── 技师手工输入结果值
        ├── 系统自动判断是否异常
        └── 根据参考范围标识状态
    │
    ↓
结果自动异常判断
    │
    ├── 正常范围 → 状态为"正常"
    ├── 超出参考范围 → 状态为"偏高"或"偏低"
    └── 超出危急值阈值 → 状态为"危急值"，触发警报
    │
    ↓
提交结果待审核
    │
    ↓
检验医师审核
    │
    ├── 审核通过 → 进入报告生成流程
    └── 审核退回 → 技师修改后重新提交
```

### 6.2 检验结果审核流程

```
检验结果待审核
    │
    ↓
检验医师查看结果列表
    │
    ├── 按急诊优先排序
    ├── 异常结果高亮显示
    ├── 危急值红色标识
    │
    ↓
查看结果详情
    │
    ├── 显示患者基本信息
    ├── 显示检验结果和参考范围
    ├── 显示历史结果对比（如有）
    │
    ↓
审核判断
    │
    ├── 结果正确 ──→ 审核通过
    │                │
    │                └── 结果状态变更为"已审核"
    │
    └── 结果有误 ──→ 审核退回
                     │
                     ├── 填写退回原因
                     ├── 通知原录入技师
                     └── 技师修改后重新提交
```

## 7. 状态机设计

### 7.1 检验任务状态机

```
┌─────────────┐
│   待执行    │ (task_status = 1)
│   [初始]    │
└──────┬──────┘
       │
       │ 技师开始检验
       ↓
┌─────────────┐
│   执行中    │ (task_status = 2)
│             │
└──────┬──────┘
       │
       │ 结果录入完成
       ↓
┌─────────────┐
│   已完成    │ (task_status = 3)
│   [终态]    │
└─────────────┘

特殊情况：
┌─────────────┐
│   已取消    │ (task_status = 4)
│   [终态]    │
└─────────────┘
```

### 7.2 检验结果审核状态机

```
┌─────────────┐
│   待审核    │ (audit_status = 1)
│   [初始]    │
└──────┬──────┘
       │
       ├──────────────────┐
       │                  │
       │ 审核通过         │ 审核退回
       ↓                  ↓
┌─────────────┐    ┌─────────────┐
│   已审核    │    │   已退回    │
│   [终态]    │    │             │
└─────────────┘    └──────┬──────┘
                          │
                          │ 修改后重新提交
                          ↓
                   ┌─────────────┐
                   │   待审核    │
                   │             │
                   └─────────────┘
```

```

│   绿色      │
└─────────────┘

┌─────────────┐
```

### 8.2 关键业务规则详解

#### BR-LIS-005: 检验结果范围校验

**规则描述**: 检验结果必须在合理范围内，超出范围需确认后才能保存。

**触发时机**: 检验结果录入时

**校验逻辑**:
```
// 获取结果合理范围
min_value = lab_item.result_min  // 如白细胞最低0
max_value = lab_item.result_max  // 如白细胞最高100

IF result_value < min_value OR result_value > max_value THEN
    RETURN warning("结果值超出合理范围，请确认")
END IF
```

#### BR-LIS-008: 危急值阈值校验

**规则描述**: 检验结果超出危急值阈值时自动识别为危急值。

**触发时机**: 检验结果录入/保存时

**校验逻辑**:
```
// 获取危急值阈值（按年龄/性别）
critical_threshold = SELECT * FROM lis_critical_threshold
                     WHERE item_id = :item_id
                       AND (age_min IS NULL OR :patient_age >= age_min)
                       AND (age_max IS NULL OR :patient_age <= age_max)
                       AND (gender IS NULL OR gender = :patient_gender)

IF critical_threshold EXISTS THEN
    IF result_value < critical_threshold.low_limit OR result_value > critical_threshold.high_limit THEN
        // 标记为危急值
        is_critical = 1
        result_status = 4
        // 触发危急值警报
        SEND_critical_alert()
    END IF
END IF
```

#### BR-LIS-015: 检验结果状态流转

**规则描述**: 检验结果状态流转：待录入→已录入→待审核→已审核。

**状态定义**: 1-待录入, 2-已录入, 3-待审核, 4-已审核, 5-已退回

**流转规则**:
```
valid_transitions = {
    1: [2],        // 待录入 -> 已录入
    2: [3],        // 已录入 -> 待审核
    3: [4, 5],     // 待审核 -> 已审核/已退回
    4: [],         // 已审核 -> 终态
    5: [2]         // 已退回 -> 已录入（修改后重新提交）
}
```

#### BR-LIS-019: 检验结果修改流程

**规则描述**: 检验结果修改必须保留原值记录，记录修改人、修改时间、修改原因。

**触发时机**: 修改检验结果时

**处理逻辑**:
```
// 保存修改记录
INSERT INTO lis_result_history (
    result_id, original_value, new_value, 
    modify_reason, modify_by, modify_time
) VALUES (:result_id, :original_value, :new_value, :modify_reason, :current_user, NOW())

// 更新结果
UPDATE lis_result 
SET result_value = :new_value,
    modify_count = modify_count + 1,
    last_modify_time = NOW(),
    last_modify_by = :current_user
WHERE result_id = :result_id
```

#### BR-LIS-021: 检验结果审核权限

**规则描述**: 检验结果审核需要检验医师权限，技师不能审核自己录入的结果。

**触发时机**: 审核操作时

**校验逻辑**:
```
// 检查审核权限
IF NOT has_permission(current_user, 'LAB_RESULT_AUDIT') THEN
    RETURN error("无检验结果审核权限")
END IF

// 检查是否审核自己的结果
IF result.test_by = current_user.user_id THEN
    RETURN error("不能审核自己录入的结果")
END IF
```

#### BR-LIS-025: 检验结果异常判断

**规则描述**: 根据参考范围自动判断检验结果是否异常，分为正常、偏高、偏低、危急值。

**触发时机**: 结果录入时

**计算逻辑**:
```
// 获取参考范围（按年龄/性别）
reference_range = SELECT * FROM lis_reference_range
                  WHERE item_id = :item_id
                    AND (gender IS NULL OR gender = :patient_gender)
                    AND (age_min IS NULL OR :patient_age >= age_min)
                    AND (age_max IS NULL OR :patient_age <= age_max)
                  ORDER BY priority DESC
                  LIMIT 1

// 获取危急值阈值
critical_threshold = SELECT * FROM lis_critical_threshold
                     WHERE item_id = :item_id
                       AND (gender IS NULL OR gender = :patient_gender)
                       AND (age_min IS NULL OR :patient_age >= age_min)
                       AND (age_max IS NULL OR :patient_age <= age_max)

result_status = 'NORMAL'
is_abnormal = 0
is_critical = 0

// 检查危急值（优先）
IF critical_threshold EXISTS THEN
    IF result_value < critical_threshold.low_limit OR result_value > critical_threshold.high_limit THEN
        result_status = 'CRITICAL'
        is_critical = 1
        is_abnormal = 1
    END IF
END IF

// 检查异常（非危急值情况）
IF result_status = 'NORMAL' AND reference_range EXISTS THEN
    IF result_value < reference_range.low_limit THEN
        result_status = 'LOW'
        is_abnormal = 1
    ELSE IF result_value > reference_range.high_limit THEN
        result_status = 'HIGH'
        is_abnormal = 1
    END IF
END IF
```
