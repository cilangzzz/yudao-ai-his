## 1. 功能概述

### 1.1 功能定位

标本管理是检验管理子系统(LIS)的核心模块之一，负责标本采集、条码生成、运送、接收、拒收、追踪的全流程管理。本模块实现标本条码全程追溯，确保标本从采集到检验的每个环节都有完整记录，符合HIMSS EMRAM Stage 5+标准要求。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 安全目标 | 实现患者身份核对机制 | 身份核对覆盖率100% |
| 效率目标 | 标本全程条码追踪 | 标本追踪覆盖率100% |
| 质量目标 | 标本接收质量检查 | 标本拒收率≤5% |
| 时效目标 | 标本运送时效管理 | 急诊标本运送≤15分钟 |

### 1.3 功能范围

```

│   ├── 患者身份核对（腕带/就诊卡扫描）
```

## 3. 用户故事

### US-M04-02-001 标本采集

**故事卡片**
```
作为 标本采集员/护士
我想要 完成标本采集并生成条码标签
以便于 标本能够被识别和追踪
```

**优先级**: P0（MVP必需）

**故事点**: 8

**验收标准 (Gherkin)**
```gherkin
Feature: 标本采集

  Scenario: 正常标本采集流程
    Given 检验申请状态为"待采集"
    And 患者已到达采集点
    When 采集员扫描患者腕带或就诊卡
    And 确认患者身份信息
    And 完成标本采集操作
    Then 系统生成标本条码
    And 打印标本标签
    And 申请状态变更为"已采集"
    And 记录采集时间、采集人

  Scenario: 生成标本条码标签
    Given 标本采集完成
    When 系统生成条码
    Then 条码格式符合规范"BB+日期+序号"
    And 标签包含患者信息、检验项目、标本类型
    And 标签包含采集时间、采集人

  Scenario: 患者身份核对
    Given 采集员扫描患者腕带
    When 系统读取腕带信息
    Then 系统显示患者基本信息
    And 系统核对申请单患者与腕带患者一致性
    And 信息一致后允许采集

  Scenario: 急诊标本优先采集
    Given 检验申请标记为"急诊"
    When 采集员查看待采集列表
    Then 急诊申请排在列表最前
    And 急诊申请显示红色警示标识

  Scenario Exception: 患者身份不匹配
    Given 采集员扫描患者腕带
    And 腕带信息与申请单患者不一致
    When 系统比对信息
    Then 系统提示"患者身份不匹配"
    And 阻止采集操作
    And 提供人工核实入口

  Scenario Boundary: 标本条码唯一性
    Given 系统正在生成标本条码
    When 新条码与已有条码重复
    Then 系统自动重新生成
    And 记录重复尝试日志
```

### US-M04-02-002 标本运送

**故事卡片**
```
作为 标本运送员
我想要 完成标本运送交接
以便于 标本能够及时送达检验科
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 标本运送

  Scenario: 正常运送流程
    Given 标本状态为"已采集"
    When 运送员扫描标本条码接收标本
    And 运送员确认标本数量
    And 运送员送达检验科
    Then 标本状态变更为"运送中"
    And 记录运送人、运送开始时间

  Scenario: 标本交接确认
    Given 运送员到达检验科
    When 检验技师扫描标本条码
    And 确认标本数量和质量
    Then 系统记录交接完成
    And 标本状态变更为"已接收"

  Scenario: 标本追踪查询
    Given 标本已采集
    When 用户查询标本位置
    Then 系统显示标本当前位置
    And 显示标本流转轨迹
    And 显示各环节处理时间

  Scenario: 批量运送
    Given 有多个待运送标本
    When 运送员选择批量运送
    And 扫描多个标本条码
    Then 系统生成运送清单
    And 打印运送交接单

  Scenario Exception: 运送超时告警
    Given 标本状态为"运送中"
    And 运送时间超过规定时限
    When 系统检查运送状态
    Then 系统发送超时告警
    And 通知运送员和接收科室

  Scenario Boundary: 急诊标本运送时效
    Given 标本为急诊标本
    When 运送员接收标本
    Then 系统显示急诊优先标识
    And 提示需要在15分钟内送达
```

### 5.1 相关业务规则引用

| 规则编号 | 规则名称 | 规则类型 | 适用场景 |
|----------|----------|----------|----------|
| BR-LIS-001 | 标本条码格式校验 | 校验规则 | 标本条码生成时 |
| BR-LIS-002 | 标本条码唯一性校验 | 校验规则 | 标本条码生成时 |
| BR-LIS-003 | 患者身份核对校验 | 校验规则 | 标本采集时 |
| BR-LIS-004 | 标本质量检查校验 | 校验规则 | 标本接收时 |
| BR-LIS-009 | 标本采集超时校验 | 校验规则 | 定时检查 |
| BR-LIS-010 | 标本运送超时校验 | 校验规则 | 定时检查 |

| BR-LIS-018 | 标本拒收处理流程 | 流转规则 | 标本拒收时 |
| BR-LIS-027 | 标本编号生成 | 计算规则 | 标本创建时 |

### 5.2 关键业务规则详解

#### BR-LIS-001: 标本条码格式校验

标本条码必须符合格式规范，格式为"BB+日期(8位)+序号(4位)"，如BB202606190001。

**触发时机**: 标本条码生成时

**校验逻辑**:
- 条码必须以"BB"开头
- 日期部分必须为有效日期
- 序号部分必须为4位数字

#### BR-LIS-002: 标本条码唯一性校验

标本条码必须全局唯一，不允许重复。

**触发时机**: 标本条码生成时

**处理逻辑**:
- 生成新条码时检查是否已存在
- 如存在重复，自动重新生成
- 记录重复尝试日志

#### BR-LIS-003: 患者身份核对校验

标本采集时必须核对患者身份，腕带/就诊卡信息与申请单患者信息必须一致。

**触发时机**: 标本采集确认时

**校验逻辑**:
- 核对姓名一致性
- 核对患者编号一致性
- 核对性别一致性
- 信息不一致时阻止采集操作

#### BR-LIS-014: 标本状态流转

标本状态流转规则：已采集(1) -> 运送中(2) -> 已接收(3)/已拒收(4) -> 已销毁(5)

**状态流转约束**:
```
valid_transitions = {
    1: [2, 4],     // 已采集 -> 运送中/已拒收
    2: [3, 4],     // 运送中 -> 已接收/已拒收
    3: [5],        // 已接收 -> 已销毁
    4: [],         // 已拒收 -> 终态
    5: []          // 已销毁 -> 终态
}
```

#### BR-LIS-027: 标本编号生成

标本编号格式：BB+日期(YYYYMMDD)+序号(4位)

**生成逻辑**:
- 日期部分取当前日期
- 序号为当日递增序号，从0001开始
- 格式化为4位，不足补0

## 7. 交互流程

### 7.1 标本采集流程

```
1. 采集员登录系统，进入标本采集页面
2. 系统显示待采集标本列表（急诊优先）
3. 采集员选择待采集标本
4. 采集员扫描患者腕带或就诊卡
5. 系统显示患者信息并进行身份核对
   - 一致：继续采集流程
   - 不一致：提示错误，阻止采集
6. 采集员确认标本类型、容器信息
7. 采集员完成标本采集操作
8. 系统生成标本条码
9. 系统打印标本标签
10. 采集员将标签贴于标本容器
11. 系统记录采集信息，更新申请状态为"已采集"
```

### 7.2 标本运送流程

```
1. 运送员登录系统，进入标本运送页面
2. 系统显示待运送标本列表
3. 运送员扫描标本条码接收标本
4. 系统记录运送人、运送开始时间
5. 标本状态变更为"运送中"
6. 运送员将标本送达检验科
7. 检验技师扫描标本条码
8. 技师进行标本质量检查
9. 技师确认接收或拒收
   - 接收：标本状态变更为"已接收"，进入待检验队列
   - 拒收：标本状态变更为"已拒收"，通知申请科室
```

### 7.3 标本拒收处理流程

```
1. 检验技师扫描标本条码
2. 技师进行标本质量检查
3. 发现标本不合格
4. 技师选择拒收操作
5. 系统显示拒收原因选择界面
6. 技师选择或输入拒收原因
7. 系统记录拒收信息
8. 标本状态变更为"已拒收"
9. 系统发送通知给申请科室
10. 申请科室收到通知，安排重新采集或取消申请
```
