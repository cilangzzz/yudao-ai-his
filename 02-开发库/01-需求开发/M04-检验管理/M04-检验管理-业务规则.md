#### BR-LIS-008: 危急值阈值校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-LIS-008 |
| 规则名称 | 危急值阈值校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 检验结果(lab_result) |
| 规则描述 | 检验结果超出危急值阈值时自动识别为危急值 |
| 触发时机 | 检验结果录入/保存时 |
| 校验逻辑 | ```
// 伪代码
lab_item = SELECT * FROM lis_item WHERE item_id = :item_id
patient = SELECT * FROM patient WHERE patient_id = :patient_id

// 获取危急值阈值（按年龄/性别）
critical_threshold = SELECT * FROM lis_critical_threshold
                     WHERE item_id = :item_id
                       AND (age_min IS NULL OR :patient_age >= age_min)
                       AND (age_max IS NULL OR :patient_age <= age_max)
                       AND (gender IS NULL OR gender = :patient_gender)

IF critical_threshold EXISTS THEN
    is_critical = false
    
    // 检查低限
    IF result_value < critical_threshold.low_limit THEN
        is_critical = true
    END IF
    
    // 检查高限
    IF result_value > critical_threshold.high_limit THEN
        is_critical = true
    END IF
    
    IF is_critical THEN
        // 标记为危急值
        UPDATE lis_result SET is_critical = 1 WHERE result_id = :result_id
        RETURN critical_alert({
            'item_name': lab_item.item_name,
            'result_value': result_value,
            'threshold': critical_threshold
        })
    END IF
END IF

RETURN success()
``` |
| 失败处理 | 自动标记为危急值，触发危急值通报流程 |
| 优先级 | P0 |

| 规则描述 | 检验申请状态流转：待缴费→待采集→已采集→运送中→已接收→检验中→已完成 |

valid_transitions = {
    1: [2, 8],     // 待缴费 -> 待采集/已撤回
    2: [3, 8],     // 待采集 -> 已采集/已撤回
    3: [4],        // 已采集 -> 运送中

    RETURN error("状态流转不合法：当前状态[" + current_status + "]不可变更为[" + target_status + "]")
END IF

// 记录状态变更
INSERT INTO lis_request_status_log (request_id, from_status, to_status, operator, operate_time)
VALUES (:request_id, :current_status, :target_status, :current_user, NOW())

RETURN success()
``` |
