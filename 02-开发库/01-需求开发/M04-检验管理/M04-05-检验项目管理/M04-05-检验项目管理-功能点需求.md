## 1. 功能概述

### 1.1 功能定位

检验项目管理是检验管理子系统(LIS)的基础配置模块，负责检验项目、参考范围、危急值阈值、检验套餐、检验仪器、检验科室等基础数据的管理。本模块为检验申请、检验执行、报告发布等核心业务流程提供基础数据支撑，是LIS系统运行的必要前提。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提供完善的检验项目配置能力 | 配置响应时间<=2秒 |
| 质量目标 | 确保参考范围和危急值阈值准确 | 配置错误率<=0.1% |
| 管理目标 | 支持检验项目全生命周期管理 | 项目状态追踪覆盖率100% |
| 集成目标 | 支持检验仪器对接配置 | 仪器对接覆盖率>=80% |

|--------|----------|------|------|

| item_name | VARCHAR(100) | 是 | 项目名称 |

| min_specimen_volume | DECIMAL(10,2) | 否 | 最小标本量(ml) |
| storage_condition | VARCHAR(50) | 否 | 保存条件：室温/冷藏/冷冻 |

```
  │
  ↓

  │
  ↓

  ↓
```

```
  │
  ├── 校验通过 -> 保存成功

```

## 7. 状态机设计

### 7.1 检验项目状态机

```
                    ┌─────────────┐
                    │   创建项目   │
                    └──────┬──────┘
                           │
                           ↓
                    ┌─────────────┐
          ┌────────│   已启用     │←───────┐
          │        └──────┬──────┘        │
          │               │               │
          │               ↓               │
          │        ┌─────────────┐        │
          │        │    停用     │────────┘
          │        └──────┬──────┘
          │               │
          ↓               ↓
   ┌─────────────┐ ┌─────────────┐
   │  修改项目   │ │  删除项目   │
   └─────────────┘ └─────────────┘
        (已停用且无关联数据时可删除)
```

**状态说明**:
- **已启用**: 项目可正常使用，可被申请
- **已停用**: 项目不再开展，不可被新申请，历史数据保留

**状态流转规则**:
- 新创建项目默认为"已启用"状态
- 启用状态可变更为停用状态
- 停用状态可重新启用
- 已停用且无关联数据的项目可删除

### 7.2 参考范围优先级匹配规则

```
患者信息（性别、年龄）
        │
        ↓
┌─────────────────────────────────────┐
│ 优先级1: 性别+年龄精确匹配          │
│ (gender = 患者性别 AND              │
│  age_min <= 患者年龄 <= age_max)      │
└──────────────┬──────────────────────┘
               │ 找到?
               ├──是──→ 返回该参考范围
               │
               ↓ 否
┌─────────────────────────────────────┐
│ 优先级2: 年龄匹配(通用性别)         │
│ (gender IS NULL AND                 │
│  age_min <= 患者年龄 <= age_max)      │
└──────────────┬──────────────────────┘
               │ 找到?
               ├──是──→ 返回该参考范围
               │
               ↓ 否
┌─────────────────────────────────────┐
│ 优先级3: 性别匹配(通用年龄)         │
│ (gender = 患者性别 AND              │
│  age_min IS NULL AND age_max IS NULL)│
└──────────────┬──────────────────────┘
               │ 找到?
               ├──是──→ 返回该参考范围
               │
               ↓ 否
┌─────────────────────────────────────┐
│ 优先级4: 默认参考范围               │
│ (gender IS NULL AND                 │
│  age_min IS NULL AND age_max IS NULL)│
└──────────────┬──────────────────────┘
               │
               ↓
          返回默认参考范围
```

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-LIS-011 |
| 规则名称 | 检验项目编码唯一性校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 检验项目(lis_item) |
| 规则描述 | 检验项目编码必须全局唯一 |
| 触发时机 | 创建/修改检验项目时 |
| 校验逻辑 | 检查是否存在相同编码的项目，排除自身 |

#### BR-LIS-012: 参考范围重叠校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-LIS-012 |
| 规则名称 | 参考范围重叠校验 |
| 规则类型 | VAL(校验规则) |
| 适用对象 | 检验项目参考范围(lis_reference_range) |
| 规则描述 | 同一检验项目的参考范围配置不能有重叠 |
| 触发时机 | 配置/修改参考范围时 |
| 校验逻辑 | 检查是否存在年龄范围和性别重叠的配置 |
| 失败处理 | 返回错误"参考范围存在重叠"，显示冲突配置 |

| 失败处理 | 返回错误"无检验项目配置权限" |
| 优先级 | P0 |

#### BR-LIS-026: 参考范围获取规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-LIS-026 |
| 规则名称 | 参考范围获取规则 |
| 规则类型 | CALC(计算规则) |
| 适用对象 | 检验项目参考范围(lis_reference_range) |

| 计算逻辑 | 按优先级匹配：性别+年龄精确匹配 > 年龄匹配 > 性别匹配 > 默认范围 |
| 失败处理 | - |
| 优先级 | P0 |
