## 1. 功能概述

### 1.1 功能定位

智能分诊是AI辅助子系统的核心模块，利用自然语言处理(NLP)和机器学习技术，根据患者描述的症状智能推荐就诊科室，提供分诊建议。系统支持多种输入方式（文字、语音），提供分诊优先级建议，并支持分诊护士人工复核确认，确保分诊准确性。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短患者分诊等待时间 | 平均分诊时间<=2分钟 |
| 准确目标 | 提升分诊准确率 | 分诊准确率>=85% |
| 便捷目标 | 支持多渠道分诊方式 | 自助分诊占比>=60% |
| 体验目标 | 减少患者就诊困惑 | 患者满意度>=90% |

### 1.3 功能范围


## 3. 用户故事

### US-M13-01-001 智能分诊症状录入

**故事卡片**
```
作为 患者
我想要 通过小程序或自助机描述症状并获得分诊建议
以便于 快速找到合适的科室就诊
```

**优先级**: P0（MVP必需）

**故事点**: 8

**验收标准 (Gherkin)**
```gherkin
Feature: 智能分诊症状录入

  Scenario: 文字描述症状获得分诊建议
    Given 患者已进入智能分诊页面
    When 患者输入症状描述"我头痛、发热、咳嗽三天了"
    And 患者点击"开始智能分诊"
    Then 系统解析症状为"头痛"、"发热"、"咳嗽"
    And 系统显示推荐科室列表
    And 每个推荐科室显示置信度
    And 置信度最高的科室标记为"推荐"

  Scenario: 语音输入症状
    Given 患者已进入智能分诊页面
    When 患者点击"语音输入"按钮
    And 患者说出"我肚子疼，还有点恶心"
    Then 系统将语音转换为文字
    And 系统解析症状并显示分诊建议

  Scenario: 快速选择常见症状
    Given 患者已进入智能分诊页面
    When 患者点击症状标签"发热"和"咳嗽"
    Then 系统自动添加症状到描述区
    And 系统显示对应的分诊建议

  Scenario: 分诊结果展示
    Given 系统已完成症状分析
    When 显示分诊结果
    Then 显示推荐科室（置信度>=70%）
    And 显示可选科室列表
    And 显示分诊优先级
    And 显示AI置信度
    And 显示提示"AI分诊仅供参考"

  Scenario Exception: 症状无法识别
    Given 患者输入的症状描述无法解析
    When 系统分析症状
    Then 系统提示"未能识别您的症状，请重新描述或选择人工咨询"
    And 提供"人工咨询"入口

  Scenario Boundary: 症状描述长度限制
    Given 患者正在输入症状描述
    When 输入内容超过500字
    Then 系统提示"症状描述不能超过500字"
    And 阻止继续输入
```

### US-M13-01-002 分诊确认

**故事卡片**
```
作为 分诊护士
我想要 复核和确认AI分诊结果
以便于 确保分诊准确性并处理特殊情况
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 分诊护士确认

  Scenario: 查看待确认分诊列表
    Given 分诊护士已登录分诊系统
    And 有患者完成智能分诊待确认
    When 护士进入分诊确认页面
    Then 系统显示待确认分诊列表
    And 显示患者信息和症状描述
    And 显示AI推荐科室和置信度

  Scenario: 确认AI分诊结果
    Given 护士正在查看分诊详情
    And AI推荐科室为"内科"
    When 护士点击"确认分诊"
    Then 分诊状态变更为"已确认"
    And 系统记录确认护士和确认时间
    And 患者可进入挂号流程

  Scenario: 调整分诊科室
    Given 护士正在查看分诊详情
    And AI推荐科室为"内科"
    And 护士判断应挂"呼吸内科"
    When 护士选择调整科室为"呼吸内科"
    And 护士输入调整原因"症状主要为咳嗽，呼吸科更专业"
    Then 分诊科室变更为"呼吸内科"
    And 系统记录调整记录

  Scenario: 标记分诊不准确
    Given 护士确认分诊后患者就诊完成
    When 护士标记实际就诊科室与分诊不符
    Then 系统记录分诊准确性反馈
    And 反馈数据用于模型优化

  Scenario Exception: 置信度过低需人工判断
    Given AI分诊置信度低于70%
    When 患者完成智能分诊
    Then 系统自动标记为"需人工分诊"
    And 分诊列表特殊提示
    And 护士需进行人工判断

  Scenario Boundary: 分诊确认超时
    Given 患者完成智能分诊
    When 30分钟内无人确认
    Then 系统发送超时提醒
    And 分诊列表高亮显示
```

#### BR-AI-020: 智能分诊置信度计算

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-AI-020 |
| 规则名称 | 智能分诊置信度计算 |
| 规则类型 | CALC(计算规则) |
| 适用对象 | 分诊记录(triage) |
| 规则描述 | 根据症状匹配度、症状权重、历史数据计算科室推荐置信度 |
| 触发时机 | AI分诊分析时 |

**计算逻辑**:
```
// 症状实体提取
symptom_entities = NLP_EXTRACT(symptom_text)

// 症状标准化
standardized_symptoms = STANDARDIZE(symptom_entities)

// 科室匹配计算
FOR each dept IN department_list DO
    // 计算症状匹配度
    match_score = CALCULATE_symptom_match(standardized_symptoms, dept.symptom_keywords)
    
    // 计算历史匹配度
    history_score = CALCULATE_history_match(patient.history, dept)
    
    // 综合置信度
    confidence = match_score * 0.7 + history_score * 0.3
    
    dept_confidence_list.add({dept, confidence})
END FOR

// 排序取Top3
SORT(dept_confidence_list, by=confidence, desc=true)
recommended_depts = dept_confidence_list[0:3]

RETURN recommended_depts
```

#### BR-AI-029: 分诊优先级计算

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-AI-029 |
| 规则名称 | 分诊优先级计算 |
| 规则类型 | CALC(计算规则) |
| 适用对象 | 分诊记录(triage) |
| 规则描述 | 根据症状严重程度、生命体征计算分诊优先级（普通/优先/紧急） |
| 触发时机 | AI分诊分析时 |

**计算逻辑**:
```
priority_score = 0

// 症状严重程度评分
FOR each symptom IN symptom_entities DO
    IF symptom.severity = 'SEVERE' THEN
        priority_score = priority_score + 30
    ELSE IF symptom.severity = 'MODERATE' THEN
        priority_score = priority_score + 15
    ELSE
        priority_score = priority_score + 5
    END IF
END FOR

// 确定优先级
IF priority_score >= 60 THEN
    triage_level = 3  // 紧急
ELSE IF priority_score >= 30 THEN
    triage_level = 2  // 优先
ELSE
    triage_level = 1  // 普通
END IF

RETURN triage_level
```
