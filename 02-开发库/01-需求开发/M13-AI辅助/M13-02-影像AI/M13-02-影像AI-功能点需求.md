## 1. 功能概述

### 1.1 功能定位

影像AI模块是AI辅助子系统的核心功能之一，对DICOM影像进行自动分析，识别病灶并标注定位，计算病灶尺寸，生成AI辅助报告草稿。支持多种影像类型（CT、MRI、X线、超声），检测结果供影像医生复核确认。影像AI遵循"辅助而非替代"原则，所有AI结果必须经人工复核确认后方可生效。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提高影像诊断效率，缩短报告书写时间 | 报告书写时间缩短30% |
| 安全目标 | 降低漏诊误诊风险，AI辅助检测异常 | 影像异常检出率>=95% |
| 质量目标 | 提供标准化病灶测量，减少人工误差 | 病灶测量精度0.1mm |
| 体验目标 | 减轻影像医生工作负担 | 医生满意度>=80% |

## 6. 交互流程

### 6.1 影像AI分析流程

```
影像检查完成
    │
    ↓
影像上传至PACS
    │
    ↓
系统自动触发AI分析请求
    │
    ├── 检查影像质量 ──→ 质量不满足 ──→ 提示医生，独立诊断
    │
    ↓ (质量满足)
影像预处理
    │
    ├── 格式标准化
    ├── 图像增强
    │
    ↓
调用AI模型进行推理
    │
    ├── 病灶检测
    ├── 病灶分类
    ├── 位置定位
    ├── 尺寸测量
    │
    ↓
计算置信度
    │
    ├── 总体置信度
    ├── 各病灶置信度
    │
    ↓
生成AI辅助报告草稿
    │
    ↓
结果返回并展示
    │
    ↓
影像医生查看AI结果
    │
    ├── 确认采用 ──→ AI结果合并入正式报告
    ├── 部分采用 ──→ 选中病灶合并入报告
    ├── 调整后采用 ──→ 记录调整内容后合并
    └── 忽略 ──→ 医生独立书写报告
    │
    ↓
记录复核反馈
    │
    └── 用于模型效果评估和优化
```

### 6.2 AI结果复核流程步骤

1. **查看AI分析结果**: 医生打开影像查看页面，系统自动加载AI分析结果
2. **查看病灶列表**: 左侧面板显示AI检测到的所有病灶列表
3. **选择病灶查看**: 点击病灶列表项，影像区域高亮显示对应病灶标注
4. **查看病灶详情**: 显示病灶类型、位置、大小、AI判断、置信度等详情
5. **查看置信度**: 
   - 置信度>=80%: 正常显示，无警告
   - 置信度60%-80%: 显示警告提示"置信度较低，请谨慎参考"
   - 置信度<60%: 显示低置信度警告"置信度过低，建议谨慎判断"
6. **查看AI报告草稿**: 右侧面板显示AI生成的报告草稿
7. **选择复核选项**:
   - 确认AI结果: 完全采用AI分析结果
   - 部分采用: 选择部分病灶信息采用
   - 调整后采用: 修改AI结果后采用
   - 忽略AI结果: 不采用AI结果，独立书写报告
8. **录入复核意见**: 医生可录入复核备注说明
9. **反馈评分**: 对AI结果进行1-5分评分（可选）
10. **提交复核结果**: 系统记录复核信息和反馈数据

```
                    ┌─────────────┐
                    │  1-待分析   │

                           │
           ┌───────────────┼───────────────┐

           ↓               │               ↓
    ┌─────────────┐        │        ┌─────────────┐
    │  3-已分析   │        │        │  4-分析失败 │
    └──────┬──────┘        │        └──────┬──────┘
           │               │               │
           │               │               │ 重试
           │               │               ↓
           │               │        ┌─────────────┐
           │               └───────→│  2-分析中   │
           │                        └─────────────┘
           │

           ↓
    ┌─────────────┐
    │  5-已复核   │
    └─────────────┘
```

### 7.2 医生复核状态流转

```
医生复核状态 (doctor_confirm):
0 - 未确认: 初始状态，AI结果待复核
1 - 已确认: 医生确认采用AI结果
2 - 已调整: 医生调整后采用AI结果
3 - 已忽略: 医生忽略AI结果，独立诊断

状态流转规则:
- 0(未确认) → 1(已确认): 医生确认采用
- 0(未确认) → 2(已调整): 医生调整后采用
- 0(未确认) → 3(已忽略): 医生忽略AI结果
- 状态变更后不可逆转

采用状态 (is_adopted):
0 - 不采用: 忽略AI结果
1 - 部分采用: 仅采用部分病灶信息
2 - 完全采用: 完全采用AI结果
```

### 8.2 关键业务规则详解

#### BR-AI-003: 影像AI分析置信度阈值

**规则描述**: 影像AI检测结果置信度>=80%才显示正常，低于80%显示警告提示

**触发时机**: 影像AI分析完成时

**校验逻辑**:
```
对于每个病灶:
  IF 置信度 >= 80% THEN
    显示状态 = 正常
  ELSE IF 置信度 >= 60% THEN
    显示状态 = 警告
    提示信息 = "置信度较低，请谨慎参考"
  ELSE
    显示状态 = 低置信度
    提示信息 = "置信度过低，建议谨慎判断"
  END IF

总体置信度 < 80% 时，显示警告标识
```

#### BR-AI-004: 影像质量AI分析条件校验

**规则描述**: 影像必须满足AI分析的质量要求（分辨率、完整性、格式）才能进行AI分析

**触发时机**: AI分析请求时

**校验逻辑**:
- 影像分辨率必须满足最低要求
- 影像数据必须完整
- 影像格式必须支持（DICOM/JPEG/PNG）
- 切片数量必须大于0

#### BR-AI-021: 影像病灶测量计算

**规则描述**: 自动计算病灶最大径、体积，测量精度0.1mm

**触发时机**: AI检测到病灶时

**计算规则**:
- 最大径: 病灶轮廓的最长轴
- 体积: 三维影像时计算体积（cm3）
- 面积: 病灶轮廓面积（mm2）
- 测量精度: 最大径精度0.1mm，体积精度0.01cm3

#### BR-AI-030: AI服务调用超时控制

**规则描述**: 影像AI超时60秒，超时后返回错误并提供降级处理

**超时配置**:
- 影像AI分析超时: 60秒
- 超时后处理: 提示"AI分析服务暂时不可用"，允许医生独立诊断

#### BR-AI-032: AI服务结果缓存

**规则描述**: 影像AI结果缓存24小时，避免重复调用

**缓存策略**:
- 缓存键: study_id + series_id
- 缓存时间: 24小时
- 缓存命中: 直接返回缓存结果，跳过AI调用
