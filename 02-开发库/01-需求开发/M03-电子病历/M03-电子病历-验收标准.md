## 1. 病历模板管理验收标准

### 1.1 创建病历模板

#### Feature: 创建病历模板功能

```gherkin
Feature: 创建病历模板
  作为科室管理员
  我想要创建标准化的病历模板
  以便于医生快速书写规范病历

  # ========== 正常流程场景 ==========

  Scenario: 创建入院记录模板
    Given 科室管理员已登录系统
    And 管理员有"内科"科室模板管理权限
    When 管理员进入模板管理页面
    And 管理员点击"新建模板"
    And 管理员选择模板类型"入院记录"
    And 管理员设置模板名称"内科入院记录模板"
    And 管理员选择适用范围"科室"
    And 管理员编辑模板内容
    And 管理员点击"保存模板"
    Then 系统创建模板记录，状态为"草稿"
    And 模板关联科室"内科"
    And 系统记录创建人和创建时间

  Scenario: 配置模板变量
    Given 管理员正在创建病历模板
    When 管理员在模板内容中插入变量"{患者姓名}"
    And 管理员在模板内容中插入变量"{入院日期}"
    Then 系统记录模板变量定义
    And 使用模板时自动填充对应患者信息

  Scenario: 创建个人模板
    Given 医生已登录系统
    When 医生创建模板并选择适用范围"个人"
    Then 模板仅对该医生可见可用

  # ========== 异常流程场景 ==========

  Scenario: 模板内容必填校验
    Given 管理员正在保存模板
    When 模板名称为空或模板内容为空
    Then 系统提示"模板名称和内容不能为空"
    And 模板保存失败

  Scenario: 全院模板权限校验
    Given 管理员仅有科室模板管理权限
    When 管理员尝试创建"全院"范围的模板
    Then 系统提示"无权限创建全院模板"

  # ========== 边界条件场景 ==========

  Scenario: 模板名称长度限制
    Given 管理员正在创建模板
    When 管理员输入模板名称超过100字符
    Then 系统提示"模板名称不能超过100字符"

  Scenario: 模板数量上限
    Given 科室已有50个模板
    When 管理员尝试创建第51个模板
    Then 系统提示"模板数量已达上限"
```

### 1.2 模板审核发布

#### Feature: 模板审核发布功能

```gherkin
Feature: 模板审核发布
  作为科室主任
  我想要审核并发布病历模板
  以便于模板能够在科室正式使用

  # ========== 正常流程场景 ==========

  Scenario: 提交模板审核
    Given 管理员已创建模板，状态为"草稿"
    When 管理员点击"提交审核"
    Then 模板状态变更为"待审核"
    And 系统通知科室主任审核

  Scenario: 审核通过发布
    Given 科室主任有模板审核权限
    And 有模板待审核
    When 科室主任审核模板内容
    And 科室主任点击"审核通过"
    Then 模板状态变更为"已发布"
    And 模板对科室医生可见可用

  Scenario: 审核退回修改
    Given 科室主任审核模板
    When 科室主任发现模板内容不符合规范
    And 科室主任点击"退回修改"
    And 输入退回原因"缺少体格检查项"
    Then 模板状态变更为"草稿"
    And 系统通知创建人修改

  # ========== 异常流程场景 ==========

  Scenario: 重复提交审核
    Given 模板状态为"待审核"
    When 管理员尝试再次提交审核
    Then 系统提示"模板正在审核中，请勿重复提交"

  # ========== 边界条件场景 ==========

  Scenario: 停用已发布模板
    Given 模板状态为"已发布"
    When 科室主任点击"停用模板"
    Then 模板状态变更为"已停用"
    And 模板不再显示在可选列表中

  Scenario: 已发布模板修改
    Given 模板状态为"已发布"
    When 管理员尝试修改模板内容
    Then 系统提示"已发布模板不可修改，请复制后修改"
```

## 2. 病历编辑验收标准

### 2.1 入院记录编辑

#### Feature: 入院记录编辑功能

```gherkin
Feature: 入院记录编辑
  作为住院医师
  我想要使用模板编辑入院记录
  以便于规范记录患者入院信息

  # ========== 正常流程场景 ==========

  Scenario: 使用模板创建入院记录
    Given 住院医师已接诊住院患者"张三"
    And 患者入院时间为"2026-06-19 10:00"
    When 医生进入病历编辑页面
    And 医生选择"入院记录"模板
    Then 系统自动填充患者基本信息
    And 系统自动填充入院日期
    And 医生可编辑病历内容

  Scenario: 保存病历草稿
    Given 医生正在编辑入院记录
    When 医生点击"保存草稿"
    Then 病历状态为"草稿"
    And 系统记录最后保存时间
    And 医生可继续编辑

  Scenario: 引用检验结果
    Given 医生正在编辑入院记录
    And 患者有检验结果
    When 医生点击"引用检验"
    And 选择某次检验报告
    Then 系统将检验结果摘要插入病历内容

  Scenario: 引用影像报告
    Given 医生正在编辑入院记录
    And 患者有影像报告
    When 医生点击"引用影像"
    And 选择某次影像报告
    Then 系统将影像诊断结论插入病历内容

  Scenario: 病历自动保存
    Given 医生正在编辑病历
    When 编辑时间超过5分钟未手动保存
    Then 系统自动保存草稿
    And 系统提示"已自动保存"

  # ========== 异常流程场景 ==========

  Scenario: 病历必填项校验
    Given 医生正在提交病历
    When 主诉或现病史为空
    Then 系统提示"主诉和现病史为必填项"
    And 病历提交失败

  Scenario: 病历保存失败
    Given 医生点击保存病历
    When 网络异常导致保存失败
    Then 系统提示"保存失败，请重试"
    And 系统保留本地编辑内容

  # ========== 边界条件场景 ==========

  Scenario: 入院记录时限
    Given 患者入院时间为"2026-06-19 10:00"
    And 当前时间为"2026-06-20 12:00"
    When 医生提交入院记录
    Then 系统提示"入院记录超过24小时时限"
    And 记录超时信息

  Scenario: 病历内容长度校验
    Given 医生正在编辑病历
    When 现病史内容少于100字符
    Then 系统显示"现病史内容建议不少于100字符"提示
```

## 3. 病历审签验收标准

### 3.1 提交病历审签

#### Feature: 提交病历审签功能

```gherkin
Feature: 提交病历审签
  作为住院医师
  我想要提交病历进行审签
  以便于上级医师审核确认病历质量

    Given 住院医师已完成病历编辑

    Then 病历状态变更为"待审签"
    And 系统通知主治医师审核

  Scenario: 病历完整性校验
    Given 医生点击提交审签
    When 病历存在必填项未填写
    Then 系统提示未填写的必填项
    And 提交审签失败

  Scenario: 提交指定审签人
    Given 病历需特定上级医师审核
    When 医生提交审签时选择审签人"李主任"
    Then 系统通知指定审签人

  # ========== 异常流程场景 ==========

  Scenario: 审签服务异常
    Given 医生提交审签
    When 审签服务无响应
    Then 系统提示"审签服务异常，请稍后重试"

  Scenario: 重复提交审签
    Given 病历状态为"待审签"
    When 医生尝试再次提交审签
    Then 系统提示"病历正在审签中，请勿重复提交"
```

## 4. 病历质控验收标准

### 4.1 时限质控检查

#### Feature: 时限质控检查功能

```gherkin
Feature: 时限质控检查
  作为质控医师
  我想要查看病历时限质控结果
  以便于督促医生按时完成病历

  # ========== 正常流程场景 ==========

    Then 系统标记入院记录"超时"
```
