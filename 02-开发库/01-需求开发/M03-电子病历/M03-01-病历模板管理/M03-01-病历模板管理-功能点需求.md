## 1. 功能概述

### 1.1 功能定位

病历模板管理是电子病历子系统的配置模块，负责病历模板的创建、分类、权限控制、变量配置、审核发布等功能。系统支持科室模板、个人模板、全院模板三级管理，提供模板变量自动填充能力，提高病历书写效率和质量一致性。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提供标准化病历模板，减少重复录入 | 模板使用率>=80% |
| 质量目标 | 确保病历内容规范一致 | 病历质控合格率>=90% |
| 管理目标 | 实现模板分级管理 | 全院/科室/个人模板分级管控 |
| 合规目标 | 模板审核发布流程符合规范 | 模板审核覆盖率100% |

### 1.3 功能范围

```
M03-01 病历模板管理
├── 模板创建
│   ├── 模板基础信息录入（名称、类型、适用范围）
│   ├── 模板内容编辑（所见即所得编辑器）
│   ├── 模板变量配置（患者信息、诊断、医生信息）
│   ├── 模板草稿保存
│   └── 模板复制创建
├── 模板分类管理
│   ├── 按文书类型分类（入院记录、病程记录、出院记录、知情同意书）
│   ├── 按适用范围分类（全院模板、科室模板、个人模板）
│   ├── 按病种分类（常见病模板、专科病种模板）
│   └── 模板关键词搜索
├── 模板权限控制
│   ├── 全院模板权限（系统管理员）
│   ├── 科室模板权限（科室管理员）
│   ├── 个人模板权限（仅本人）
│   └── 模板可见性控制
├── 模板审核发布
│   ├── 提交审核
│   ├── 审核通过发布
│   ├── 审核退回修改
│   ├── 模板停用
│   └── 审核历史记录
└── 模板变量管理
    ├── 患者信息变量（姓名、性别、年龄、住院号等）
    ├── 诊断变量（主诊断、主诊断编码、次诊断）
    ├── 医生信息变量（主治医师、住院医师、科主任）
    └── 时间变量（当前日期、当前时间、记录日期）
```

## 3. 用户故事

### US-M03-01-001 创建病历模板

**故事卡片**
```
作为 科室管理员
我想要 创建病历模板
以便于 医生使用标准化模板快速书写病历
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 创建病历模板

  Scenario: 创建入院记录模板
    Given 科室管理员已登录系统
    And 管理员有"内科"科室模板管理权限
    When 管理员进入模板管理页面
    And 管理员点击"新建模板"
    And 管理员选择模板类型"入院记录"
    And 管理员设置模板名称"内科入院记录模板"
    And 管理员选择适用范围"科室"
    And 管理员编辑模板内容
    And 管理员点击"保存模板"
    Then 系统创建模板记录，状态为"草稿"
    And 模板关联科室"内科"
    And 系统记录创建人和创建时间

  Scenario: 配置模板变量
    Given 管理员正在创建病历模板
    When 管理员在模板内容中插入变量"{患者姓名}"
    And 管理员在模板内容中插入变量"{入院日期}"
    Then 系统记录模板变量定义
    And 使用模板时自动填充对应患者信息

  Scenario: 模板内容必填校验
    Given 管理员正在保存模板
    When 模板名称为空或模板内容为空
    Then 系统提示"模板名称和内容不能为空"
    And 模板保存失败

  Scenario: 全院模板权限校验
    Given 管理员仅有科室模板管理权限
    When 管理员尝试创建"全院"范围的模板
    Then 系统提示"无权限创建全院模板"

  Scenario Boundary: 模板名称长度限制
    Given 管理员正在创建模板
    When 管理员输入模板名称超过100字符
    Then 系统提示"模板名称不能超过100字符"
```

### US-M03-01-003 模板审核发布

**故事卡片**
```
作为 科室主任
我想要 审核并发布病历模板
以便于 模板能够在科室正式使用
```

**优先级**: P0（MVP必需）

**故事点**: 3

**验收标准 (Gherkin)**
```gherkin
Feature: 模板审核发布

  Scenario: 提交模板审核
    Given 管理员已创建模板，状态为"草稿"
    When 管理员点击"提交审核"
    Then 模板状态变更为"待审核"
    And 系统通知科室主任审核

  Scenario: 审核通过发布
    Given 科室主任有模板审核权限
    And 有模板待审核
    When 科室主任审核模板内容
    And 科室主任点击"审核通过"
    Then 模板状态变更为"已发布"
    And 模板对科室医生可见可用

  Scenario: 审核退回修改
    Given 科室主任审核模板
    When 科室主任发现模板内容不符合规范
    And 科室主任点击"退回修改"
    And 输入退回原因"缺少体格检查项"
    Then 模板状态变更为"草稿"
    And 系统通知创建人修改

  Scenario: 停用模板
    Given 模板状态为"已发布"
    When 科室主任点击"停用模板"
    Then 模板状态变更为"已停用"
    And 模板不再显示在可选列表中
```

### US-M03-01-004 模板权限控制

**故事卡片**
```
作为 系统管理员
我想要 控制病历模板的管理权限
以便于 确保模板管理安全可控
```

**优先级**: P0（MVP必需）

**故事点**: 3

**验收标准 (Gherkin)**
```gherkin
Feature: 模板权限控制

  Scenario: 全院模板管理权限
    Given 用户有系统管理员权限
    When 用户进入模板管理
    Then 用户可创建、修改、删除全院模板

  Scenario: 科室模板管理权限
    Given 用户有科室管理员权限
    When 用户进入模板管理
    Then 用户只能创建、修改本科室模板
    And 用户无法操作其他科室模板

  Scenario: 个人模板权限
    Given 医生创建个人模板
    When 模板适用范围选择"个人"
    Then 模板仅对该医生可见可用
    And 其他医生无法查看或使用

  Scenario: 已发布模板修改限制
    Given 模板状态为"已发布"
    When 管理员尝试修改模板内容
    Then 系统提示"已发布模板不可修改，请复制后修改"
```

## 6. 状态机设计

### 6.1 模板状态流转

```
                    ┌─────────────────────────────────────────┐
                    │                                         │
                    ↓                                         │
┌──────┐  提交审核  ┌──────┐  审核通过  ┌──────┐  停用      │
│ 草稿 │ ────────→ │待审核│ ────────→ │已发布│ ────────→ ┌──────┐
│  1   │           │  2   │           │  3   │           │已停用│
└──────┘           └──────┘           └──────┘           │  4   │
    ↑                  │                                  └──────┘
    │                  │ 审核退回                              │
    │                  ↓                                      │
    └──────────────────┘                                      │
                                                              │
                         ←─────────────────────── 启用 ───────┘

状态说明:
- 1-草稿: 初始状态，可编辑、可删除、可提交审核
- 2-待审核: 等待审核，不可编辑、不可删除
- 3-已发布: 正式发布，不可编辑、不可删除，可使用
- 4-已停用: 停用状态，不可使用，可启用

状态流转规则:
- 草稿 → 待审核: 提交审核
- 待审核 → 已发布: 审核通过
- 待审核 → 草稿: 审核退回
- 已发布 → 已停用: 停用操作
- 已停用 → 已发布: 启用操作
```

### 7.2 关键业务规则详解

#### BR-EMR-001: 病历模板必填校验

创建或修改病历模板时，必须填写模板名称和模板内容。模板名称长度不超过100字符。

**触发时机**: 创建/修改病历模板时

**校验逻辑**:
- 模板名称不能为空
- 模板名称长度<=100字符
- 模板内容不能为空
- 科室模板必须选择适用科室

#### BR-EMR-002: 病历模板权限校验

全院模板需要系统管理员权限，科室模板需要科室管理员权限，个人模板仅本人可创建。

**触发时机**: 创建/修改病历模板时

**校验逻辑**:
```
CASE template.scope_type
    WHEN 1 THEN  // 全院模板
        IF NOT has_permission(current_user, 'SYSTEM_ADMIN') THEN
            RETURN error("无权限创建全院模板")
        END IF
    WHEN 2 THEN  // 科室模板
        IF NOT has_permission(current_user, 'DEPT_ADMIN') THEN
            RETURN error("无权限创建科室模板")
        END IF
        IF current_user.dept_id != template.dept_id THEN
            RETURN error("只能创建本科室模板")
        END IF
    WHEN 3 THEN  // 个人模板
        template.doctor_id = current_user.user_id
END CASE
```

#### BR-EMR-011: 模板数量上限校验

科室模板数量上限为50个，个人模板数量上限为20个。

**触发时机**: 创建模板时

**校验逻辑**:
- 科室模板: 查询该科室已有模板数量，>=50则拒绝创建
- 个人模板: 查询该医生已有模板数量，>=20则拒绝创建

#### BR-EMR-017: 模板审核发布流程

模板状态流转:草稿→待审核→已发布/已停用;已发布模板不可直接修改,需复制后重新审核。

**状态定义**: 1-草稿, 2-待审核, 3-已发布, 4-已停用

**状态流转矩阵**:

| 当前状态 | 可流转状态 | 触发操作 |
|----------|------------|----------|
| 草稿(1) | 待审核(2)、已停用(4) | 提交审核、停用 |
| 待审核(2) | 草稿(1)、已发布(3)、已停用(4) | 审核退回、审核通过、停用 |
| 已发布(3) | 已停用(4) | 停用 |
| 已停用(4) | 草稿(1) | 启用 |

#### BR-EMR-027: 病历模板管理权限控制

模板CRUD操作的权限控制规则。

**权限矩阵**:

| 操作 | 全院模板 | 科室模板 | 个人模板 |
|------|----------|----------|----------|
| 创建 | 系统管理员 | 科室管理员 | 任何医生 |
| 修改 | 系统管理员 | 本科室管理员 | 仅本人 |
| 删除 | 系统管理员 | 本科室管理员 | 仅本人 |
| 查看 | 全院可见 | 本科室可见 | 仅本人 |

#### BR-EMR-039: 模板变量自动填充规则

使用模板时自动填充患者信息、诊断、医生信息等变量。

**变量映射表**:
