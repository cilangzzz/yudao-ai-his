## 1. 功能概述

### 1.1 功能定位

病历归档是电子病历子系统的终点模块，负责病历的归档提交、锁定保护、病案检索、封存管理和借阅审批等核心功能。系统实现病历的长期保存（>=30年），支持医疗纠纷封存、病历借阅审批等合规管理，确保病历数据的完整性和法律效力。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 合规目标 | 病历保存期限>=30年 | 归档完整性100% |
| 安全目标 | 归档病历不可修改 | 归档锁定覆盖率100% |
| 管理目标 | 病案检索高效便捷 | 检索响应时间<=5秒 |
| 体验目标 | 借阅审批流程便捷 | 借阅审批时间<=24小时 |

## 6. 交互流程

### 6.1 病历归档流程

```
病历归档流程：

患者出院
    │
    ↓
系统检查病历状态
    │
    ├── 患者是否已出院？
    │   ├── 否 ──→ 提示"患者尚未出院，不可归档"
    │   └── 是 ──→ 继续
    │
    ├── 病历是否完成三级审签？
    │   ├── 否 ──→ 提示"病历未完成三级审签，不可归档"
    │   └── 是 ──→ 继续
    │
    ├── 病历质控是否合格？
    │   ├── 否 ──→ 提示"病历质控不合格，请先整改"
    │   └── 是 ──→ 继续
    │
    ↓
病案管理员提交归档
    │
    ↓
系统生成病案编号（BA+年份+科室代码+序号）
    │
    ↓
系统锁定所有病历文书（状态变更为"已归档"）
    │
    ↓
系统记录归档信息（文书数量、归档时间、操作人）
    │
    ↓
归档完成
```

### 6.2 病历封存流程

```
病历封存流程：

发生医疗纠纷/行政需求
    │
    ↓
病案管理员申请封存
    │
    ├── 选择病案编号
    ├── 选择封存类型（医疗纠纷封存/行政封存）
    ├── 填写封存原因
    ├── 设置封存期限（默认180天）
    └── 上传证明材料
    │
    ↓
提交封存申请（状态：待审批）
    │
    ↓
医疗院长/相关负责人审批
    │
    ├── 审批通过 ──→ 病历状态变更为"已封存"
    │   │              病历完全锁定
    │   │              记录封存审批信息
    │   │
    │   └── 审批拒绝 ──→ 封存申请被拒绝
    │                    通知申请人
    │
    ↓
封存生效
    │
    ↓
封存期间管理
    │
    ├── 到期提醒（提前7天）
    ├── 延期申请（如需要）
    └── 解封申请（医疗纠纷解决后）
    │
    ↓
解封审批流程
    │
    ├── 提交解封申请
    ├── 上传解封证明材料
    ├── 医疗院长审批
    │
    └── 审批通过 ──→ 病历状态恢复为"已归档"
```

### 6.3 病历借阅流程

```
病历借阅流程：

医生需要查阅/复印/借出病历
    │
    ↓
提交借阅申请
    │
    ├── 选择病案编号
    ├── 选择借阅类型（查阅/复印/借出）
    ├── 填写借阅原因
    │
    ↓
系统检查借阅权限
    │
    ├── 是否为本科室病历？
    │   ├── 是 ──→ 正常审批流程
    │   └── 否 ──→ 需要额外审批环节
    │
    ├── 是否为封存病历？
    │   ├── 是 ──→ 检查封存病历借阅权限
    │   └── 否 ──→ 继续
    │
    ↓
病案管理员审批
    │
    ├── 审批通过 ──→ 借阅状态变更为"已批准"
    │   │              设置归还期限（7天）
    │   │              通知借阅人
    │   │
    │   └── 审批拒绝 ──→ 借阅状态变更为"已拒绝"
    │                    通知借阅人并说明原因
    │
    ↓
借阅执行
    │
    ├── 查阅 ──→ 系统开放病历查看权限
    ├── 复印 ──→ 病案管理员处理复印
    └── 借出 ──→ 病案管理员移交病历实物
    │
    ↓
归还确认
    │
    ├── 借阅人归还病历
    ├── 病案管理员确认归还
    └── 系统更新借阅状态为"已归还"
    │
    ↓
借阅完成
```

### 8.2 关键业务规则详解

#### BR-EMR-008: 归档条件校验

病历归档前必须完成三级审签且质控合格。

**触发时机**: 病历归档时

**校验逻辑**:
1. 检查患者是否已出院
2. 检查所有病历是否完成三级审签
3. 检查病历质控是否合格（得分>=90分）
4. 检查必备文书是否完整（入院记录、首次病程记录、出院记录）

**失败处理**: 返回具体不满足条件的原因，阻止归档

#### BR-EMR-009: 封存病历保护校验

已封存病历禁止任何修改和删除操作，需解封审批后方可操作。

**触发时机**: 病历修改/删除操作时

**校验逻辑**:
1. 检查病历封存状态
2. 如已封存，检查是否在封存期限内
3. 如在封存期限内，检查是否有解封审批
4. 如无解封审批，拒绝操作

**失败处理**: 返回"病历已封存，禁止操作"，阻止修改/删除

#### BR-EMR-010: 借阅权限校验

借阅非本科室病历需要额外审批，封存病历借阅需要特殊权限。

**触发时机**: 病历借阅申请时

**校验逻辑**:
1. 检查是否为本科室病历，非本科室需额外审批
2. 检查封存状态，封存病历需特殊权限
3. 检查借阅类型权限（复印需复印权限、借出需借出权限）

**失败处理**: 返回权限不足或需要审批的提示

#### BR-EMR-038: 病案编号生成规则

病案编号格式：BA+年份+科室代码+6位序号

**示例**: BA2026010001

**生成逻辑**:
1. 获取年份（如2026）
2. 获取科室代码（如01）
3. 获取当年该科室最大序号+1
4. 序号6位补零（如000001）
5. 组装编号
