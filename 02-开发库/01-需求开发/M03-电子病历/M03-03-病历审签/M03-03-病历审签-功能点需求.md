## 1. 功能概述

### 1.1 功能定位

病历审签是电子病历子系统的核心质量管控模块，实现病历三级医师审签流程，集成CA电子签名，确保病历的真实性和法律效力。系统支持多级审签流转、审签退回修改、审签历史记录等功能，符合国家电子病历应用水平分级评价标准中对病历审签的要求。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 合规目标 | 实现病历三级医师审签 | 审签覆盖率100% |
| 安全目标 | 所有审签必须使用CA电子签名 | CA签名覆盖率100% |
| 效率目标 | 缩短病历审签流转时间 | 平均审签周期<=48小时 |
| 质量目标 | 提高病历审签通过率 | 首次审签通过率>=85% |

## 3. 用户故事

### US-M03-03-001 提交病历审签

**故事卡片**
```
作为 住院医师
我想要 提交病历进行审签
以便于 上级医师审核确认病历质量
```

**优先级**: P0（MVP必需）

**故事点**: 3

**验收标准 (Gherkin)**
```gherkin
Feature: 提交病历审签

  Scenario: 提交一级审签
    Given 住院医师已完成病历编辑
    And 病历必填项已填写完整
    When 医生点击"提交审签"
    Then 病历状态变更为"待审签"
    And 系统通知主治医师审核

  Scenario: 病历完整性校验
    Given 医生点击提交审签
    When 病历存在必填项未填写
    Then 系统提示未填写的必填项
    And 提交审签失败

  Scenario: 提交指定审签人
    Given 病历需特定上级医师审核
    When 医生提交审签时选择审签人"李主任"
    Then 系统通知指定审签人

  Scenario Exception: 审签服务异常
    Given 医生提交审签
    When 审签服务无响应
    Then 系统提示"审签服务异常，请稍后重试"

  Scenario Boundary: 重复提交审签
    Given 病历状态为"待审签"
    When 医生尝试再次提交审签
    Then 系统提示"病历正在审签中，请勿重复提交"
```

### US-M03-03-002 上级医师审核

**故事卡片**
```
作为 主治医师/主任
我想要 审核下级医师提交的病历
以便于 确认病历质量和诊疗方案
```

**优先级**: P0（MVP必需）

**故事点**: 5

**验收标准 (Gherkin)**
```gherkin
Feature: 上级医师审核

  Scenario: 查看待审签列表
    Given 主治医师已登录
    When 医师进入病历审签页面
    Then 系统显示待审签病历列表
    And 列表显示患者姓名、文书类型、提交时间

  Scenario: 审签通过
    Given 主治医师查看病历内容
    When 医师点击"审核通过"
    And 插入CA电子签名
    Then 病历审核状态变更为"一级审签通过"
    And 系统记录审签人和审签时间

  Scenario: 审签退回
    Given 主治医师发现病历问题
    When 医师点击"退回修改"
    And 输入退回原因"现病史描述不完整"
    Then 病历状态变更为"审签退回"
    And 系统通知住院医师修改

  Scenario: CA电子签名
    Given 医师审核通过病历
    When 医师点击"插入签名"
    Then 系统调用CA签名服务
```

#### BR-EMR-012: 审签级别权限校验

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-EMR-012 |
| 规则名称 | 审签级别权限校验 |
| 规则类型 | VAL (校验规则) |
| 适用对象 | 病历审签(emr_sign) |
| 规则描述 | 一级审签需主治医师权限，二级审签需副主任/主任医师权限，三级审签需科主任/医疗院长权限 |
| 触发时机 | 病历审签时 |

**校验逻辑**:
```
// 伪代码
signer = SELECT * FROM user WHERE user_id = :signer_id

CASE sign_level
    WHEN 1 THEN  // 一级审签
        IF signer.title NOT IN ('主治医师', '副主任医师', '主任医师') THEN
            RETURN error("一级审签需要主治医师及以上职称")
        END IF
        
    WHEN 2 THEN  // 二级审签
        IF signer.title NOT IN ('副主任医师', '主任医师') THEN
            RETURN error("二级审签需要副主任医师及以上职称")
        END IF
        
    WHEN 3 THEN  // 三级审签
        IF signer.title NOT IN ('主任医师') THEN
            RETURN error("三级审签需要主任医师职称")
        END IF
        IF NOT has_permission(signer, 'EMR_LEVEL3_SIGN') THEN
            RETURN error("无三级审签权限")
        END IF
END CASE

// 检查是否已签过同级
existing_sign = SELECT * FROM emr_sign 
                WHERE document_id = :document_id 
                  AND sign_level = :sign_level
IF existing_sign IS NOT NULL THEN
    RETURN error("该级别审签已完成")
END IF

RETURN success()
```

#### BR-EMR-018: 三级医师审签流程

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-EMR-018 |
| 规则名称 | 三级医师审签流程 |
| 规则类型 | FLOW (流转规则) |
| 适用对象 | 病历文书(emr_document) |
| 规则描述 | 病历审签流程：草稿->待审签->一级审签->二级审签->三级审签->已审签；任何一级可退回修改 |
| 触发时机 | 病历审签操作时 |

**校验逻辑**:
```
// 伪代码
// 状态定义: 1-草稿, 2-待审签, 3-一级审签通过, 4-二级审签通过, 
//          5-三级审签通过(已审签), 6-审签退回
// 审签流程
sign_flow = [
    {'level': 1, 'status_before': 2, 'status_after': 3, 'required_title': '主治医师'},
    {'level': 2, 'status_before': 3, 'status_after': 4, 'required_title': '副主任医师'},
    {'level': 3, 'status_before': 4, 'status_after': 5, 'required_title': '主任医师'}
]

// 检查当前状态是否允许审签
IF document.document_status != sign_flow[sign_level-1].status_before THEN
    RETURN error("病历当前状态不允许此级别审签")
END IF

// 检查审签人资格
IF current_user.title NOT IN allowed_titles THEN
    RETURN error("审签人职称不符合要求")
END IF

// 审签通过
IF sign_result = 'APPROVED' THEN
    document.document_status = sign_flow[sign_level-1].status_after
    document.audit_level = sign_level
    
    // 记录签名
    INSERT INTO emr_sign (document_id, sign_level, signer_id, sign_result, sign_time)
    VALUES (:document_id, :sign_level, current_user.user_id, 'APPROVED', NOW())
    
    // 通知下一级审签人
    IF sign_level < 3 THEN
        next_signers = GET_available_signers(sign_level + 1, document.dept_id)
        SEND_notification(next_signers, "有病历待审签")
    ELSE
        // 三级审签完成，锁定病历
        document.lock_time = NOW()
        document.lock_user_id = current_user.user_id
    END IF
END IF

// 审签退回
IF sign_result = 'REJECTED' THEN
    document.document_status = 6  // 审签退回
    
    // 记录退回原因
    INSERT INTO emr_sign (document_id, sign_level, signer_id, sign_result, reject_reason, sign_time)
    VALUES (:document_id, :sign_level, current_user.user_id, 'REJECTED', :reject_reason, NOW())
    
    // 通知作者修改
    SEND_notification(document.author_id, "病历审签退回:" + reject_reason)
END IF

RETURN success()
```

#### BR-EMR-028: 病历审签权限控制

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-EMR-028 |
| 规则名称 | 病历审签权限控制 |
| 规则类型 | PERM (权限规则) |
| 适用对象 | 病历审签(emr_sign) |
| 规则描述 | 一级审签需主治医师，二级需副主任医师，三级需主任医师/科主任；不可审签自己书写的病历 |
| 触发时机 | 病历审签时 |

**校验逻辑**:
```
// 伪代码
// 检查是否为病历作者
IF current_user.user_id = document.author_id THEN
    RETURN error("不可审签自己书写的病历")
END IF

// 检查审签级别权限
CASE sign_level
    WHEN 1 THEN
        required_titles = ['主治医师', '副主任医师', '主任医师']
        IF current_user.title NOT IN required_titles THEN
            RETURN error("一级审签需要主治医师及以上职称")
        END IF
        
    WHEN 2 THEN
        required_titles = ['副主任医师', '主任医师']
        IF current_user.title NOT IN required_titles THEN
            RETURN error("二级审签需要副主任医师及以上职称")
        END IF
        
    WHEN 3 THEN
        IF current_user.title != '主任医师' THEN
            RETURN error("三级审签需要主任医师职称")
        END IF
        IF NOT has_permission(current_user, 'EMR_LEVEL3_SIGN') THEN
            RETURN error("无三级审签权限")
        END IF
END CASE

// 检查科室权限
IF document.dept_id != current_user.dept_id THEN
    IF NOT has_permission(current_user, 'CROSS_DEPT_SIGN') THEN
        RETURN error("无权限审签其他科室病历")
    END IF
END IF

RETURN success()
```

#### BR-EMR-043: CA电子签名集成规则

| 属性 | 内容 |
|------|------|
| 规则编号 | BR-EMR-043 |
| 规则名称 | CA电子签名集成规则 |
| 规则类型 | INT (集成规则) |
| 适用对象 | 病历审签、知情同意书 |
| 规则描述 | 所有审签操作必须通过CA电子签名服务验证，记录签名数据和证书编号 |
| 触发时机 | 病历审签/知情同意书签署时 |

**校验逻辑**:
```
// 伪代码
// 检查用户CA证书
ca_cert = SELECT * FROM user_ca_cert 
          WHERE user_id = current_user.user_id 
            AND cert_status = 'VALID'
            AND expire_time > NOW()

IF ca_cert IS NULL THEN
    RETURN error("CA证书无效或已过期，请更新证书")
END IF

// 调用CA签名服务
sign_request = {
    'user_id': current_user.user_id,
    'cert_no': ca_cert.cert_no,
    'document_id': document.document_id,
    'sign_content': HASH(document.document_content),
    'sign_time': NOW()
}

TRY
    ca_result = CALL ca_sign_service.sign(sign_request)
    
    IF ca_result.success THEN
        // 记录签名信息
        INSERT INTO emr_sign (document_id, sign_level, signer_id, signer_name,
                              signer_title, sign_result, ca_signature, ca_cert_no, sign_time)
        VALUES (:document_id, :sign_level, current_user.user_id, current_user.user_name,
                current_user.title, 'APPROVED', ca_result.signature_data, 
                ca_cert.cert_no, NOW())
        
        RETURN success()
    ELSE
        RETURN error("CA签名失败:" + ca_result.error_message)
    END IF
CATCH exception
    LOG error("CA签名服务异常", exception)
    RETURN error("CA签名服务异常，请稍后重试")
END TRY
```

## 8. 状态机设计

### 8.1 病历审签状态机

```
状态定义:
- S1: 草稿 (DRAFT)
- S2: 待审签 (PENDING_SIGN)
- S3: 一级审签通过 (LEVEL1_APPROVED)
- S4: 二级审签通过 (LEVEL2_APPROVED)
- S5: 已审签 (SIGNED) - 三级审签通过
- S6: 审签退回 (REJECTED)

状态流转图:
                    +------------------------------------------+
                    |                                          |
                    v                                          |
+--------+    +--------+    +--------+    +--------+    +--------+
|  草稿  |-->|待审签  |-->|一级通过|-->|二级通过|-->|已审签  |
|  S1   |    |  S2   |    |  S3   |    |  S4   |    |  S5   |
+--------+    +--------+    +--------+    +--------+    +--------+
                   |              |              |
                   |              |              |
                   v              v              v
              +----------------------------------------+
              |              审签退回                   |
              |                S6                      |
              +----------------------------------------+
                              |
                              | 修改后重新提交
                              v
                        返回待审签(S2)

状态转换表:
| 当前状态 | 事件 | 目标状态 | 触发条件 |
|----------|------|----------|----------|
| S1(草稿) | 提交审签 | S2(待审签) | 病历完整性校验通过 |
| S2(待审签) | 一级审签通过 | S3(一级通过) | 主治医师审核通过 |
| S2(待审签) | 审签退回 | S6(审签退回) | 任意级别审签人退回 |
| S3(一级通过) | 二级审签通过 | S4(二级通过) | 副主任/主任医师审核通过 |
| S3(一级通过) | 审签退回 | S6(审签退回) | 二级审签人退回 |
| S4(二级通过) | 三级审签通过 | S5(已审签) | 科主任/医疗院长审核通过 |
| S4(二级通过) | 审签退回 | S6(审签退回) | 三级审签人退回 |
| S6(审签退回) | 重新提交 | S2(待审签) | 住院医师修改后重新提交 |
```

### 8.2 审签级别状态说明

| 状态码 | 状态名称 | 说明 | 可执行操作 |
|--------|----------|------|------------|
| 1 | 草稿 | 病历正在编辑中 | 保存、提交审签 |
| 2 | 待审签 | 等待一级审签 | 查看详情（审签人） |
| 3 | 一级审签通过 | 等待二级审签 | 查看详情（审签人） |
| 4 | 二级审签通过 | 等待三级审签 | 查看详情（审签人） |
| 5 | 已审签 | 三级审签完成，病历锁定 | 归档、查看 |
| 6 | 审签退回 | 病历需要修改 | 修改、重新提交（作者） |
