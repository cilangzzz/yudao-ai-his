## 1. 功能概述

### 1.1 功能定位

病历编辑模块是电子病历子系统的核心功能，提供病历文书的编辑功能，支持所见即所得编辑器，实现结构化录入和自由文本编辑，可引用检验结果、影像报告，支持草稿保存和打印预览。系统覆盖入院记录、病程记录、出院记录、知情同意书等多种病历文书类型，满足住院医师日常病历书写需求。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 提供模板化病历编辑，提高书写效率 | 模板使用率>=80% |
| 质量目标 | 结构化录入保证病历规范性 | 必填项覆盖率100% |
| 时限目标 | 时限提醒督促按时完成病历 | 时限达标率>=90% |
| 安全目标 | 自动保存防止数据丢失 | 数据丢失率<1% |
| 引用目标 | 支持检验影像结果引用 | 引用功能可用 |

## 6. 交互流程

### 6.1 入院记录编辑流程

```
入院记录编辑流程：

1. 医生选择患者
   │
   ↓
2. 进入病历编辑页面
   │
   ↓
3. 选择入院记录模板
   │
   ├── 系统自动填充患者基本信息
   ├── 系统自动填充入院日期
   ├── 系统自动填充科室、主治医师
   │
   ↓
4. 医生编辑病历内容
   │
   ├── 录入主诉
   ├── 录入现病史
   ├── 录入既往史
   ├── 录入体格检查
   ├── 录入初步诊断（关联ICD-10）
   ├── 可选：引用检验结果
   ├── 可选：引用影像报告
   │
   ↓
5. 保存草稿/提交审签
   │
   ├── 点击"保存草稿" → 病历状态为"草稿"
   │   ├── 系统记录保存时间
   │   ├── 系统记录字数统计
   │   └── 可继续编辑
   │
   ├── 点击"提交审签" → 病历状态为"待审签"
   │   ├── 系统校验必填项完整性
   │   ├── 校验主诉>=10字符
   │   ├── 校验现病史>=100字符
   │   ├── 校验诊断关联ICD-10
   │   ├── 校验时限是否超时
   │   └── 校验通过后提交审签
   │
   ↓
6. 系统通知上级医师审核
```

## 7. 状态机设计

### 7.1 病历文书状态机

```
病历文书状态流转：

┌─────────┐     编辑/保存     ┌─────────┐
│  草稿   │ ───────────────→ │  草稿   │
│  (1)    │                   │  (1)    │
└─────────┘                   └─────────┘
     │
     │ 提交审签（必填项校验通过）
     ↓
┌─────────┐     一级审签通过   ┌─────────┐
│ 待审签  │ ───────────────→ │ 一级审签 │
│  (2)    │                   │ 通过(3)  │
└─────────┘                   └─────────┘
     │                              │
     │ 审签退回                      │ 二级审签通过
     ↓                              ↓
┌─────────┐                   ┌─────────┐
│审签退回 │                   │ 二级审签 │
│  (6)    │ ──修改后重新提交──→ │ 通过(4)  │
└─────────┘                   └─────────┘
                                   │
                                   │ 三级审签通过
                                   ↓
                              ┌─────────┐
                              │ 已审签  │
                              │  (5)    │
                              └─────────┘
                                   │
                                   │ 归档
                                   ↓
                              ┌─────────┐
                              │ 已归档  │
                              │  (7)    │ ──→ 锁定不可修改
                              └─────────┘
                                   │
                                   │ 封存
                                   ↓
                              ┌─────────┐
                              │ 已封存  │
                              │  (8)    │ ──→ 完全锁定
                              └─────────┘

状态说明：
- 草稿(1)：病历正在编辑，可随时修改
- 待审签(2)：已提交审签，等待上级医师审核
- 一级审签通过(3)：主治医师审核通过
- 二级审签通过(4)：副主任/主任医师审核通过
- 已审签(5)：三级审签完成，病历锁定
- 审签退回(6)：上级医师退回修改
- 已归档(7)：病历归档，锁定不可修改
- 已封存(8)：医疗纠纷封存，完全锁定
```

### 8.2 关键业务规则详解

#### BR-EMR-003: 病历必填项校验

病历提交审签前必须校验必填项完整性，不同文书类型有不同的必填项要求。

**触发时机**: 病历提交审签时

**必填项规则**:
| 文书类型 | 必填项 | 最小字符要求 |
|----------|--------|--------------|
| 入院记录 | 主诉、现病史、既往史、体格检查、初步诊断 | 主诉>=10字符，现病史>=100字符 |
| 首次病程记录 | 病例特点、诊断依据、鉴别诊断、诊疗计划 | 病例特点>=50字符 |
| 出院记录 | 入院诊断、诊疗经过、出院诊断、出院医嘱 | 诊疗经过>=100字符 |
| 知情同意书 | 手术名称、手术风险、患者签名 | - |

#### BR-EMR-005: 病历内容长度校验

各类病历文书内容有最小字符要求，保证病历质量。

**触发时机**: 病历保存/提交时

**校验逻辑**:
```
IF LENGTH(主诉) < 10 THEN
    RETURN warning("主诉内容建议不少于10字符")
END IF

IF LENGTH(现病史) < 100 THEN
    RETURN warning("现病史内容建议不少于100字符")
END IF

IF LENGTH(病例特点) < 50 THEN
    RETURN warning("病例特点内容建议不少于50字符")
END IF
```

#### BR-EMR-035: 时限质控扣分计算

各类病历文书有完成时限要求，超时按规则扣分。

**时限规则**:
| 文书类型 | 完成时限 | 扣分规则 |
|----------|----------|----------|
| 入院记录 | 入院后24小时内 | 超时扣5分/小时，上限30分 |
| 首次病程记录 | 入院后8小时内 | 超时扣5分/小时，上限20分 |
| 日常病程记录 | 每日至少1次 | 缺一日扣5分 |
| 出院记录 | 出院后48小时内 | 超时扣5分/小时，上限20分 |
| 知情同意书 | 操作前签署 | 未签署扣20分 |

#### BR-EMR-026: 病历自动保存流程

病历编辑超过5分钟未手动保存，系统自动保存草稿。

**触发时机**: 编辑超时/页面关闭时

**执行逻辑**:
```
SET_INTERVAL(5 MINUTES, function() {
    IF is_editing AND last_save_time < NOW() - 5 MINUTES THEN
        SAVE_document_draft({
            'document_id': current_document.document_id,
            'content': editor.get_content(),
            'auto_save': true
        })
        SHOW_toast("已自动保存草稿")
        last_save_time = NOW()
    END IF
})
```
