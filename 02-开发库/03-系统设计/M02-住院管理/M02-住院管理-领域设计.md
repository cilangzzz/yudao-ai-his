## 1. 界限上下文

### 1.1 住院上下文 (InpatientContext)

| 属性 | 值 |
|------|------|
| 上下文编号 | BC-002 |
| 英文标识 | InpatientContext |
| 边界范围 | 住院业务域 |

```
│   - AdmissionService (入院服务)                                 │
│   - OrderService (医嘱服务)                                     │
│   - BedAllocationService (床位分配服务)                         │
│   - ClosedLoopMedicationService (闭环给药服务)                  │
│   - DischargeService (出院服务)                                 │
├─────────────────────────────────────────────────────────────────┤
│ 领域事件:                                                       │
│   - PatientAdmitted (患者入院)                                  │
│   - OrderCreated (医嘱开立)                                     │
│   - OrderVerified (医嘱审核)                                    │
│   - MedicationAdministered (给药完成)                           │
│   - PatientDischarged (患者出院)                                │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 业务规则约束

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-IP-001 | 入院关联诊断 | 入院必须有初步诊断 |
| BR-IP-003 | 床位分配校验 | 根据性别/病种/感染因素分配 |
| BR-IP-005 | 住院医嘱CDS校验 | 医嘱开立时进行CDS校验 |
| BR-IP-007 | 医嘱状态流转 | 开立→审核→执行→完成 |
| BR-IP-010 | 闭环给药双重核对 | 腕带+条码双重核对 |
| BR-IP-011 | 腕带条码不匹配停止 | 不匹配时禁止给药 |
| BR-IP-012 | 出院前停止长期医嘱 | 出院前必须停止所有长期医嘱 |

```
│                  [领域服务: 闭环给药]                            │
├─────────────────────────────────────────────────────────────────┤
│ 职责描述:                                                       │
│   实现HIMSS EMRAM Stage 5级闭环给药流程：                       │
```

### 5.1 住院上下文与其他上下文的关系

| 上游上下文 | 下游上下文 | 关系模式 | 集成方式 | 说明 |
|------------|------------|----------|----------|------|
| 共享内核 | 住院上下文 | 共享内核 (SK) | 直接引用 | 患者聚合、领域事件定义共享 |

| 住院上下文 | 财务上下文 | 客户-供应商 (CF) | 领域事件 | 医嘱执行自动记账 |
| 住院上下文 | 药品上下文 | 防腐层 (ACL) | 领域事件+API | 医嘱转处方，库存校验 |
| 检验上下文 | 住院上下文 | 防腐层 (ACL) | 领域事件 | 危急值通知、报告回传 |
| 影像上下文 | 住院上下文 | 防腐层 (ACL) | 领域事件 | 报告回传 |
| 住院上下文 | 电子病历上下文 | 开放主机服务 (OHS) | 领域事件 | 病历数据归档 |
